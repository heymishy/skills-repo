## Coverage gaps

**Partial gap — AC4 network dependency:** A full integration test of `fetch()` with a real git remote would require network access and an actual remote repository. Unit tests use a mocked `execSync` to validate the command string. The manual verification scenario covers the real git path.

**Risk:** 🟢 Low — the command string is testable; real-network behaviour is standard git and requires no custom code.

## Unit Tests

### T1 — `init()` creates `.github/skills/` directory on fresh repo

- **Verifies:** AC1
- **Precondition:** Temp dir with no `.github/skills/` directory
- **Action:** Call `init(tempRoot)`; assert `fs.existsSync(path.join(tempRoot, '.github', 'skills'))` returns true
- **Expected result:** Directory created
- **Edge case:** No

### T2 — `init()` writes stub lockfile with correct schema

- **Verifies:** AC1
- **Precondition:** T1 setup
- **Action:** Read lockfile at `<tempRoot>/.github/skills/skill-lockfile.json`; parse JSON; assert `schemaVersion === "1.0.0"`; assert `skills` is an empty array; assert `pinnedAt` is a non-empty string
- **Expected result:** All assertions pass — stub lockfile is a valid empty-skills snapshot
- **Edge case:** No

### T3 — `init()` returns `{status:'ok', command:'init', created:true}` on fresh repo

- **Verifies:** AC1
- **Precondition:** T1 setup
- **Action:** Capture return value of `init(tempRoot)`; assert `result.status === 'ok'`; assert `result.command === 'init'`; assert `result.created === true`
- **Expected result:** All assertions pass
- **Edge case:** No

### T4 — `init()` returns `created: false` when lockfile already exists

- **Verifies:** AC2
- **Precondition:** Temp dir with pre-existing lockfile at `.github/skills/skill-lockfile.json` containing `{schemaVersion:"1.0.0",pinnedAt:"old",skills:[{skill:"existing",path:"...",sha256:"abc"}]}`
- **Action:** Call `init(tempRoot)`; assert `result.created === false`
- **Expected result:** Assertion passes
- **Edge case:** No

### T5 — `init()` does not modify existing lockfile content when it already exists

- **Verifies:** AC2 (non-destructive when lockfile present)
- **Precondition:** T4 setup — pre-existing lockfile with known content
- **Action:** Record lockfile content before `init()`; call `init(tempRoot)`; re-read lockfile; assert content is identical (or at minimum the `skills` array entry is unchanged)
- **Expected result:** Assertion passes — existing lockfile not clobbered
- **Edge case:** No

### T6 — `fetch()` with `skills_upstream.remote: null` returns `{status:'not-configured'}` without throwing

- **Verifies:** AC3
- **Precondition:** Temp dir with context.yml `skills_upstream:\n  remote: null`
- **Action:** Call `fetch(tempRoot)` in try/catch; assert no exception thrown; assert `result.status === 'not-configured'`; assert `result.command === 'fetch'`
- **Expected result:** All assertions pass — graceful no-op
- **Edge case:** No

### T7 — `fetch()` with no `skills_upstream` key returns `{status:'not-configured'}` without throwing

- **Verifies:** AC3 (absent config case)
- **Precondition:** Temp dir with context.yml that has no `skills_upstream` key at all
- **Action:** Call `fetch(tempRoot)` in try/catch; assert no exception; assert `result.status === 'not-configured'`
- **Expected result:** Assertion passes — handles missing key gracefully
- **Edge case:** No

### T8 — `fetch()` with valid remote executes `git fetch <remote>` via execSync

- **Verifies:** AC4 (command executed)
- **Precondition:** Temp dir with context.yml `skills_upstream:\n  remote: https://github.com/heymishy/skills-repo.git`; execSync mocked to capture called command and return empty string
- **Action:** Replace `execSync` in cli-adapter module with a spy; call `fetch(tempRoot)`; assert spy was called with a string containing `git fetch` and `https://github.com/heymishy/skills-repo.git`
- **Expected result:** Assertion passes — confirms ADR-004 compliance (remote URL read from context.yml)
- **Edge case:** No

### T9 — `fetch()` with valid remote returns `{status:'ok', command:'fetch', remote:'<url>'}`

- **Verifies:** AC4 (return value)
- **Precondition:** T8 setup
- **Action:** Assert `result.status === 'ok'`; assert `result.command === 'fetch'`; assert `result.remote === 'https://github.com/heymishy/skills-repo.git'`
- **Expected result:** All assertions pass
- **Edge case:** No

## NFR Tests

### NFR1 — `init()` completes in under 1 second

- **Verifies:** Performance NFR (init < 1s)
- **Precondition:** Temp dir (no pre-existing files)
- **Action:** Record `Date.now()` before call; call `init(tempRoot)`; assert elapsed < 1000ms
- **Expected result:** Assertion passes

### NFR2 — context.yml is read via ADR-004 path; no credentials appear in return value

- **Verifies:** Security NFR (no credentials in config returned)
- **Precondition:** context.yml containing a fake token: `tools:\n  api_token: sk-faketoken456`; valid `skills_upstream.remote`
- **Action:** Call `fetch(tempRoot)`; assert `JSON.stringify(result)` does NOT contain `sk-faketoken456`
- **Expected result:** Return value contains only `status`, `command`, `remote` fields — no config dump
