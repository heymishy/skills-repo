### Scenario 1: `pin()` creates lockfile with correct structure

**Covers:** AC1, AC4

**Steps:**
1. In a terminal, start a Node.js REPL: `node`
2. Run the following:
   ```js
   const os = require('os'), fs = require('fs'), path = require('path');
   const { pin } = require('./src/enforcement/cli-adapter.js');
   const tmp = fs.mkdtempSync(path.join(os.tmpdir(), 'p11-'));
   fs.mkdirSync(path.join(tmp, '.github', 'skills', 'my-skill'), { recursive: true });
   fs.writeFileSync(path.join(tmp, '.github', 'skills', 'my-skill', 'SKILL.md'), '# My Skill\n\nContent here.');
   const r1 = pin(tmp);
   const lockfile = JSON.parse(fs.readFileSync(path.join(tmp, '.github', 'skills', 'skill-lockfile.json'), 'utf8'));
   console.log(JSON.stringify(lockfile, null, 2));
   // AC4 idempotency
   const r2 = pin(tmp);
   const lockfile2 = JSON.parse(fs.readFileSync(path.join(tmp, '.github', 'skills', 'skill-lockfile.json'), 'utf8'));
   console.log('sha256 match:', lockfile.skills[0].sha256 === lockfile2.skills[0].sha256);
   ```
3. Observe the printed lockfile and idempotency result.

**Expected outcome:**
> Lockfile printed with `schemaVersion: "1.0.0"`, `pinnedAt` as an ISO-8601 string (e.g. `"2026-04-28T..."`), `skills` array with one entry containing `skill` (string), `path` (string ending in `SKILL.md`), and `sha256` (64-char hex string). `sha256 match: true` printed at the end.

**Steps:**

   ```js
   const os = require('os'), fs = require('fs'), path = require('path');
   const { pin } = require('./src/enforcement/cli-adapter.js');
   const tmp = fs.mkdtempSync(path.join(os.tmpdir(), 'p11-perf-'));
   ```
