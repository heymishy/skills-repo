## MEDIUM findings — resolve or acknowledge in /decisions

- **1-M1** [C — AC quality] — AC4 describes implementation mechanism rather than observable behaviour. The "then" clause reads: "then it executes `git fetch <remote>` using `child_process.execSync` and returns `{ status: 'ok', command: 'fetch', remote: '<remote>' }`." The implementation mechanism (`child_process.execSync`) is internal and untestable at the unit level without mocking internals. The observable behaviour is the return value and the side effect (git fetch executed). The "then" clause should state only the observable outcome: "then it returns `{ status: 'ok', command: 'fetch', remote: '<remote>' }` and invokes a `git fetch` command for `<remote>`." The test would mock `execSync` and assert it was called with the right argument — that is the observable, not the specific Node.js API used.
  Risk if proceeding: Test writer may feel compelled to test `child_process` module selection specifically rather than the functional outcome, creating brittle tests tied to implementation detail.
  To acknowledge: run /decisions, category RISK-ACCEPT.
