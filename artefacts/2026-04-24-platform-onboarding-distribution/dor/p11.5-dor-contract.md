# DoR Contract: Implement `init` and `fetch` commands in CLI adapter

**Story:** `artefacts/2026-04-24-platform-onboarding-distribution/stories/p11.5-cli-init-fetch.md`
**Date approved:** 2026-04-28

---

## What will be built

The `init()` and `fetch()` stub functions in `src/enforcement/cli-adapter.js` will be replaced with real implementations. `init()` creates `.github/skills/` if it does not exist and writes a minimal stub lockfile at `.github/skills/skill-lockfile.json` (path per ADR-016). `fetch()` reads the remote URL from `.github/context.yml` (per ADR-004), runs a `git clone` via `child_process.execSync`, and copies fetched SKILL.md files into `.github/skills/`. Both implementations use Node.js built-ins only — no new npm dependencies.

## What will NOT be built

- SHA-256 hashing or lockfile hash entries — this is p11.4 (`pin()`)
- Any authentication or token handling for the remote URL — the remote is expected to be a public or already-authenticated git URL in context.yml
- CLI argument parsing, routing, or a user-facing command runner

## AC verification

| AC | Test approach | Type |
|----|---------------|------|
| AC1 — init() creates .github/skills/ if absent | tmp dir test: call init() on empty dir; assert .github/skills/ exists | Unit |
| AC2 — init() writes stub lockfile if no lockfile exists | tmp dir test: call init(); assert .github/skills/skill-lockfile.json written with version:1 and empty skills:{} | Unit |
| AC3 — init() is idempotent when called twice | tmp dir test: call init() twice; assert no error thrown and lockfile not overwritten if already present | Unit |
| AC4 — fetch() reads remote URL from context.yml and invokes execSync | Mock execSync; provide context.yml with skills_upstream.url; call fetch(); assert execSync called with correct git command | Unit |
| AC5 — fetch() returns error when no remote URL configured | Provide context.yml with null remote; call fetch(); assert returned object has error field | Unit |

## Assumptions

1. `fetch()` performs a fresh `git clone` to a temp directory on each call — it does not use `git fetch` on an existing repo. The temp directory is cleaned up after copying.
2. Context.yml remote URL field is `skills_upstream.url` (consistent with existing `context.yml` structure). If the field does not exist or is null, return error.
3. `init()` does NOT overwrite an existing lockfile — it is idempotent.
4. `p11.4` will have already established the lockfile format and path before this story is dispatched — `init()` writes only the minimal stub `{ "version": 1, "generated": "...", "skills": {} }` that conforms to that format.
