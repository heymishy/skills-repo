# DoR Contract: Implement H-GOV hard block in `/definition-of-ready` SKILL.md

**Story:** `artefacts/2026-04-24-platform-onboarding-distribution/stories/p11.3-dor-hgov-block.md`
**Date approved:** 2026-04-28

---

## What will be built

A new `H-GOV` section will be added to `.github/skills/definition-of-ready/SKILL.md` after the existing H9 hard block. H-GOV reads the discovery artefact associated with the story, checks for a populated `## Approved By` section, and fails with actionable guidance if the section is absent, empty, or contains engineering-only entries. A `<!-- h-gov-block -->` marker will be added inside the section. The `h-gov-block` string will be added to the definition-of-ready contract in `check-skill-contracts.js`.

## What will NOT be built

- Changes to any template file (`discovery.md`, `benefit-metric.md`, etc.) — attribution sections are delivered by p11.1 and p11.2
- H-GOV checks for `/benefit-metric` attribution — not in scope for this story
- Schema validation on `approved-by` entries (e.g. regex matching of role strings) — this story checks structural presence only, relying on the bounded attribution model ADR for role interpretation
- Any changes to H1–H9 (or their numbering)

## Assumptions

1. The H-GOV block reads the discovery artefact path from the story's `Discovery reference` field (or similar pointer). If the field is absent, H-GOV fails safe (treat as missing section — AC3 path).
2. Non-engineering role detection: the bounded attribution model ADR (decisions.md) provides the authoritative role strings. The coding agent MUST read that ADR before implementing role detection.
3. The `h-gov-block` marker is inside the `H-GOV` section prose and does not need to be on its own line.
4. H-GOV is processed by the AI agent running `/definition-of-ready`, not by automated CI — there is no file-read API to stub. Test approach is file content assertion (SKILL.md contains correct instructions).

`schemaDepends: []` — H-GOV reads artefact files directly; no new `pipeline-state.json` fields are introduced.

2. Bounded attribution model ADR is written in decisions.md
