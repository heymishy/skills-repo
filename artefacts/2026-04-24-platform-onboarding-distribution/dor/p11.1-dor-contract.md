# DoR Contract: Add attribution sections to `/discovery` template and SKILL.md

**Story:** `artefacts/2026-04-24-platform-onboarding-distribution/stories/p11.1-discovery-attribution-fields.md`
**Date approved:** 2026-04-28

---

## What will be built

Three new sections (`## Contributors`, `## Reviewers`, `## Approved By`) will be added to `.github/templates/discovery.md`, each with an HTML comment explaining expected content and the distinction between engineering and non-engineering roles. The `/discovery` SKILL.md completion step will be updated to explicitly instruct the operator to populate all three fields before marking a discovery artefact Approved, and to warn that an empty `approved-by` section will trigger H-GOV at DoR. A required contract marker string (`<!-- approved-by-required -->`) will be added to the SKILL.md and registered in `check-skill-contracts.js`.

## What will NOT be built

- H-GOV hard block in `/definition-of-ready` SKILL.md — this is p11.3 and must not be included
- Attribution fields in `/benefit-metric` template — this is p11.2
- Any enforcement or validation logic — the change is purely additive template and instruction text
- Backfilling of existing discovery artefacts in `artefacts/`

## Assumptions

1. The `## Approved By` section name must match exactly what H-GOV in p11.3 will search for (case-sensitive, no trailing punctuation). The format defined here constrains p11.3's implementation — this choice is intentional.
2. The contract marker `<!-- approved-by-required -->` may appear as an HTML comment inside the SKILL.md completion step prose; it does not need to be on its own line as long as the string is present verbatim.
3. No changes to `check-pipeline-artefact-paths.js` are required — the template file is not an output path.
