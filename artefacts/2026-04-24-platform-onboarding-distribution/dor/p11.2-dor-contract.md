# DoR Contract: Add attribution fields to `/benefit-metric` template and SKILL.md

**Story:** `artefacts/2026-04-24-platform-onboarding-distribution/stories/p11.2-benefit-metric-attribution-fields.md`
**Date approved:** 2026-04-28

---

## What will be built

`Metric owner` and `Reviewers` fields will be added to the document header section of `.github/templates/benefit-metric.md`, each with an HTML comment explaining expected content (named person with role; non-engineering reviewer expected for Tier 1 product metrics). The `/benefit-metric` SKILL.md completion step will be updated to prompt the operator to confirm both fields are populated before marking the artefact active. A required contract marker string (`<!-- metric-owner-required -->`) will be added to the SKILL.md and registered in `check-skill-contracts.js`.

## What will NOT be built

- H-GOV hard block — this is p11.3 and must not be included
- Changes to `/discovery` template — this is p11.1
- Enforcement of reviewer participation (structural presence only, not verification)
- Backfilling attribution fields in the existing `artefacts/2026-04-24-platform-onboarding-distribution/benefit-metric.md`

## Assumptions

1. The `Metric owner` and `Reviewers` fields are added to the template header section (alongside existing fields like `Metric ID`, `Status`, etc.) rather than as `##` section headers — consistent with the existing template structure.
2. The contract marker `<!-- metric-owner-required -->` may appear as an HTML comment inline in the SKILL.md prose; it does not need its own line.
3. This story is parallel to p11.1 — these two stories may be delivered in either order.
