## Contract Proposal

**What will be built:**
An `instrumentation:` YAML block added to `contexts/personal.yml` — commented out by default, with four fields documented inline: `enabled` (boolean, default false), `experiment_id` (string), `model_label` (string), `cost_tier` (string, one of standard | premium). Each field has a type annotation and a short explanatory comment. The block includes a usage comment explaining how to activate it. The comment also references `workspace/experiments/[experiment-id]/` as the output directory (cross-referencing spc.4).

**What will NOT be built:**
- No validation logic for model_label or cost_tier allowed values
- No modifications to any SKILL.md file
- No reading of the instrumentation block in any skill (owned by spc.3)
- No experiment workspace directories (owned by spc.4)

**How each AC will be verified:**

| AC | Test approach | Type |
|----|---------------|------|
| AC1 — instrumentation block with 4 fields present | Read contexts/personal.yml, assert instrumentation: block exists with all 4 field names | Unit (file inspection) |
| AC2 — block parseable when enabled | Read contexts/personal.yml, uncomment and parse YAML, assert all 4 keys accessible | Unit (YAML parse) |
| AC3 — missing experiment_id detectable | Read contexts/personal.yml, assert experiment_id is declared but can be null/absent in a config without the field | Unit (schema inspection) |
| AC4 — no blocks when disabled (manual gap) | Manual session scenario — deferred to spc.3 test plan (RISK-ACCEPT 1-M1) | Manual |
| AC5 — workflow documented in comment | Read file, assert presence of comment text explaining the enable workflow | Unit (comment text inspection) |

**Assumptions:**
- `contexts/personal.yml` is the correct file to modify (not `.github/context.yml` directly, which is generated by copying personal.yml)
- YAML comments are the appropriate documentation vehicle for schema field descriptions in this file

**Estimated touch points:**
Files: `contexts/personal.yml` (one file, additive YAML block). Services: none. APIs: none.
