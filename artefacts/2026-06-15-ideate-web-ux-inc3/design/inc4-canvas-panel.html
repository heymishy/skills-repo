inc4 — implementation contract
  • Replaces: #draft-content (flex:1 below conditions + assumptions panels). Section header label changes from "Artefact draft" → "Lens canvas".
  • Data source: Lens blocks build from draftChunk SSE events — same stream already parsed for artefact content. Each lens section (A1–E4) becomes a canvas block when its text arrives.
  • Block types: Detected by first heading in draftChunk text (e.g. ## Opportunity mapping → type A). Default block = plain text card with lens ID inferred from heading.
  • Progress track: Five pips A–E; pip turns solid when the corresponding lens heading has arrived, active (faded) while that section is streaming.
  • Collapse: All cards collapsed by default except the most-recently-completed one. Click to expand. State preserved in session.
  • Synthesis (S): Auto-generated when all selected lenses complete. Produced by a final prompt turn that distils key signals, risks, and strategic bets into 3–5 rows. Always shown expanded, pinned at bottom of canvas.
  • Assumption quadrant: Pulls from existing session.assumptionCards — no new data. Risk axis from risk field (high/medium/low); knowness axis from knowness field (unknown-unknown → known-unknown → known-known).
  • Export: Canvas state serialised as structured markdown at commit-preview (same path as today). Canvas blocks become named H2 sections.
  • Fonts used: Bebas Neue (lens IDs), Lora (body), Overpass Mono (labels/meta) — load via Google Fonts in <head>.
