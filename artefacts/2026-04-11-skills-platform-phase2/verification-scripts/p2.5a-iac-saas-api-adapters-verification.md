# AC Verification Script: IaC and SaaS-API surface adapters (p2.5a)

**Story reference:** artefacts/2026-04-11-skills-platform-phase2/stories/p2.5a-iac-saas-api-adapters.md
**Date:** 2026-04-11

---

## Overview

The story adds two new surface adapters — `iac` and `saas-api` — to the surface adapter model. Both adapters expose the `execute(surface, context) → result` interface, read a discipline-specific POLICY.md floor, and produce surface-appropriate findings vocabulary. Check-surface-adapter.js is extended to cover both new paths.

---

## Scenarios

### Scenario 1 — IaC adapter returns a valid result object with trace fields

**AC:** AC1

**Steps:**
1. Load the IaC adapter from `src/surface-adapter/adapters/` (or resolved path).
2. Call `execute('iac', { /* minimal context */ })`.
3. Inspect the returned object.

**Expected result:** The returned object has a `status` field, a `findings` array, and a `trace` object. `trace.adapterName` is non-null. `trace.adapterVersion` is non-empty. No exception is thrown.

**Pass/Fail:** Pass if all fields present and no exception.

---

### Scenario 2 — SaaS-API adapter returns a valid result object with trace fields

**AC:** AC2

**Steps:**
1. Load the SaaS-API adapter.
2. Call `execute('saas-api', { /* minimal context */ })`.
3. Inspect the returned object.

**Expected result:** Same shape as Scenario 1. `trace.adapterName` identifies the SaaS-API adapter.

**Pass/Fail:** Pass if all fields present and non-null.

---

### Scenario 3 — IaC finding vocabulary uses IaC-specific terms

**AC:** AC3, AC6

**Steps:**
1. Provide a context that triggers at least one finding for the IaC adapter (e.g. missing state backend config).
2. Call `execute('iac', context)`.
3. Inspect `result.findings[0]` or the first finding object.

**Expected result:** The finding `key` or `label` field references an IaC-specific term sourced from `standards/iac/POLICY.md` (e.g. `iac-state-backend` or similar). No software-engineering generic term appears. `result.trace.policySource` confirms `standards/iac/POLICY.md`.

**Pass/Fail:** Pass if finding vocabulary matches IaC domain and trace confirms POLICY.md source.

---

### Scenario 4 — SaaS-API finding vocabulary uses API-appropriate terms

**AC:** AC4, AC6

**Steps:**
1. Provide a context that triggers at least one finding for the SaaS-API adapter (e.g. missing API versioning).
2. Call `execute('saas-api', context)`.
3. Inspect `result.findings[0]`.

**Expected result:** The finding references a SaaS-API-specific term from `standards/saas-api/POLICY.md`. No IaC-specific term appears. `result.trace.policySource` confirms `standards/saas-api/POLICY.md`.

**Pass/Fail:** Pass if vocabulary matches API domain.

---

### Scenario 5 — Both adapters loaded: no shadowing

**AC:** AC5

**Steps:**
1. Load both the IaC and SaaS-API adapters from the adapter registry.
2. Call `getAdapter('iac')` — verify it returns the IaC adapter.
3. Call `getAdapter('saas-api')` — verify it returns the SaaS-API adapter.
4. Call `execute` on each; compare `trace.adapterName`.

**Expected result:** Each adapter returns its own `trace.adapterName` value. Neither adapter's results bleed into the other.

**Pass/Fail:** Pass if both adapters are independently reachable and produce distinct trace identifiers.

---

### Scenario 6 — npm test passes: check-surface-adapter.js covers both adapters

**AC:** AC6

**Steps:**
1. Run `npm test` with the implementation in place.
2. Review the `[check-surface-adapter]` output.

**Expected result:** Both IaC and SaaS-API adapter test cases pass. No other test suite entries fail. Exit code = 0.

**Pass/Fail:** Pass if all `[check-surface-adapter]` cases green and overall suite exits 0.

---

### Scenario 7 — Error path: missing POLICY.md returns error result (no exception)

**AC:** AC1, AC2 (error branch)

**Steps:**
1. Remove or rename `standards/iac/POLICY.md` in a test environment.
2. Call `execute('iac', minimalContext)`.
3. Check the return value and the process state.

**Expected result:** Returns `{ status: "error", error: "<message referencing POLICY.md path>" }`. No uncaught exception. No process crash.

**Pass/Fail:** Pass if result.status = "error" and process continues.
