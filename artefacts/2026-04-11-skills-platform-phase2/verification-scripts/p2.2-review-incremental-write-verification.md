## Setup

**Before you start:**
1. Open the repository in VS Code with the Copilot agent available.
2. Prepare a feature with at least 3 stories in `artefacts/[feature]/stories/`.
3. Ensure `artefacts/[feature]/review/` directory exists but contains no review artefacts yet.

**Reset between scenarios:** Delete any review artefacts in `artefacts/[feature]/review/` before starting a new scenario unless the scenario explicitly builds on prior state.

### Scenario 1: Each story gets its review file written immediately after being reviewed

**Covers:** AC1

**Steps:**
1. Start a /review session on a feature with 3 stories.
2. Allow the skill to review the first story completely and produce its findings.
3. Before the skill moves to the second story, check the file system: open `artefacts/[feature]/review/` in VS Code Explorer.

**Expected outcome:**
> A file named `[story-slug]-review-1.md` already exists in the review folder. It is not empty — it contains the story slug, a review date, and either findings or an explicit "no findings" notation.

### Scenario 2: Resuming a review session starts from the first un-reviewed story, not from the beginning

**Covers:** AC2

**Steps:**
1. Run a review session and review exactly 2 of the 3 stories. End the session (close or abandon it).
2. Start a new /review session on the same feature.

**Expected outcome:**
> The skill detects the two existing review artefacts and announces that it is resuming from the third story. It does not re-review stories 1 and 2. The existing artefact files for stories 1 and 2 are unchanged.

### Scenario 3: A review artefact for a PASS story still contains all required fields

**Covers:** AC3

**Expected outcome:**
> The file contains all of: the story slug in the header, the review date, a findings list (may say "No findings" or equivalent), severity classification for each finding (or "None"), and recommended actions (or "None — story passes").
