## Unit Tests

### GitHub vcs.type: output path is .github/copilot-instructions.md

- **Verifies:** AC1
- **Precondition:** Context fixture with `vcs.type: github`
- **Action:** Run assembly script (or assembly function) with the fixture; check which output path is returned
- **Expected result:** Output path is `.github/copilot-instructions.md`; no `AGENTS.md` file is created
- **Edge case:** No

### GitHub vcs.type: AGENTS.md is NOT created

- **Verifies:** AC1
- **Precondition:** Context fixture with `vcs.type: github`; clean working directory
- **Action:** Run assembly script; check file system for `AGENTS.md` at repo root
- **Expected result:** `AGENTS.md` absent
- **Edge case:** No

### Non-GitHub vcs.type (bitbucket): output path is AGENTS.md at repo root

- **Verifies:** AC2
- **Precondition:** Context fixture with `vcs.type: bitbucket`
- **Action:** Run assembly script with fixture
- **Expected result:** Output path is `AGENTS.md` at repo root; `.github/copilot-instructions.md` is NOT created
- **Edge case:** No

### Non-GitHub vcs.type: .github/copilot-instructions.md is NOT created

- **Verifies:** AC2
- **Precondition:** Context fixture with `vcs.type: gitlab`; clean working directory
- **Action:** Run assembly script; check for `.github/copilot-instructions.md`
- **Expected result:** `.github/copilot-instructions.md` absent
- **Edge case:** No

### Explicit vcs.agent_instructions.format: agents-md overrides vcs.type: github

- **Verifies:** AC3
- **Precondition:** Context fixture with `vcs.type: github` AND `vcs.agent_instructions.format: agents-md`
- **Action:** Run assembly script
- **Expected result:** Output path is `AGENTS.md` regardless of `vcs.type: github`
- **Edge case:** No

### Invalid vcs.agent_instructions.format value — assembly-time error

- **Verifies:** AC3 (error case)
- **Precondition:** Context fixture with `vcs.agent_instructions.format: unsupported-value`
- **Action:** Run assembly script
- **Expected result:** Script exits with non-zero code and error message naming `unsupported-value` as an unsupported format; no output file written
- **Edge case:** Yes — invalid enum value

### Content parity: assembled content is identical for github and bitbucket contexts

- **Verifies:** AC4
- **Precondition:** Two context.yml fixtures (github and bitbucket) pointing to the same source files
- **Action:** Run assembly script twice; compare output content byte-for-byte
- **Expected result:** Content hash of `.github/copilot-instructions.md` (from run 1) equals content hash of `AGENTS.md` (from run 2)
- **Edge case:** No

### Unconfigured context: defaults to .github/copilot-instructions.md

- **Verifies:** AC5
- **Precondition:** Context fixture with no `vcs.type` and no `vcs.agent_instructions.format` keys
- **Action:** Run assembly script
- **Expected result:** Output path is `.github/copilot-instructions.md`; no error; no `AGENTS.md` created
- **Edge case:** No

## NFR Tests

### Performance: vcs.type branch logic adds at most 100ms

- **NFR addressed:** Performance
- **Measurement method:** Time the assembly script run with `vcs.type: github` context before and after the branch logic is added (or time the branch check in isolation)
- **Pass threshold:** Additional time ≤ 100ms on a standard developer machine
- **Tool:** Node.js `performance.now()` or `Date.now()` in the test

### Security: assembly script does not log credential sections from context.yml (MC-SEC-02)

- **NFR addressed:** Security
- **Measurement method:** Create a context.yml fixture with a secrets section containing a placeholder token value; run assembly script; check stdout/stderr for the placeholder value
- **Pass threshold:** No credential value appears in stdout or stderr output
- **Tool:** Node.js test script + capture stdout

### Auditability: output path logged to stdout during assembly

- **NFR addressed:** Auditability
- **Measurement method:** Run assembly script with both github and bitbucket fixtures; capture stdout; check for the output path string in each
- **Pass threshold:** stdout contains the output file path for each run (`.github/copilot-instructions.md` or `AGENTS.md`)
- **Tool:** Node.js child_process capture
