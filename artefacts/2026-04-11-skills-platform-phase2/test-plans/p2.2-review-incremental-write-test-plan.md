## Unit Tests

### Per-story write: artefact file appears at correct path after story review completes

- **Verifies:** AC1
- **Precondition:** Feature at `artefacts/2026-04-11-skills-platform-phase2/`; story slug `p2.1-definition-skill-improvements` reviewed
- **Action:** Run the /review per-story write for this story; check file system for `artefacts/2026-04-11-skills-platform-phase2/review/p2.1-definition-skill-improvements-review-1.md`
- **Expected result:** File exists immediately after the write; its content is non-empty
- **Edge case:** No

### Written artefact contains all required fields

- **Verifies:** AC3
- **Precondition:** Review artefact written for any story (use AC1 fixture)
- **Action:** Read the artefact file and verify presence of: story slug in header, review date, findings list (may be empty), severity level for each finding, recommended action for each finding
- **Expected result:** All five required fields present; file does not have ANY incomplete JSON/YAML structure
- **Edge case:** No

### Empty findings list is a valid complete write

- **Verifies:** AC3 (edge: files with no findings)
- **Precondition:** Story reviewed with PASS (0 findings)
- **Action:** Check that artefact file is written and has an explicitly empty findings list (not a missing field)
- **Expected result:** File present; findings list is `[]` or equivalent empty marker; not absent or null
- **Edge case:** Yes — pass result

### Session recovery: existing artefacts for S1 and S2 are detected, S3 is the resume point

- **Verifies:** AC2
- **Precondition:** `artefacts/[feature]/review/` contains `story-s1-review-1.md` and `story-s2-review-1.md`; story S3 artefact does not exist
- **Action:** Run /review session recovery detection; identify the first story without an existing review artefact
- **Expected result:** Stories S1 and S2 are detected as already reviewed; S3 is identified as the resume point; detection does not require operator input to determine
- **Edge case:** No

### Session recovery: existing artefacts are not overwritten unless operator explicitly requests re-review

- **Verifies:** AC2
- **Precondition:** `story-s1-review-1.md` exists with known content (a specific finding text)
- **Action:** Trigger resume; verify S1 artefact content unchanged
- **Expected result:** S1 artefact has identical content to before the resume; no new write to S1 file
- **Edge case:** No

- **NFR addressed:** Durability
