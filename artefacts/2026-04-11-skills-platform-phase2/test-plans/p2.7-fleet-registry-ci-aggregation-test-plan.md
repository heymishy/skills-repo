## Unit Tests

### Squad file validation: valid file with all required fields passes

- **Verifies:** AC1
- **Precondition:** `fleet/squads/squad-a.json` fixture with all required fields: `squadId`, `repoUrl`, `pipelineStateUrl`, `registeredAt`
- **Action:** Run squad file validation against the fixture
- **Expected result:** Validation passes; no error emitted; file is accepted
- **Edge case:** No

### Squad file validation: file missing required field is rejected with descriptive error naming the field

- **Verifies:** AC1
- **Precondition:** Squad file fixture with `squadId` field removed
- **Action:** Run squad file validation
- **Expected result:** Validation fails; error message names `"squadId"` as the missing field; file rejected
- **Edge case:** Yes

### Squad file validation: file missing multiple required fields — error lists all missing fields

- **Verifies:** AC1 (multi-field error quality)
- **Precondition:** Squad file fixture with both `pipelineStateUrl` and `registeredAt` removed
- **Action:** Run squad file validation
- **Expected result:** Error message lists both missing fields; not just the first one found
- **Edge case:** Yes

### CI aggregation: reads all squad files from fleet/squads/ and writes fleet-state.json

- **Verifies:** AC2
- **Precondition:** Two valid squad fixture files; stub pipelineStateUrl responses for each
- **Action:** Run CI aggregation job (or aggregation function) with fixtures
- **Expected result:** `fleet-state.json` written; `squads` array has exactly 2 entries (one per fixture); no extra entries; file is valid JSON
- **Edge case:** No

### CI aggregation: each entry includes data fetched from pipelineStateUrl

- **Verifies:** AC2 (fetch + write)
- **Precondition:** One squad file where stub `pipeline-state.json` returns `{ "stage": "test-plan", "health": "green" }`
- **Action:** Run aggregation; inspect entry for this squad
- **Expected result:** Entry has `stage: "test-plan"` and `health: "green"` — sourced from the fetched `pipeline-state.json`, not from the squad file itself
- **Edge case:** No

### CI aggregation: updatedAt in each entry reflects aggregation run time

- **Verifies:** AC2, AC3 (updatedAt field)
- **Precondition:** Two squad fixtures; run aggregation
- **Action:** Inspect `result.squads[0].updatedAt` and `result.squads[1].updatedAt`
- **Expected result:** Both entries have `updatedAt` set to a valid ISO 8601 datetime; the value reflects the aggregation run time (within a reasonable tolerance), not the squad file `registeredAt`
- **Edge case:** No

### fleet-state.json structure: each entry has all five required fields

- **Verifies:** AC3
- **Precondition:** fleet-state.json produced from two-squad fixture
- **Action:** Inspect each entry in `squads[]`
- **Expected result:** Entry contains: `squadId` (string), `stage` (string), `health` (one of green/amber/red/unknown), `updatedAt` (ISO datetime), `sourceUrl` (string matching `pipelineStateUrl` from squad file)
- **Edge case:** No

### fleet-state.json structure: sourceUrl matches pipelineStateUrl from squad file

- **Verifies:** AC3 (sourceUrl traceability)
- **Precondition:** Squad fixture with `pipelineStateUrl: "https://example/pipeline-state.json"`
- **Action:** Inspect `entry.sourceUrl`
- **Expected result:** `entry.sourceUrl === "https://example/pipeline-state.json"` — verbatim match
- **Edge case:** No

### registry_mode: publishing — squad included in fleet-state.json

- **Verifies:** AC4
- **Precondition:** Squad fixture where context.yml for that squad has `registry_mode: publishing`
- **Action:** Run aggregation; check fleet-state.json for the squad's entry
- **Expected result:** Squad entry present in `squads[]`
- **Edge case:** No

### registry_mode: none — squad excluded from fleet-state.json

- **Verifies:** AC4 (exclusion)
- **Precondition:** Squad fixture where `registry_mode: none`
- **Action:** Run aggregation; check fleet-state.json
- **Expected result:** No entry for this squad in `squads[]`; aggregation job completes without error
- **Edge case:** No

### Graceful degradation: unreachable pipelineStateUrl — squad entry has health:"unknown" and error field

- **Verifies:** AC6
- **Precondition:** One squad file with `pipelineStateUrl` pointing to an HTTP 404 stub; one valid squad file
- **Action:** Run aggregation; inspect fleet-state.json
- **Expected result:** Unreachable squad entry has `health: "unknown"` and an `error` field with a descriptive failure message; valid squad entry is unaffected
- **Edge case:** Yes

### Graceful degradation: aggregation job does not abort when one squad is unreachable

- **Verifies:** AC6 (job continues)
- **Precondition:** Three squad files: one unreachable, two valid
- **Action:** Run aggregation; capture exit code and fleet-state.json
- **Expected result:** Exit code 0 (job succeeds overall); fleet-state.json has 3 entries — 1 with health:"unknown" and 2 with live health values; no aborted run
- **Edge case:** No

## NFR Tests

### Performance: CI aggregation job completes in ≤60 seconds for 20 squad fixtures

- **NFR addressed:** Performance
- **Measurement method:** Create 20 synthetic squad fixture files with stub pipelineStateUrl responses (all resolvable); time the aggregation job from start to fleet-state.json write
- **Pass threshold:** ≤ 60 seconds
- **Tool:** Node.js `Date.now()` wrapper or CI job timing

### Security: fleet-state.json and squad files contain no credential, token, or personal data fields (MC-SEC-02)

- **NFR addressed:** Security
- **Measurement method:** Inspect the schema for fleet/squads/{id}.json and fleet-state.json; verify no field is defined for credentials, access tokens, or personal identifiers; run a fixture scanner checking output for common credential patterns
- **Pass threshold:** No credential-like fields in schema or output; no values matching common token patterns
- **Tool:** Schema inspection + fixture scan

### Accessibility: interactive fleet viz elements are keyboard-reachable (MC-A11Y-01)

- **NFR addressed:** Accessibility
- **Measurement method:** In the fleet viz panel HTML, verify that all interactive elements (links, expand/collapse if any) have either: (a) a `tabindex` attribute ≥ 0, or (b) are natively focusable elements (`<a>`, `<button>`); no interactive element uses `pointer-events: none` without a keyboard alternative
- **Pass threshold:** All interactive elements reachable via keyboard; no pointer-only interaction pattern
- **Tool:** Automated HTML attribute check in check-viz-behaviour.js
