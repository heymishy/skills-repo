## Unit Tests

### Path A: resolver constructs HTTP GET request to registry_url with app_id as query parameter

- **Verifies:** AC1
- **Precondition:** Resolver module loaded; HTTP client stubbed to capture outgoing requests; context.yml fixture with `registry_url: "https://stub.ea-registry/apps"` and `app_id: "app-001"`
- **Action:** Call `resolver.resolve(context)` — inspect the captured HTTP request
- **Expected result:** One HTTP GET request made; URL contains `app_id=app-001` (as query parameter); no other HTTP calls made
- **Edge case:** No

### Path A: resolver reads registry_url and app_id from context — not hardcoded

- **Verifies:** AC1 (ADR-004 compliance)
- **Precondition:** Two context fixtures with different `registry_url` and `app_id` values; HTTP stub captures requests
- **Action:** Call `resolver.resolve()` for each context; compare captured request URLs
- **Expected result:** Request URL matches the `registry_url` + `app_id` from each context respectively — no shared hardcoded URL or ID used
- **Edge case:** No

### Path A: technology.hosting "saas" maps to saas-api adapter (confirmed EA registry field)

- **Verifies:** AC2
- **Precondition:** YAML stub returns an EA-registry-format application entry: `technology: { hosting: "saas" }, owner: "squad-a"` (RESOLUTION-ASSUMPTION-02 confirmed fields); context.yml has no `adapter_override`
- **Action:** Call `resolver.resolve(context)` with Path A config; capture resolved adapter name
- **Expected result:** Adapter `"saas-api"` is selected via mapping (`saas` → `saas-api`); outcome identical to a Path B context.yml with `surface: saas-api`
- **Edge case:** No

### Path A: technology.hosting "cloud" maps to iac adapter

- **Verifies:** AC2 (mapping table — cloud branch)
- **Precondition:** YAML stub returns `technology: { hosting: "cloud" }, owner: "squad-b"`; no `adapter_override` in context.yml
- **Action:** Call `resolver.resolve(context)`; inspect resolved adapter
- **Expected result:** Adapter `"iac"` is selected
- **Edge case:** No

### Path A: context.yml adapter_override takes precedence over registry-derived type

- **Verifies:** AC2 (override precedence; replaces registry adapterOverride — see RESOLUTION-ASSUMPTION-02)
- **Precondition:** YAML stub returns `technology: { hosting: "saas" }, owner: "squad-c"`; context.yml contains `adapter_override: "iac"`
- **Action:** Call `resolver.resolve(context)`; inspect resolved adapter
- **Expected result:** Adapter `"iac"` is selected (context.yml `adapter_override`) — NOT `"saas-api"` (registry-derived type)
- **Edge case:** No

### Path A unavailable (timeout): resolver falls back to Path B when surface: declared

- **Verifies:** AC3 (fallback branch)
- **Precondition:** HTTP stub simulates a timeout (3s+); context includes both `registry_source: ea-registry` AND `surface: iac`
- **Action:** Call `resolver.resolve(context)`
- **Expected result:** Resolver selects `"iac"` adapter (from Path B fallback); run output log contains substring "EA registry unavailable — falling back to explicit surface declaration"; no exception thrown
- **Edge case:** No

### Path A unavailable (HTTP 503): resolver returns error result when no Path B fallback exists

- **Verifies:** AC3 (error branch)
- **Precondition:** HTTP stub returns HTTP 503; context has NO `surface:` declaration
- **Action:** Call `resolver.resolve(context)`
- **Expected result:** Returns `{ status: "error", error: "EA registry unavailable and no fallback surface declaration found" }`; resolver does NOT default to git-native adapter
- **Edge case:** Yes

### Path A unavailable (DNS failure): resolver returns explicit error — no silent git-native default

- **Verifies:** AC3 (resilience — silent fallback guard)
- **Precondition:** HTTP stub simulates DNS failure; no Path B fallback
- **Action:** Call `resolver.resolve(context)`; inspect result and log output for any git-native selection
- **Expected result:** Error result returned; `result.error` contains the unavailability message; no `"git-native"` or similar surface type appears in the result
- **Edge case:** Yes

### Path B: resolver uses explicit surface: declaration and makes no HTTP call

- **Verifies:** AC4
- **Precondition:** Context fixture with `surface: saas-gui` and no `registry_source` or `registry_url` fields; HTTP stub records all outgoing requests
- **Action:** Call `resolver.resolve(context)`
- **Expected result:** Adapter `"saas-gui"` selected; zero HTTP requests captured — no EA registry call made; no error; behaviour identical to pre-p2.6
- **Edge case:** No

### Path B: resolver does not require app_id when registry_source is absent

- **Verifies:** AC4 (backward-compat)
- **Precondition:** Path B context with no `app_id` field
- **Action:** Call `resolver.resolve(context)`
- **Expected result:** Resolver resolves successfully; no error about missing `app_id`; surface type returned correctly
- **Edge case:** No

### Both paths declared: Path B takes precedence, warning logged

- **Verifies:** AC5
- **Precondition:** Context fixture contains both `registry_source: ea-registry` AND `surface: saas-gui`; HTTP stub records requests
- **Action:** Call `resolver.resolve(context)`; capture result and log output
- **Expected result:** Adapter `"saas-gui"` selected (from Path B); zero HTTP requests (Path A not queried); log output contains "Both Path A (registry_source) and Path B (surface:) declared — Path B (explicit) takes precedence"
- **Edge case:** No

### Both paths declared: warning message includes instruction to remove one

- **Verifies:** AC5 (warning completeness)
- **Precondition:** As above
- **Action:** Inspect the log output string for the warning
- **Expected result:** Warning includes "Remove one to resolve." (or equivalent phrasing matching the AC)
- **Edge case:** No

## NFR Tests

### Performance: EA registry HTTP call timeout ≤3 seconds

- **NFR addressed:** Performance (gate budget — P1.3 NFR)
- **Measurement method:** Set HTTP stub to delay response by exactly 3s; assert resolver returns an error result at or before 3100ms (allowing 100ms margin); stub delay at 2900ms produces a successful response
- **Pass threshold:** Timeout fires at ≤3000ms; no indefinite hang
- **Tool:** HTTP stub with configurable delay + timer assertion

### Security: resolver does not log full EA registry response body (MC-SEC-02)

- **NFR addressed:** Security
- **Measurement method:** YAML stub returns an EA-registry-format entry with an additional unexpected field: `technology: { hosting: "cloud" }, owner: "t1", notes: "secretApiKey: FAKE_SECRET_12345"` (simulates a credential accidentally placed in the notes field); run resolver with stdout capture; check stdout/stderr for `"FAKE_SECRET_12345"`
- **Pass threshold:** The placeholder secret value does NOT appear in stdout or stderr; only the resolved surface type string (e.g. `"iac"`) is logged
- **Tool:** Node.js stdout capture + string search

### Security: resolver uses HTTPS only — plain HTTP registry_url rejected (MC-SEC-03)

- **NFR addressed:** Security
- **Measurement method:** Set `registry_url` to `http://` (non-HTTPS) in context fixture; call resolver
- **Pass threshold:** Resolver returns an error result with message referencing HTTPS requirement; no HTTP call made
- **Tool:** HTTP stub + assertion on result
