## Unit Tests

### D1: reports missing-upstream-slug for unresolvable dependency

- **Verifies:** AC1
- **Precondition:** Story artefact text contains `Dependencies: upstream: p2.x-nonexistent-story`; no file exists at `artefacts/[feature]/stories/p2.x-nonexistent-story.md`
- **Action:** Parse the story Dependencies block and check each slug against the expected artefact path
- **Expected result:** The slug `p2.x-nonexistent-story` is returned in a warnings list indicating it does not resolve; the returned warning message contains the slug name and the three recovery options
- **Edge case:** No — happy-path missing slug

### D1: resolves slug correctly when upstream story file exists

- **Verifies:** AC1 (negative — no false warning)
- **Precondition:** Story Dependencies block lists `p2.1-definition-skill-improvements`; this file exists at the expected path
- **Action:** Parse and check slug resolution
- **Expected result:** No warning is emitted; slug is resolved cleanly
- **Edge case:** No

### D1: external dependency annotation is preserved in artefact text

- **Verifies:** AC2
- **Precondition:** Story Dependencies block has been updated with `[External: <description> — confirmed by operator on <date>]` for a non-resolvable slug
- **Action:** Read the story Dependencies block and check for the `[External:` marker
- **Expected result:** The annotation is present verbatim; no subsequent warning for that slug is emitted for the same session
- **Edge case:** No

### D1: external annotation format — must contain description and date fields

- **Verifies:** AC2
- **Precondition:** Story Dependencies text with an External annotation missing the date component
- **Action:** Validate the annotation format using a regex or structured parse
- **Expected result:** Validation report flags the annotation as malformed (missing date portion); well-formed annotation passes without warning
- **Edge case:** Yes — malformed annotation

### D2: testability filter flags "should" in AC text

- **Verifies:** AC3 (anti-pattern a)
- **Precondition:** AC text: `Given X, when Y, then the system should display Z`
- **Action:** Run the D2 testability heuristic against the AC text
- **Expected result:** The heuristic returns a warning entry with reason containing "should" flagged as asserting hypothetical rather than current-state observable behaviour
- **Edge case:** No

### D2: testability filter flags internal state description

- **Verifies:** AC3 (anti-pattern b)
- **Precondition:** AC text: `Given X, when Y, then pipeline-state.json has field health set to green in the internal store`
- **Action:** Run the D2 testability heuristic
- **Expected result:** Warning returned with reason containing "internal system state not visible to test runner or reviewer"
- **Edge case:** No

### D2: testability filter flags AC coupling reference

- **Verifies:** AC3 (anti-pattern c)
- **Precondition:** AC text: `Given the previous AC has passed and set up state, when Y, then Z`
- **Action:** Run the D2 testability heuristic
- **Expected result:** Warning returned with reason containing "cannot be evaluated independently without first running a prior AC"
- **Edge case:** No

### D2: accepted AC receives testability annotation

- **Verifies:** AC4
- **Precondition:** AC text flagged with a testability warning; operator accepts with `[Testability: accepted by operator on 2026-04-11]`
- **Action:** Check that the annotation text appears in the written artefact and matches the required format
- **Expected result:** Annotation present and parseable; no re-warning for the same AC in the same session
- **Edge case:** No

### D2: advisory filter — story writing continues without AC removal

- **Verifies:** AC4
- **Precondition:** AC flagged as not independently testable
- **Action:** Verify that the story writing procedure does not halt or remove the AC from the artefact
- **Expected result:** Story artefact contains the original AC text plus the annotation; write continues to next story
- **Edge case:** No

### D3: completion output contains learnings exit prompt

- **Verifies:** AC5
- **Precondition:** definition/SKILL.md file exists at `.github/skills/definition/SKILL.md`
- **Action:** Read the SKILL.md Completion Output section and check for the learnings prompt text
- **Expected result:** Section contains: "Before ending this session — are there any learnings from this decomposition to write to `workspace/learnings.md`?" and "Reply with the learning text, or `skip` to proceed to /estimate"
