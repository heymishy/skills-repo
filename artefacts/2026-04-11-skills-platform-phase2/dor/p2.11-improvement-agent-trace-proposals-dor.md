## Contract Proposal

### What will be built

- **Queryable trace interface** (`src/improvement-agent/trace-interface.js`): Exposes `queryTraces(filters)` function — callable from CLI (thin wrapper) and from improvement-agent SKILL.md procedure. Filters: `storySlug`, `patternLabel` (kebab-case, case-insensitive exact match), `dateRange` (`from`/`to` ISO strings). Returns only traces matching all specified filters from `workspace/traces/`. This is the canonical trace data access point — no other module reads trace files directly.
- **Failure signal detection and diff proposal generation** (`AC2`): Sliding window of 10 most recent traces per surface type by `createdAt`. If ≥3 traces carry the same `failurePattern` value (exact string, case-insensitive), a diff proposal is written to `workspace/proposals/YYYY-MM-DD-[skill-slug]-failure-proposal.md` with: evidence (trace IDs), proposed_diff (before/after SKILL.md section), confidence, anti_overfitting_gate result, status: pending_review, created_at. Evaluation scenario: `s-improvement-agent-failure-clustering`.
- **Staleness signal detection and staleness proposal generation** (`AC3`): If a surface type has no trace activity for ≥90 days (or `context.yml` `improvement_agent.staleness_window_days`), a staleness proposal is written to `workspace/proposals/YYYY-MM-DD-[skill-slug]-staleness-proposal.md`.
- **Anti-overfitting gate** (`AC4`): If a proposed removal/weakening affects a check that has passed on every trace in the current filter window for that surface type, the proposal is blocked. A warning file is written instead. No weakening proposals written without this gate passing.
- **workspace/state.json proposals block update** (`AC6`): After any session with ≥1 proposal written, `state.json` is updated with a `proposals` block listing each proposal file path, `created_at`, and `status`.

### What will NOT be built

- Automated application of accepted proposals — proposal generation only; review and acceptance are in p2.12
- Cross-repository trace ingestion — local `workspace/traces/` only
- Automatic re-triggering on each new trace write — on-demand or scheduled CI
- ML-based clustering — lexical pattern matching on `failurePattern` field only
