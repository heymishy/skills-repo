## Contract Proposal

### What will be built

- **D1 — Dependency chain validation** in `.github/skills/definition/SKILL.md`: When a story is saved with at least one upstream dependency declared, the skill checks whether each named upstream story slug resolves to `artefacts/[feature]/stories/[slug].md`. If any slug does not resolve, a 3-option prompt is surfaced before proceeding to the next story. For operator-confirmed external dependencies, the annotation `[External: <description> — confirmed by operator on <date>]` is recorded in the Dependencies block.
- **D2 — Testability filter** in `.github/skills/definition/SKILL.md`: After each AC is written, the skill applies a heuristic check for three anti-patterns: (a) "should"/"would" phrasing, (b) internal system state invisible to test runner, (c) AC coupled to a prior AC. Flagged ACs are surfaced as advisory warnings; the operator can accept as-is with annotation `[Testability: accepted by operator on <date>]`. The filter is advisory only — it does not block story creation.
- **D3 — Learnings exit step** in `.github/skills/definition/SKILL.md`: At the Completion Output (after scope accumulator check), an explicit prompt is added: "Before ending this session — are there any learnings from this decomposition to write to `workspace/learnings.md`? Reply with the learning text, or `skip` to proceed to /estimate."

### What will NOT be built

- Changes to any skill other than `.github/skills/definition/SKILL.md`
- Hard-blocking story progression when a dependency slug is missing (gate belongs at /definition-of-ready H1)
- ML-based testability scoring — D2 uses only the three named heuristic patterns
- Backfilling D1/D2/D3 checks onto Phase 1 story artefacts
