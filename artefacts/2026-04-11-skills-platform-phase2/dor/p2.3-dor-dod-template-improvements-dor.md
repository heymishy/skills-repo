## Contract Proposal

### What will be built

- **D8 — H8-ext hard block** added to `.github/skills/definition-of-ready/SKILL.md`: When a story's Dependencies block lists at least one upstream story, the DoR contract template prompts for a `schemaDepends: [field1, field2]` declaration. H8-ext then checks each declared field against `pipeline-state.schema.json` — absent fields fire as a hard block. Stories with `Dependencies: None` pass H8-ext without a check.
- **B1-enforce — H-NFR-profile hard block** added to `.github/skills/definition-of-ready/SKILL.md`: When a story declares at least one NFR, the DoR H-checks require `artefacts/[feature]/nfr-profile.md` to exist. Absent profile fires hard block. Stories with `NFRs: None` or blank are not blocked.
- **D9 — Verification method field** added to `.github/templates/definition-of-done.md`: Each AC entry in the DoD template's AC Coverage section gains a `Verification method: [automated test slug / code review / manual run]` prompt field — one of three enumerated values.
