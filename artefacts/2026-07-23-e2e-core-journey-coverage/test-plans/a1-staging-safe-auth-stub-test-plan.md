## Integration Tests

### Staging auth-stub credential is absent from production Fly config

- **Verifies:** AC3
- **Components involved:** `fly.toml` (production config), the stub-enabling environment variable/credential name introduced by this story's implementation
- **Precondition:** Both `fly.toml` (production) and `fly.staging.toml` exist in the repo
- **Action:** A Node script (non-Playwright, runs via the unit test chain per ADR-018) reads `fly.toml` and asserts the stub-enabling variable name is not present as a key anywhere in it
- **Expected result:** Assertion passes — the variable name string does not appear in `fly.toml`'s `[env]` block or anywhere else in the file

### ADR-018 addendum documents the staging-safe auth stub mechanism

- **Verifies:** AC4
- **Components involved:** `.github/architecture-guardrails.md`
- **Precondition:** ADR-018's section exists in the file (it does, pre-dating this story)
- **Action:** A Node script (non-Playwright) reads `.github/architecture-guardrails.md`, locates the ADR-018 section, and asserts an addendum block exists after it containing the words "staging" and "addendum" (or an equivalent explicit marker agreed at implementation time, e.g. `#### Addendum — 2026-07-23`)
- **Expected result:** Assertion passes — the addendum block is present and non-empty

## E2E Tests

<!-- This story's core behaviour is browser/session-driven against real staging — E2E (Playwright) is the primary test type, not a template deviation. -->

### Creates a real staging user via the GitHub OAuth stub and establishes a valid session

- **Verifies:** AC1
- **Precondition:** Test targets `wuce-staging`'s real base URL (not `localhost`); the staging-safe stub mechanism is deployed and its enabling credential is set as a CI secret for the test run only
- **Action:** Playwright spec authenticates via the new stub flow
- **Expected result:** A subsequent authenticated request (e.g. `GET /api/user/me` or equivalent) returns a real user record with a generated `e2e-test-` tagged identity, and the response's `Set-Cookie` header contains a valid session cookie that authenticates follow-up requests
- **Edge case:** No — covered by AC2 below

### Creates a real staging user via email/password signup, independent of the GitHub stub path

- **Verifies:** AC2
- **Precondition:** Same as above, distinct `e2e-test-` tagged email from AC1's test run
- **Action:** Playwright spec drives the email/password signup form
- **Expected result:** A real user record is created and a valid session is established — verified independently of AC1 (this test does not reuse or depend on AC1's session/user)
- **Edge case:** No

## NFR Tests

### Auth stub round-trip completes in under 5 seconds

- **NFR addressed:** Performance
- **Measurement method:** Playwright's built-in test timing, asserted within the AC1/AC2 E2E tests (not a separate test file) — the auth round-trip (stub or email) is timed from initial request to session-cookie-confirmed response
- **Pass threshold:** < 5000ms
- **Tool:** Playwright Test (`test.step` timing or `Date.now()` delta assertion)

### Stub-enabling credential is never logged in plaintext

- **NFR addressed:** Security
- **Measurement method:** Manual — grep CI job logs for the literal credential value after a real CI run; no automated test can assert "nothing is ever logged" across all possible log-emitting code paths without exhaustive log capture
- **Pass threshold:** Zero occurrences of the literal credential value in CI job output
- **Tool:** Manual (see verification script) — classified as a coverage gap below, not silently assumed

### Audit log records which test run created which staging user

- **NFR addressed:** Audit
- **Measurement method:** After AC1/AC2's E2E tests run, an Integration test queries the audit log (table or file, per implementation) for an entry matching the generated `e2e-test-` identity and asserts it is present
- **Pass threshold:** Exactly one audit entry per generated test identity
- **Tool:** Node script (non-Playwright), run as part of the unit test chain against the staging audit store
