# Decisions: admin-credits-topup rate-limit bypass gap

## FINDING — admin-credits-topup.js never sends the existing staging rate-limit bypass header (2026-07-23)

**Context:** After `catc-s1` (PR #568) and `bssm-s1` (PR #572) both merged and were independently confirmed passing standalone against real `wuce-staging`, a full-suite real CI run (PR #572, run 30041728074) still showed a3's AC3 and a4's combined AC2&AC3 test skipping. Re-running the same tests standalone against the identical live environment moments later passed immediately — ruling out a real regression and pointing at run-shape-dependent contention.

**Root cause:** `auth-email.js`'s `_checkRateLimit()` (from `serlb-s1`) already implements a narrow, triple-gated bypass for the real 10-attempts/5-minute per-IP limiter (env secret + `x-e2e-rate-limit-bypass` header + `e2e-test-`-prefixed email). `staging-auth.js`'s `signUpEmail()` correctly sends this header. `admin-credits-topup.js`'s `_adminLogin()`/`_adminSignupOnce()` — called by a3, a4, and b1 to top up the same fixed `e2e-test-admin@example.test` identity's credits before every real-turn test — never send it, even though the identity's email already satisfies the third gate. When a1/a2/a3/a4 (and separately b1) all run in the same CI job, the admin identity's un-bypassed login attempts can accumulate past the real rate-limit threshold within the job's runtime window, causing the credits top-up to fail and the dependent test to skip — non-deterministically, depending on exactly how many spec files share the job and how quickly they run.

**Decision:** Fix by wiring the SAME existing bypass header into `admin-credits-topup.js`'s two HTTP calls — do not touch `auth-email.js`'s rate-limiter logic itself (already correct and appropriately narrow). See `stories/eatrl-s1.md` and `dor/eatrl-s1-dor.md` for the full fix and DoR sign-off.

**Verification performed:** Direct code inspection of `_checkRateLimit()`'s exact three gates (`auth-email.js` lines 55-140) and `admin-credits-topup.js`'s `_adminLogin()`/`_adminSignupOnce()` (confirmed no header sent in either). Reproducing evidence: real CI run 30041728074's compact reporter dot-pattern (`···°°°··°°·`) showing a3 AC3 (position 9) and a4 AC2&AC3 (position 10) skipped, contrasted against an immediate standalone pass on the same live environment. No fix applied yet in this pass — root cause confirmed and DoR signed off; implementation deferred to a dispatched coding agent per this repo's short-track pipeline.
