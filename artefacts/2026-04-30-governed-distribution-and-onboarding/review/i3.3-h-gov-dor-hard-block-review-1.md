## MEDIUM findings — resolve or acknowledge in /decisions

- **1-M1** [C — AC quality] — AC4 remediation path is ambiguous. AC4 says H-GOV fires when the benefit-metric artefact carries the "Attribution incomplete" note from i3.2 AC5, and that "the story is blocked until the discovery `Approved By` field is corrected." However, the AC does not describe the pass condition for the following scenario: the operator corrects the discovery `Approved By` field after the benefit-metric note has been written, but the benefit-metric artefact still carries the note. Does H-GOV then pass (because discovery is now populated) or still fire (because the note is evidence attribution was incomplete at benefit-metric time)? The current AC wording is consistent with either interpretation.
  Risk if proceeding: test plan writers will either (a) write conflicting tests for this scenario, or (b) leave the scenario untested entirely — creating a gap in the acceptance gate.
  To acknowledge: run /decisions, category RISK-ACCEPT, and add a clarifying note to AC4 in the story artefact before /test-plan runs. Suggested resolution: specify that H-GOV checks the discovery `Approved By` field directly (live read), and the benefit-metric note is used as a trigger-to-check signal only — so if the discovery field is subsequently corrected, H-GOV passes on the next run.
