## FINDINGS

### 1-L1 — LOW — AC2 missing WHEN clause; Given/When/Then structure incomplete

AC2 states: "Given /orient has detected Entry B, then the routing output names the specific detection signal: e.g., 'Found: src/ directory with application files; no artefacts/ folder found. This looks like Entry B (code-first brownfield).' — so the operator can confirm or correct the classification." There is no WHEN clause — the AC jumps directly from Given to Then. The implicit When is "when /orient produces its output" (i.e. when /orient runs), which is the same as AC1's When. Without an explicit When, the AC is structurally incomplete and may confuse the test author into treating AC2 as an additional assertion on AC1's output rather than a standalone test.

**Recommended action:** Add a WHEN clause — e.g. "Given /orient has detected Entry B, **when /orient produces its routing output**, then the output names the specific detection signal...". This makes AC2 independently exercisable as a test case that verifies the detection transparency requirement separately from AC1's full output check.
