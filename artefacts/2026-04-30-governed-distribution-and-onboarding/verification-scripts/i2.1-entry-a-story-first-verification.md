## Scenarios

### Scenario 1 — Repo with story artefacts, no discovery.md: routes to Entry A (AC1)

**How to verify (SKILL.md inspection):**
Read the Entry A routing block in `.github/skills/orient/SKILL.md`.

**Expected outcome (check all):**
- `[ ]` The routing block is labelled "Entry A — story-first brownfield" (or equivalent canonical phrase)
- `[ ]` The block explains that existing story artefacts can be adopted — the operator does NOT need to discard and re-derive them from scratch
- `[ ]` `/tdd` is explicitly named as the inner loop entry point for continuing against existing stories
- `[ ]` The block states that existing story artefacts can be attached to /tdd "without re-deriving" or equivalent wording
- `[ ]` `/discovery` is named as the outer loop anchor, with a note to use existing stories as input scope

**Pass:** All five checkboxes satisfied.
**Fail:** Any checkbox unchecked.

**How to verify (SKILL.md inspection or live test):**

*Inspection path:*
Read the Entry A detection condition. Confirm it requires absence of `discovery.md`.

*Live test path (if orient SKILL.md is functional):*
```
# In a test repo that has both artefacts/[feature]/stories/ AND artefacts/[feature]/discovery.md

```

**Expected outcome:**
- `[ ]` The SKILL.md detection condition for Entry A explicitly includes "discovery.md not found" or "no discovery.md"
- `[ ]` If running live, the response routes to the discovery-in-progress state (AC2a or AC2b from i1.1), NOT to Entry A

**Pass:** Detection condition explicit; live test (if run) routes correctly.
**Fail:** Detection condition allows Entry A even when discovery.md is present.
