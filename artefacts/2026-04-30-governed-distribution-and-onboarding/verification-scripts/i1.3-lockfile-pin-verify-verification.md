## Setup

**Before you start:**
1. Ensure Node.js is available.
2. Run `npm run platform:init` in a temp directory so skill files are present to hash.
3. Commands below assume you are in that temp directory.

**Reset between scenarios:** Each scenario runs in its own temp directory.

---

## Scenarios

### Scenario 1 — `platform:pin` creates `platform-lock.json` with correct schema (AC1)

**Command:**
```
cd /tmp/test-pin-1
npm run platform:init
npm run platform:pin
cat platform-lock.json
```

**Expected outcome (check all):**
- `[ ]` `platform-lock.json` was created
- `[ ]` Field `"version": "1"` present (string, not number)
- `[ ]` Field `"pinnedAt"` present with a valid ISO-8601 timestamp
- `[ ]` Field `"platform"` present and non-empty (a git SHA or version string)
- `[ ]` Field `"skills"` present as an object with at least one key
- `[ ]` Each key in `skills` is a forward-slash relative path (no backslashes, no drive letters)
- `[ ]` Each value in `skills` is a 64-character hex string (SHA-256)

**Pass:** All seven checkboxes satisfied.
**Fail:** Any checkbox unchecked.

### Scenario 2 — `platform:verify` exits 0 on a clean repo (AC2)

**Command:**
```
cd /tmp/test-verify-clean
npm run platform:init
npm run platform:pin
npm run platform:verify
echo "Exit code: $?"
```

**Expected outcome:**
- `[ ]` Exit code is `0`
- `[ ]` A single confirmation line is printed — e.g. `All N skill files match pinned version [identifier]. ✓`

**Pass:** Exit code 0; one-line confirmation shown.
**Fail:** Non-zero exit code; no confirmation; multiple lines of output when all files match.

---

### Scenario 3 — `platform:verify` exits non-zero and shows three-part error when a file drifts (AC3)

**Commands:**
```
cd /tmp/test-verify-drift
npm run platform:init
npm run platform:pin
# Introduce drift
echo "EXTRA LINE" >> .github/skills/orient/SKILL.md
npm run platform:verify
echo "Exit code: $?"
```

**Expected outcome (check all):**
- `[ ]` Exit code is non-zero
- `[ ]` Output names the drifted file (the relative path to the modified SKILL.md)
- `[ ]` Output shows the **expected** SHA-256 (the hash from platform-lock.json)
- `[ ]` Output shows the **actual** SHA-256 (the hash of the current file)
- `[ ]` Expected and actual hashes are different 64-character hex strings
- `[ ]` Output includes a fix command (`npm run platform:pin` or `npm run platform:fetch`)

**Pass:** All six checkboxes satisfied.
**Fail:** Any checkbox unchecked, or output is not self-contained (requires a separate file to interpret).

### Scenario 4 — `platform-lock.json` is portable (AC4)

**Commands:**
```
# On Machine A (or after init on current machine):
npm run platform:init
npm run platform:pin

# Copy platform-lock.json to a different directory structure (or simulate by checking path format):
node -e "
const lock = require('./platform-lock.json');
const keys = Object.keys(lock.skills);
const hasBackslash = keys.some(k => k.includes('\\\\'));
const hasAbsolute = keys.some(k => k.startsWith('/') || /^[A-Za-z]:/.test(k));
console.log('All paths use forward slashes:', !hasBackslash);
console.log('All paths are relative:', !hasAbsolute);
"
```

**Expected outcome:**
- `[ ]` `All paths use forward slashes: true`
- `[ ]` `All paths are relative: true`

**Pass:** Both lines print `true`.
**Fail:** Either line prints `false`.

### Scenario 5 — `platform:verify` exits non-zero with helpful message when no lock file exists (AC5)

**Command:**
```
cd /tmp/test-no-lock
npm run platform:init
# Do NOT run platform:pin
npm run platform:verify
echo "Exit code: $?"
```

**Expected outcome:**
- `[ ]` Exit code is non-zero
- `[ ]` stderr (or stdout) includes `npm run platform:pin` verbatim
- `[ ]` Error message is clear that the lock file is missing — not a generic crash

**Pass:** All three checkboxes satisfied.
**Fail:** The command crashes without a clear message, or does not include the fix command.
