### Scenario 5: A "Attribution incomplete" note in the benefit-metric artefact triggers a live re-check of the discovery field — it does not automatically block the story

**Covers:** AC4

**Steps:**
1. In `.github/skills/definition-of-ready/SKILL.md`, find the H-GOV block.
2. Find the instruction that describes what H-GOV does when it detects a benefit-metric artefact containing the note "Attribution incomplete" (added by `/benefit-metric` when option (b) was chosen).
3. Read the instruction carefully.

**Expected outcome:**
> When H-GOV detects the "Attribution incomplete" note in the benefit-metric artefact, it does **not** automatically fire and block the story. Instead, it performs a fresh read of the discovery artefact's `Approved By` field.
>
> - If `Approved By` is now substantively populated (the operator corrected it since the benefit-metric was written), H-GOV **passes** on this run.
> - If `Approved By` is still empty or a placeholder, H-GOV **fires** with the standard fail message.
>
> The benefit-metric note is a "check this again" signal — not a permanent block. Each DoR run re-reads the discovery field.

**Steps:**
1. In `.github/skills/definition-of-ready/SKILL.md`, find the trigger-to-check instruction (the part that handles the benefit-metric note).

> The note cannot permanently block a story — the live discovery field always has the final say.
