## Contract Proposal — i2.1

**What will be built:**
An extension to `.github/skills/orient/SKILL.md` adding an Entry A detection section:

1. Detection heuristic: Entry A fires when the repo contains `.md` files matching `[story-id]-*.md` under `artefacts/[feature]/stories/` — AND no `discovery.md` is present (presence of discovery.md, even unapproved, disqualifies Entry A).
2. Routing output for Entry A: Identifies state as "Entry A — story-first brownfield" by name; explains existing stories can be adopted by creating a discovery artefact; explicitly names `/tdd` as the inner loop entry point with a statement that existing story artefacts can be attached without re-derivation; routes to `/discovery` as the outer loop anchor.
3. Detection signal transparency: The routing output names what was found (e.g. "Found: story artefacts under artefacts/[feature]/stories/; no discovery.md found. This looks like Entry A (story-first brownfield).") and uses language that invites operator confirmation rather than asserting the classification.
4. Priority rule: Any `discovery.md` present (regardless of approval status) prevents Entry A classification — standard discovery-in-progress routing from i1.1 takes precedence.

**What will NOT be built:**
- Auto-importing or converting existing story files into the platform's story template format
- Handling repos with both stories AND a discovery in progress (standard i1.1 greenfield routing)
- Validating completeness or quality of detected story artefacts (presence-based detection only)

**How each AC will be verified:**

| AC | Test approach | Type |
|----|---------------|------|
| AC1 — Entry A detected + four-part routing output (a–d) | Assert orient SKILL.md has Entry A section; routing output names state, explains adoption, names /tdd with no-re-derivation statement, routes to /discovery | Unit (5 tests) |
| AC2 — When routing output displayed, it names the specific detection signal | Assert Entry A section includes instruction to state what was found; assert language invites operator correction | Unit (2 tests) |
| AC3 — Repo with discovery.md present NOT classified Entry A | Assert Entry A detection condition requires explicit absence of discovery.md; assert detection condition gates on discovery.md NOT present | Unit + Integration |

**Assumptions:**
- `.github/skills/orient/SKILL.md` exists at time of implementation (created by i1.1)
- "No discovery.md" means no file matching `discovery.md` is present under `artefacts/` (any location)
- Story file presence check uses `artefacts/[feature]/stories/` path convention
