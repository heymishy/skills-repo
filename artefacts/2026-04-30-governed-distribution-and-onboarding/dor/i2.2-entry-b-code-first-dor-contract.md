## What will be built

An extension to `.github/skills/orient/SKILL.md` adding an Entry B detection and routing section:

1. Entry B detection signal: application logic files with extensions `.js`, `.ts`, `.py`, `.java`, `.rb`, `.go`, `.cs`, `.cpp`, `.rs`, `.swift`, or `.kt` present under `src/`, `app/`, or `lib/` directories AND no `artefacts/` folder (or `artefacts/` is empty)
2. Entry B routing output: acknowledge the existing codebase; recommend `/reverse-engineer` to extract business rules and generate artefacts; then proceed with `/discovery`
3. Priority rule (explicit in skill file): Entry A takes precedence — if story artefacts under `artefacts/` are present, Entry A fires, not Entry B
4. Boundary rule (explicit in skill file): repos containing only config/infrastructure files (`.yml`, `.json`, Dockerfile, Terraform files) do NOT trigger Entry B; fall through to Entry C

## What will NOT be built

- Auto-execution of /reverse-engineer
- Language-specific detection variants
- Detection for repos where artefacts/ is present and non-empty (Entry A handles this)

## Entry B detection boundary (canonical reference)

| Signal type | Examples | Entry B fires? |
|-------------|---------|---------------|
| App logic files in src/app/lib | src/index.js, app/main.py | Yes |
| App logic files NOT in src/app/lib | utils.js, config.js in root | No (not in target dirs) |
| Config/infra only | docker-compose.yml, terraform/ | No — falls through to Entry C |
| App logic + artefacts/ present | src/index.js + artefacts/story.md | No — Entry A fires instead |

- Detection heuristic is expressed as natural language in the skill file — not executable code
