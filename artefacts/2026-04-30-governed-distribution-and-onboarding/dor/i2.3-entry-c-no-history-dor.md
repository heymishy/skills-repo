**What will be built:**
An extension to `.github/skills/orient/SKILL.md` adding an Entry C fallback section:

1. Entry C fires as the fallback when all of the following are true:
   - Entry A does NOT fire (no pipeline story artefacts under `artefacts/`)
   - Entry B does NOT fire (no application logic files in `src/`, `app/`, `lib/`)
   - The repo is not empty (has at least some files or git history)
2. Entry C routing output: acknowledges the consumer's position (delivered software, no accessible source); recommends using `.github/templates/retrospective-story.md` to create a retrospective story artefact; then proceed with `/discovery`.
3. The word "retrospective" must appear explicitly in the routing output.
4. Critical AC3 boundary: a truly empty new repo (no git history, no files) → Entry C does NOT fire → greenfield path (`/discovery`) fires instead. This is the boundary between Entry C and the greenfield case covered by i1.1 AC1.

**What will NOT be built:**
- Auto-creation of retrospective discovery
- Distinguishing archived vs active legacy systems
- CI/CD-only repos treated as Entry B (they fall into Entry C)

**How each AC will be verified:**

| AC2 — Entry C fallback only fires if Entry A and Entry B did not fire | Assert Entry A fires when artefacts present (Entry C does not); assert Entry B fires when code-first signals present (Entry C does not) | Unit |
| AC3 — truly empty repo does NOT trigger Entry C; greenfield fires instead | Assert entry C boundary: no history + no files → greenfield routing, not Entry C | Unit + Integration |
| NFR-retrospective | Assert word "retrospective" appears in Entry C routing output | Unit |

**Assumptions:**
- `.github/templates/retrospective-story.md` exists at implementation time (pre-existing template in this repo)
- "Truly empty repo" condition: no git history (clean `git init`) AND no files in the working tree (or only hidden system files)
