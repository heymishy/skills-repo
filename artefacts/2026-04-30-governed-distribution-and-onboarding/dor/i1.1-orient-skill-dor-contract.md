## What will be built

Create `.github/skills/orient/SKILL.md` — a new pipeline skill file with the following:

1. YAML frontmatter: `name: orient`, description identifying /orient as distinct from /start and /workflow
2. Routing instructions for 8 pipeline states:
   - AC1: New repo (no artefacts) → `/discovery`
   - AC2a: Discovery exists but incomplete → `/clarify` with named incompleteness
   - AC2b: Discovery complete but unapproved → `/discovery` for approval
   - AC3: Discovery approved + no benefit-metric → `/benefit-metric`
   - AC4: Benefit-metric active + no stories → `/definition`
   - AC5: Stories exist + story missing test plan → `/test-plan` naming the story
   - AC6a: DoR signed-off + branch/worktree found → `/verify-completion`
   - AC6b: DoR signed-off + no branch → `/branch-setup`
3. Explicit statement distinguishing /orient from /start (greenfield only, single-use) and /workflow (diagnostic for experienced operators)

## What will NOT be built

- Brownfield Entry A/B/C routing (i2.1/i2.2/i2.3 scope)
- Automated skill execution — /orient routes only
- Artefact quality validation — /workflow and /trace scope
- Multi-repo or fleet-level orientation
- Any change to an existing SKILL.md file
