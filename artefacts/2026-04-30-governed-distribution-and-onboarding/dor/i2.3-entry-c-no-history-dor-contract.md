## What will be built

An extension to `.github/skills/orient/SKILL.md` adding an Entry C fallback section:

1. Entry C trigger condition:
   - Entry A signals absent (no pipeline story artefacts under `artefacts/`)
   - Entry B signals absent (no application logic files in `src/`, `app/`, `lib/`)
   - Repo is NOT a truly empty new repo — has at least one non-hidden file or has git history

2. Entry C routing output:
   - Acknowledges that the consumer has delivered software without an accessible codebase
   - Recommends using `.github/templates/retrospective-story.md` to create a retrospective story
   - Directs consumer to proceed with `/discovery` after creating the retrospective story
   - The word "retrospective" MUST appear explicitly in the routing text (NFR)

3. Boundary note (explicit in skill file):
   - Truly empty repo: no git history + no files → greenfield path, not Entry C → routes to `/discovery` directly (same as i1.1 AC1)

4. Routing priority order in skill file (final): Entry A → Entry B → Entry C → Greenfield

## What will NOT be built

- Auto-creation of retrospective discovery artefacts
- Distinguishing archived vs active legacy systems
- CI/CD-only repo handling (these fall into Entry C, not Entry B)

## Entry C detection table

| Condition | Entry A fires? | Entry B fires? | Repo empty? | Entry C fires? |
|-----------|---------------|----------------|-------------|----------------|
| Pipeline artefacts in artefacts/ | Yes | — | — | No |
| App logic files in src/app/lib | No | Yes | — | No |
| No artefacts, no code, config files only | No | No | No | Yes |
| No artefacts, no code, git history only | No | No | No | Yes |
| No artefacts, no code, no git history, no files | No | No | Yes (empty) | No → greenfield |
