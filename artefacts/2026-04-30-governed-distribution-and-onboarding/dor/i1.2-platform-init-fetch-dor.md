## Contract Proposal — i1.2

**What will be built:**
Two new Node.js CLI scripts and two new `package.json` entries:
1. `scripts/platform-init.js` — copies `.github/skills/`, `.github/templates/`, and `scripts/` platform files to a target repo; creates/updates `.github/copilot-instructions.md`; outputs an installation summary; skips existing files by default; supports `--force` flag.
2. `scripts/platform-fetch.js` — fetches the current HEAD skill files from the platform's master branch; copies to the repo; records the fetch timestamp in `workspace/`; requires no git remote setup from the consumer.
3. `package.json` — adds `"platform:init": "node scripts/platform-init.js"` and `"platform:fetch": "node scripts/platform-fetch.js"` to the `scripts` section.

**What will NOT be built:**
- `platform:pin` and `platform:verify` (i1.3 scope)
- Automated conflict resolution for customised files (--force flag only)
- Alternative package manager support (yarn, pnpm)
- Windows PowerShell-specific install path

**How each AC will be verified:**

| AC | Test approach | Type |
|----|---------------|------|
| AC1 — init copies files + creates copilot-instructions.md + outputs summary | Assert script exists; package.json entry exists; script copies expected files to temp dir; copilot-instructions.md is created; summary output present | Unit + Integration |
| AC2 — VS Code skills available after init | Manual scenario — requires opening VS Code after init; IDE-dependent behaviour cannot be asserted by Node.js test | Manual |
| AC3 — existing files not overwritten; --force overwrites | Assert existing files preserved on second init; --force flag causes overwrite | Unit |
| AC4 — platform:fetch retrieves latest + records timestamp | Assert script exists; package.json entry exists; fetch from local path creates files in target; timestamp recorded in workspace/ | Unit + Integration |
| AC5 — no new npm runtime deps | Assert package.json `dependencies` count unchanged after init; verify script uses only Node.js built-ins | Unit |
| NFR-idempotent | Run init twice in temp dir — assert identical state after both runs | Unit |
| NFR-error | Trigger failure condition (missing Node.js?); assert non-zero exit code and human-readable error | Unit |

**Assumptions:**
- The "platform files" source is this repository itself (scripts read from `__dirname` or repo root)
- `platform:fetch` can use a local path or `git archive` rather than a live network call for testability
- The `workspace/` directory exists or is created by the fetch script

**Estimated touch points:**
Files created: `scripts/platform-init.js`, `scripts/platform-fetch.js`
Files modified: `package.json` (adds 2 script entries)
Services: None
APIs: None
