## What will be built

Edit `.github/skills/definition-of-ready/SKILL.md` to add:

1. `H-GOV` row in the hard blocks table, positioned after the H-NFR series
2. H-GOV detail section with four cases:
   - AC1 PASS: `Approved By` section exists with ≥1 non-blank named entry → H-GOV passes; record M1 signal (role-unverified if text-only, positive M1 signal if clearly non-engineering)
   - AC2 FAIL — empty section: `Approved By` section exists but is empty → fail message: names field, names format, states remediation
   - AC3 FAIL — absent section: no `Approved By` section present (pre-i3.1 artefact) → distinct message from empty; references adding attribution section first
   - AC4 FAIL — engineer-only entries: all entries are engineering roles → distinct from empty; states entries present but role requirement not met
3. AC4 trigger-to-check clause: when benefit-metric artefact carries "Attribution incomplete" note → live read of discovery `Approved By` → pass if corrected; fail with standard AC2 message if still empty
4. All fail messages contain: (a) field name `Approved By`, (b) format `Name — Role — Date`, (c) remediation step; total under 200 words
5. Block uses identifier `H-GOV`, check description, pass condition, and fail message matching H-series format
