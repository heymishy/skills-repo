## Unit Tests

### orient-entry-a-routing-block-present

- **Verifies:** AC1 prerequisite — Entry A routing logic exists in SKILL.md
- **Precondition:** `.github/skills/orient/SKILL.md` exists
- **Action:** Read file; check for "Entry A" string
- **Expected result:** "Entry A" appears in the file
- **Edge case:** No

### orient-entry-a-names-state-story-first-brownfield

- **Verifies:** AC1(a) — output identifies state as "Entry A — story-first brownfield" by name
- **Precondition:** SKILL.md exists
- **Action:** Read file; check for the exact phrase "Entry A — story-first brownfield" or equivalent canonical label
- **Expected result:** The Entry A state is named explicitly, not described generically
- **Edge case:** No

### orient-entry-a-explains-adoption-path

- **Verifies:** AC1(b) — explains that existing stories can be adopted by creating a discovery artefact
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry A routing section contains language about adopting existing stories via discovery
- **Expected result:** Entry A section explains that a discovery artefact captures the stories' intent (adoption, not discarding)
- **Edge case:** No

### orient-entry-a-names-tdd-as-inner-loop-entry-with-no-rederivation

- **Verifies:** AC1(c) — explicitly names /tdd as inner loop entry point AND states existing artefacts can be attached without re-derivation
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry A section contains `/tdd` and language about not needing to re-derive existing story artefacts
- **Expected result:** `/tdd` named as inner loop entry; phrase about "existing story artefacts" + "without re-deriving" (or equivalent) present
- **Edge case:** No

### orient-entry-a-routes-to-discovery-as-outer-anchor

- **Verifies:** AC1(d) — routes to /discovery with note to use existing stories as input scope
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry A section routes to `/discovery` and references existing stories as input scope
- **Expected result:** `/discovery` appears as the outer loop anchor command in Entry A routing; note about using existing stories as input is present
- **Edge case:** No

### orient-entry-a-names-detection-signal

- **Verifies:** AC2 — routing output names the specific detection signal
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry A section instructs the skill to name what it found (e.g. "Found: story artefacts under artefacts/[feature]/stories/")
- **Expected result:** Entry A routing section includes an instruction to state the detection signal (what files were found, where)
- **Edge case:** No

### orient-entry-a-signal-allows-operator-correction

- **Verifies:** AC2 — operator can confirm or correct the classification
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry A section language that expresses uncertainty or invites correction (e.g. "This looks like Entry A" rather than "This IS Entry A")
- **Expected result:** Entry A classification is presented as a classification to confirm, not an assertion — language like "This looks like" or "Detected as" is present
- **Edge case:** No

### orient-entry-a-nfr-no-false-entry-a-with-discovery-present

- **Verifies:** NFR — repo with discovery.md present (even unapproved) must NOT be classified as Entry A
- **Precondition:** SKILL.md exists
- **Action:** Read file; check that Entry A detection rule explicitly requires absence of discovery.md — i.e. the detection condition states "no discovery.md present" or "discovery.md not found"
- **Expected result:** Entry A detection in SKILL.md is gated on discovery.md being absent — not just on stories being present
- **Edge case:** Yes — the priority ordering (discovery-in-progress takes precedence) is a key correctness requirement
