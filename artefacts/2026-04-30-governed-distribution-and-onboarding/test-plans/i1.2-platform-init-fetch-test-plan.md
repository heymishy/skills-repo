## Test Data Strategy

**Source:** Synthetic — tests create temporary directories with controlled content via `os.tmpdir()` + `fs.mkdtempSync`.
**PCI/sensitivity in scope:** No
**Availability:** Available now — tests use Node.js built-ins to create and destroy temp dirs.
**Owner:** Self-contained — each test creates and cleans up its own temp directory.

### Data requirements per AC

| AC | Data needed | Source | Sensitive fields | Notes |
|----|-------------|--------|-----------------|-------|
| AC1 | Source platform files to copy from (repo root) | Existing repo files | None | Tests use the actual .github/skills/, .github/templates/, scripts/ from this repo as the source |
| AC3 | Target repo with pre-existing files | Synthetic temp dir | None | Temp dir with placeholder files simulating a prior platform:init |
| AC4 | Fetch source (platform upstream) | Mocked or actual network | None | Test may use a local copy of the repo as the fetch source to avoid real network calls |
| AC5 | package.json before and after | Synthetic temp dir | None | — |

### PCI / sensitivity constraints

None.

### Gaps

AC4 (platform:fetch) requires a network call to a git remote for a live test. To keep tests zero-external-dependency, the test will verify that the fetch script exists, records a timestamp, and produces the expected output structure when given a local path as the source — network-live fetch is a manual verification step.

## Unit Tests

### platform-init-script-exists

- **Verifies:** AC1 prerequisite — script file exists
- **Precondition:** Repo checked out
- **Action:** `fs.existsSync(path.join(root, 'scripts/platform-init.js'))`
- **Expected result:** `true`
- **Edge case:** No

### package-json-has-platform-init-entry

- **Verifies:** AC1 — `npm run platform:init` command is registered
- **Precondition:** package.json exists
- **Action:** Read package.json scripts; check for `platform:init` key
- **Expected result:** `scripts['platform:init']` is defined and points to `scripts/platform-init.js`
- **Edge case:** No

### package-json-has-platform-fetch-entry

- **Verifies:** AC4 — `npm run platform:fetch` command is registered
- **Precondition:** package.json exists
- **Action:** Read package.json scripts; check for `platform:fetch` key
- **Expected result:** `scripts['platform:fetch']` is defined
- **Edge case:** No

### platform-init-copies-skills-directory

- **Verifies:** AC1 — .github/skills/ files copied to target
- **Precondition:** Temp target dir, platform:init script exists
- **Action:** Run platform-init in temp dir; check that .github/skills/ contents are present in target
- **Expected result:** At least one .md file exists under target/.github/skills/
- **Edge case:** No

### platform-init-copies-templates-directory

- **Verifies:** AC1 — .github/templates/ files copied to target
- **Precondition:** Temp target dir
- **Action:** Run platform-init in temp dir; check .github/templates/ in target
- **Expected result:** At least one template file exists under target/.github/templates/
- **Edge case:** No

### platform-init-creates-copilot-instructions

- **Verifies:** AC1 — .github/copilot-instructions.md created or updated
- **Precondition:** Temp target dir with no existing copilot-instructions.md
- **Action:** Run platform-init in temp dir; check file exists and is non-empty
- **Expected result:** `.github/copilot-instructions.md` exists and has content
- **Edge case:** No

### platform-init-outputs-install-summary

- **Verifies:** AC1 — outputs summary of what was installed
- **Precondition:** Temp target dir
- **Action:** Capture stdout of platform-init; check that it contains a summary line
- **Expected result:** stdout includes "installed" or equivalent summary confirming copied files
- **Edge case:** No

### platform-init-skips-existing-files-by-default

- **Verifies:** AC3 — non-destructive default; existing files not overwritten
- **Precondition:** Temp target dir with a pre-existing .github/skills/orient/SKILL.md containing a custom marker
- **Action:** Run platform-init; read the file again
- **Expected result:** File content still contains the custom marker (was not overwritten)
- **Edge case:** No

### platform-init-reports-skipped-files

- **Verifies:** AC3 — reports skipped files in output
- **Precondition:** Temp target dir with pre-existing files
- **Action:** Capture stdout of platform-init; check for "skipped" keyword
- **Expected result:** stdout includes "skipped" and lists at least the pre-existing file
- **Edge case:** No

### platform-init-force-flag-overwrites-existing

- **Verifies:** AC3 — --force flag overwrites existing files
- **Precondition:** Temp target dir with pre-existing file containing custom marker
- **Action:** Run platform-init --force; read the file
- **Expected result:** File content no longer contains the custom marker (was overwritten)
- **Edge case:** No

### platform-fetch-records-timestamp

- **Verifies:** AC4 — records fetch timestamp in workspace/
- **Precondition:** Temp target dir with workspace/ folder
- **Action:** Run platform-fetch (or platform-init with fetch mode) against local source; check workspace/ for a timestamp file
- **Expected result:** A file under workspace/ contains an ISO-8601 timestamp from the fetch operation
- **Edge case:** No

### platform-fetch-copies-latest-skill-files

- **Verifies:** AC4 — fetches current skill files
- **Precondition:** Temp target dir; local path used as fetch source
- **Action:** Run platform-fetch with local source path; check skills directory in target
- **Expected result:** Skills files present in target directory after fetch
- **Edge case:** No

### platform-init-adds-no-runtime-dependencies

- **Verifies:** AC5 / NFR-zero-deps — package.json dependencies unchanged
- **Precondition:** Record target package.json dependencies before; run platform-init
- **Action:** Read target package.json; compare `dependencies` field before and after
- **Expected result:** `dependencies` field identical before and after (no additions)
- **Edge case:** No

### platform-init-no-require-of-external-modules

- **Verifies:** AC5 / NFR-zero-deps — implementation uses only Node.js built-ins
- **Precondition:** platform-init.js exists
- **Action:** Read platform-init.js source; check that `require()` calls reference only built-in modules (fs, path, crypto, child_process, os, http, https, url, stream)
- **Expected result:** No require of external packages (no node_modules references in require calls)
- **Edge case:** Yes — also check platform-fetch script if separate file

### platform-init-is-idempotent

- **Verifies:** NFR-idempotent — second run of platform:init on same dir produces identical result
- **Precondition:** Temp target dir
- **Action:** Run platform-init twice; compare the state of the target dir after each run
- **Expected result:** Target directory contents are byte-for-byte identical after first and second run (same files, same content, no duplicates)
- **Edge case:** Yes — the second run must skip existing files without error

### platform-init-exits-nonzero-on-failure

- **Verifies:** NFR-error — exits with non-zero code on failure
- **Precondition:** Mock a failure condition (e.g. read-only target directory)
- **Action:** Run platform-init; check process exit code
- **Expected result:** Exit code is non-zero (not 0) when the command cannot complete
- **Edge case:** No

### platform-init-outputs-human-readable-error

- **Verifies:** NFR-error — error message is human-readable and names the specific failure
- **Precondition:** Same failure condition as above
- **Action:** Capture stderr; check that it contains a description of the failure
- **Expected result:** stderr contains a human-readable explanation — not just a stack trace or "error"
- **Edge case:** No

## Integration Tests

### platform-init-full-install-skills-accessible

- **Verifies:** AC1 end-to-end — all required directories present after init
- **Components involved:** platform-init.js, repo source files (.github/skills/, .github/templates/, scripts/)
- **Precondition:** Fresh temp target directory
- **Action:** Run platform-init; verify presence of .github/skills/, .github/templates/, scripts/, .github/copilot-instructions.md
- **Expected result:** All four paths exist in target after a single init run

### platform-fetch-produces-workspace-timestamp

- **Verifies:** AC4 integration — fetch + timestamp recording work together
- **Components involved:** platform-fetch logic, workspace/ directory management
- **Precondition:** Temp dir with workspace/ folder; local source path
- **Action:** Run platform-fetch; check workspace/ contains a fetch-record file with parseable ISO-8601 timestamp
- **Expected result:** Timestamp is a valid ISO-8601 date string within the last 10 seconds

### platform-init-then-fetch-no-duplicate-dependencies

- **Verifies:** AC5 / NFR-zero-deps integration — platform:init + platform:fetch together add no runtime deps
- **Components involved:** platform-init.js, platform-fetch.js, package.json
- **Precondition:** Temp dir with minimal package.json (no dependencies)
- **Action:** Run platform:init then platform:fetch; read package.json
- **Expected result:** `dependencies` field is absent or empty
