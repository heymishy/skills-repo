## Unit Tests

### platform-pin-script-exists

- **Verifies:** AC1 prerequisite
- **Precondition:** Repo checked out
- **Action:** `fs.existsSync(path.join(root, 'scripts/platform-pin.js'))`
- **Expected result:** `true`
- **Edge case:** No

### package-json-has-platform-pin-entry

- **Verifies:** AC1 — `npm run platform:pin` registered
- **Precondition:** package.json exists
- **Action:** Read package.json; check `scripts['platform:pin']` defined
- **Expected result:** Entry exists and references `scripts/platform-pin.js`
- **Edge case:** No

### package-json-has-platform-verify-entry

- **Verifies:** AC2 / AC3 / AC5 — `npm run platform:verify` registered
- **Precondition:** package.json exists
- **Action:** Read package.json; check `scripts['platform:verify']` defined
- **Expected result:** Entry exists
- **Edge case:** No

### platform-pin-creates-lock-file

- **Verifies:** AC1 — platform-lock.json created after pin
- **Precondition:** Temp dir with at least one skill file installed
- **Action:** Run platform:pin; check `platform-lock.json` exists in target
- **Expected result:** File exists
- **Edge case:** No

### platform-pin-lock-file-has-schema-version-1

- **Verifies:** AC1 — schema version field is "1"
- **Precondition:** platform-lock.json created
- **Action:** Parse JSON; check `lock.version === "1"`
- **Expected result:** `version: "1"` (string, not number)
- **Edge case:** No

### platform-pin-lock-file-has-iso8601-timestamp

- **Verifies:** AC1 — pinnedAt is ISO-8601
- **Precondition:** platform-lock.json created
- **Action:** Parse JSON; run `new Date(lock.pinnedAt).toISOString()` — must not throw; check the value round-trips
- **Expected result:** `lock.pinnedAt` is a valid ISO-8601 date string
- **Edge case:** No

### platform-pin-lock-file-has-platform-identifier

- **Verifies:** AC1 — platform field (git SHA or version string) is present and non-empty
- **Precondition:** platform-lock.json created
- **Action:** Parse JSON; check `lock.platform` is a non-empty string
- **Expected result:** `lock.platform` is truthy
- **Edge case:** No

### platform-pin-hashes-each-skill-file

- **Verifies:** AC1 — SHA-256 hash of each skill file written to skills object
- **Precondition:** Temp dir with 2 known-content skill files; platform:pin run
- **Action:** Parse JSON; compute expected SHA-256 for each file using `crypto.createHash`; compare against `lock.skills[path]`
- **Expected result:** Each file's hash in platform-lock.json matches independently computed SHA-256
- **Edge case:** No

### platform-pin-keys-by-relative-path

- **Verifies:** AC1 — skill keys use relative paths
- **Precondition:** platform-lock.json created with files at nested paths
- **Action:** Check keys of `lock.skills` object — they must not be absolute paths
- **Expected result:** No key starts with a drive letter (Windows) or `/` (Unix absolute)
- **Edge case:** No

### platform-pin-paths-use-forward-slashes

- **Verifies:** NFR-paths — forward slashes on Windows
- **Precondition:** Run platform:pin (may be on Windows where path.sep is `\`)
- **Action:** Parse lock.skills keys; check each key for backslash characters
- **Expected result:** No backslash in any key — all use `/`
- **Edge case:** Yes — specifically a Windows edge case

### platform-verify-exits-0-on-clean-repo

- **Verifies:** AC2 — exits 0 when all files match
- **Precondition:** Temp dir with platform-lock.json; skill files unchanged since pin
- **Action:** Run platform:verify; capture exit code
- **Expected result:** Exit code 0
- **Edge case:** No

### platform-verify-outputs-one-line-confirmation

- **Verifies:** AC2 — one-line confirmation with file count and platform identifier
- **Precondition:** Same as above
- **Action:** Capture stdout; check it is a single line containing the file count and platform identifier
- **Expected result:** stdout matches pattern `All N skill files match pinned version [identifier]. ✓` (or equivalent)
- **Edge case:** No

### platform-verify-exits-nonzero-on-drift

- **Verifies:** AC3 — exits non-zero when a file has changed
- **Precondition:** Temp dir with lock; modify one skill file after pin
- **Action:** Run platform:verify; capture exit code
- **Expected result:** Exit code is non-zero
- **Edge case:** No

### platform-verify-names-drifted-file

- **Verifies:** AC3(a) — names each drifted file in error output
- **Precondition:** Same drift scenario
- **Action:** Capture stderr; check it contains the relative path of the modified file
- **Expected result:** The relative path of the modified skill file appears in stderr
- **Edge case:** No

### platform-verify-shows-expected-vs-actual-sha

- **Verifies:** AC3(b) — shows expected SHA vs actual SHA for each drifted file
- **Precondition:** Same drift scenario
- **Action:** Capture stderr; check it contains both the expected hash (from lock) and actual hash (computed from modified file) — must be two different hex strings
- **Expected result:** Both the original (expected) hash and the new (actual) hash appear in stderr output
- **Edge case:** No

### platform-verify-names-fix-command

- **Verifies:** AC3(c) — tells operator the exact fix command
- **Precondition:** Same drift scenario
- **Action:** Capture stderr; check for `npm run platform:pin` or `npm run platform:fetch`
- **Expected result:** At least one of the two fix commands appears verbatim in stderr
- **Edge case:** No

### platform-verify-exits-nonzero-when-no-lock-file

- **Verifies:** AC5 — exits non-zero with no platform-lock.json present
- **Precondition:** Temp dir with skill files but no platform-lock.json
- **Action:** Run platform:verify; capture exit code
- **Expected result:** Exit code is non-zero
- **Edge case:** No

### platform-verify-instructs-run-pin-when-no-lock

- **Verifies:** AC5 — error message instructs running platform:pin first
- **Precondition:** Same no-lock-file scenario
- **Action:** Capture stderr; check it contains `npm run platform:pin`
- **Expected result:** `npm run platform:pin` appears verbatim in stderr
- **Edge case:** No

### platform-pin-sha256-deterministic

- **Verifies:** NFR-deterministic — same file content produces same SHA-256 each run
- **Precondition:** Temp dir with one known-content file
- **Action:** Run platform:pin twice; compare the hash values in both lock files
- **Expected result:** Hash value is identical on both runs
- **Edge case:** No

### platform-pin-sha256-is-raw-content-only

- **Verifies:** NFR-deterministic — hash computed on file content, not metadata
- **Precondition:** Temp dir with one file; compute expected hash via `crypto.createHash('sha256').update(fs.readFileSync(file)).digest('hex')`
- **Action:** Compare against hash in platform-lock.json
- **Expected result:** Hashes match exactly — confirming the implementation hashes raw content not metadata
- **Edge case:** No

### platform-verify-nfr-actionable-error

- **Verifies:** NFR-actionable — three-part error is self-contained (no need to open another document)
- **Precondition:** Drift scenario
- **Action:** Capture stderr; verify it contains: (a) the file path, (b) two SHA hex strings, (c) a fix command — all in one output block without requiring additional files to interpret
- **Expected result:** All three parts present in a single stderr output
- **Edge case:** No

## Integration Tests

### platform-pin-then-verify-clean-passes

- **Verifies:** AC2 integration — full pin + verify cycle on clean files
- **Components involved:** platform-pin.js, platform-verify.js, temp skill files
- **Precondition:** Fresh temp dir with 3 synthetic skill files
- **Action:** Run platform:pin; run platform:verify without modifying any file
- **Expected result:** platform:verify exits 0

### platform-pin-then-modify-then-verify-fails

- **Verifies:** AC3 integration — full pin + drift + verify cycle
- **Components involved:** platform-pin.js, platform-verify.js
- **Precondition:** Fresh temp dir; run platform:pin; modify one file
- **Action:** Run platform:verify
- **Expected result:** platform:verify exits non-zero; stderr names the modified file

### platform-lock-portable-across-paths

- **Verifies:** AC4 — portable across environments (path separator handling)
- **Components involved:** platform-pin.js output, platform-verify.js input
- **Precondition:** platform-lock.json created on current OS; manually rewrite skill path keys to use OS-native separator (backslash on Windows)
- **Action:** Run platform:verify with the path-modified lock file
- **Expected result:** platform:verify normalises paths and still matches correctly — OR rejects gracefully with a clear error (not a crash)
- **Edge case:** Yes — cross-OS portability is the explicit AC4 requirement
