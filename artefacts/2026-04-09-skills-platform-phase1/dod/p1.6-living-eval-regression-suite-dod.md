## DoD Observations

1. **AC6 outcome-orientation diagnostic review is a repeating process obligation:** AC6 is assessed at initial delivery only, but every new scenario addition via the AC3 growth mechanic requires the same diagnostic review. The `reviewNote` field is the audit trail. Phase 2 platform improvement agent should automate the diagnostic check (outcome-orientation checker as a growth-gate tool). **Candidate /levelup finding.**

2. **p1.4 + p1.6 closed-loop dependency at DoD was correctly sequenced:** The M4 controlled acceptance test (AC5) depended on p1.4 DoD completion. Ordering this DoD sequence (p1.4 before p1.6 in the DoD run) was intentional and correct — it unblocked AC5 validation. This sequencing pattern (validate downstream-dependent ACs in delivery order) should be noted in the DoD skill guidance. **Candidate /levelup finding.**

3. **Cross-story schema contract gap — p1.6 as producing story:** p1.6 authored `workspace/suite.json` as a core deliverable. The schema it produced contained `version`, `description`, and `scenarios` fields. p1.4's `watermark-gate.js` required two additional fields at runtime — `skillSetHash` (git tree hash of `.github/skills/`) and `surfaceType` (string, e.g. `"github-copilot"`) — that p1.6 never wrote. This caused a live CI failure on p1.4's PR #16: `[watermark-gate] ERROR: suite.json missing or invalid field: skillSetHash`. Neither the p1.6 DoR contract nor the p1.4 DoR contract made the consuming schema requirements explicit. The fix was adding both fields to `workspace/suite.json` during PR #16 conflict resolution; the corrected file is in master as of 2026-04-11. **Finding:** when a story produces a file consumed by another story, the consuming story's required schema must be documented in the producer's DoR contract as an explicit output schema constraint — not just the file path. **Actioned /levelup finding (2026-04-11):** DoD template write-back complete — `.github/templates/definition-of-done.md` DoD Observations section now names cross-story runtime failures as an explicit observation type. DoR skill cross-story schema check (require consuming story to document fields read; propagate back to producer's DoR contract) deferred to Phase 2 pipeline improvement — tracked in `workspace/learnings.md` (cross-story schema dependency entry, line ~261).
