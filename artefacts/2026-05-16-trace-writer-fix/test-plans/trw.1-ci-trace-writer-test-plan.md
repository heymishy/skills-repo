## Unit tests — `scripts/write-ci-trace.js`

### T1 — Record is written to disk
**Given** environment variables GITHUB_RUN_ID, GITHUB_SHA, GITHUB_REF are set
**When** `write-ci-trace.js` runs
**Then** a JSONL file exists in `workspace/traces/`
**File path:** `tests/check-trw1-trace-writer.js`

### T2 — Record filename follows naming convention
**Given** GITHUB_SHA = `bd7f996fXXXXXXXX` and a known timestamp
**When** the script runs
**Then** the filename matches `{ISO-timestamp}-ci-{8-char-sha}.jsonl`
**Assertion:** `fs.existsSync(path.join(tracesDir, filename))` === true

### T3 — Record content is valid JSON on one line
**Given** a fresh run of the script
**When** the output file is read
**Then** each line is valid JSON with no trailing commas
**Assertion:** `JSON.parse(line)` does not throw

### T4 — Required fields are present in the record
**Given** a record generated by the script
**When** parsed
**Then** the object contains all 7 required fields: `runId`, `commitSha`, `headRef`, `trigger`, `timestamp`, `verdict`, `surface`
**Assertion:** check each field is present and non-empty

### T5 — `trigger` field is "post-merge"
**Given** a record generated by the script
**Then** `record.trigger === "post-merge"`

### T6 — `verdict` field is "trace-committed"
**Then** `record.verdict === "trace-committed"`

### T7 — `surface` field is "ci-trace-commit"
**Then** `record.surface === "ci-trace-commit"`

### T8 — `timestamp` is a valid ISO 8601 UTC string
**Then** `new Date(record.timestamp).toISOString() === record.timestamp`

### T9 — Script creates output directory if absent
**Given** `workspace/traces/` does not exist
**When** the script runs
**Then** the directory is created and the file is written without error

### T10 — Script exits 1 on write failure
**Given** `workspace/traces/` is read-only (chmod 444 or similar)
**When** the script runs
**Then** exit code is 1 and an error message is written to stderr

### T11 — Script does not log credentials
**Given** environment variables contain a `GITHUB_TOKEN`
**When** the script runs
**Then** the output file content does not include the token value
**Assertion:** file content does not match the token string

## NFR — Security

- The `write-ci-trace.js` script must not log `GITHUB_TOKEN` or any other secret to stdout, stderr, or the output JSONL file (T11)
- Path traversal: the output path must be constructed from a hardcoded base (`workspace/traces/`) — never from user-controlled input
