# Spike Brief: H7.1 — Web UI Backend Subprocess Integration Assessment

**Created:** 2026-05-24
**Feature:** 2026-05-19-cli-deterministic-governance
**Blocking:** Phase 2 definition — `skills advance` state writes and web UI gate-confirm integration

---

## Spike question

Can the web UI backend integrate with the `skills validate` and `skills advance` CLI commands, and if so, by which mechanism — subprocess invocation or direct module import — and what is the cost/risk of each?

## Type

Technical feasibility (Type 1) — the integration mechanism is a concrete architectural question answerable by reading existing code.

## Scope

Standard — multiple sources examined (cli-outer-loop.js, cli-adapter.js, governance-package.js, pipeline-state-writer.js, journey.js gate-confirm handler, server.js wiring). Up to 6 steps.

## Done condition

Options assessed against integration cost, testability, and security risk. A recommended approach written up with implementation implications for Phase 2 definition.

## Out of scope

- Implementing Phase 2 stories — this spike assesses feasibility only
- Evaluating the `skills emit-trace` (chain-hash trace) implementation design in detail
- Windows `.cmd` CLI wrapper
