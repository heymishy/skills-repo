## Unit Tests

### T1 — advances story dorStatus atomically on valid args

- **Verifies:** AC1
- **Precondition:** Temporary `pipeline-state.json` fixture written to `workspace/test-tmp-cdg3/pipeline-state.json` containing feature `test-feature-cdg3` with story `test-story-cdg3` where `dorStatus: "not-started"`. Environment variable `PIPELINE_STATE_PATH` (or equivalent) points to the temp file.
- **Action:** Execute `node bin/skills advance test-feature-cdg3 test-story-cdg3 dorStatus=signed-off` with the temp file in scope (pass `repoRoot` or state path as environment/arg). Read `pipeline-state.json` after the command exits.
- **Expected result:** Exit code is 0. `pipeline-state.json` has `dorStatus: "signed-off"` for the target story. `stdout` contains "test-feature-cdg3" and "test-story-cdg3".
- **Edge case:** No — this is the happy path.

### T2 — exits 8 when feature slug is not in pipeline-state.json

- **Verifies:** AC2
- **Precondition:** Temp `pipeline-state.json` exists but does NOT contain feature slug `unknown-slug-cdg3`. `pipeline-state.json` content is valid JSON with an unrelated feature.
- **Action:** Execute `node bin/skills advance unknown-slug-cdg3 some-story dorStatus=signed-off`.
- **Expected result:** Exit code is 8. `stderr` contains "unknown-slug-cdg3". `pipeline-state.json` is byte-for-byte identical to the pre-command state (not modified).
- **Edge case:** Yes — file integrity check: `JSON.stringify(before) === JSON.stringify(after)`.

### T3 — exits non-zero when enum field value is invalid

- **Verifies:** AC3
- **Precondition:** Temp fixture with feature `test-feature-cdg3` and story `test-story-cdg3`. Valid `prStatus` enum values are `none`, `draft`, `open`, `merged`.
- **Action:** Execute `node bin/skills advance test-feature-cdg3 test-story-cdg3 prStatus=invalid-value`.
- **Expected result:** Exit code is non-zero (not 0). `stderr` contains "invalid-value" and lists the allowed values (or at minimum names the field and the rejected value). `pipeline-state.json` is not modified.
- **Edge case:** Yes — verify file not written even though the feature/story path is valid.

### T4a — exits 8 when invoked with zero positional arguments

- **Verifies:** AC4
- **Precondition:** No temp fixture needed.
- **Action:** Execute `node bin/skills advance` (no arguments).
- **Expected result:** Exit code is 8. `stderr` contains usage instructions (at minimum names the required arguments: feature-slug, story-id, field=value).
- **Edge case:** No.

### T4b — exits 8 when invoked with feature-slug only (story-id and field missing)

- **Verifies:** AC4
- **Precondition:** No temp fixture needed.
- **Action:** Execute `node bin/skills advance test-feature-cdg3`.
- **Expected result:** Exit code is 8. `stderr` contains usage instructions.
- **Edge case:** No.

### T5 — exits 8 on malformed field argument (no equals sign)

- **Verifies:** AC5
- **Precondition:** Temp fixture with feature/story.
- **Action:** Execute `node bin/skills advance test-feature-cdg3 test-story-cdg3 invalidarg`.
- **Expected result:** Exit code is 8. `stderr` names "invalidarg" (or equivalent "expected `field=value` format"). `pipeline-state.json` is not modified.
- **Edge case:** Yes — the feature and story are valid; only the field arg is malformed. Confirm the error is attributed to the malformed arg, not "unknown feature".

### T6 — writes multiple fields atomically in a single pass

- **Verifies:** AC6
- **Precondition:** Temp fixture with feature/story where both `dorStatus` and `reviewStatus` are not set to the target values.
- **Action:** Execute `node bin/skills advance test-feature-cdg3 test-story-cdg3 dorStatus=signed-off reviewStatus=passed`.
- **Expected result:** Exit code is 0. `pipeline-state.json` has `dorStatus: "signed-off"` AND `reviewStatus: "passed"` for the target story. Only one write occurred (check by verifying the file is valid JSON and both fields are present — not two sequential partial writes).
- **Edge case:** No — single-write atomicity is satisfied if both fields are present after one command invocation.

### T7 — existing validate routing is unaffected by advance subcommand addition

- **Verifies:** AC8
- **Precondition:** A valid artefact fixture exists at a path that `bin/skills validate` can check (can reuse an existing fixture from the check-cli-outer-loop.js test suite).
- **Action:** Execute `node bin/skills validate <fixture-path>` after the `advance` subcommand has been added.
- **Expected result:** Exit code is 0 (passes, as before). The output format is unchanged from the pre-advance state. No regression in the validate routing path.
- **Edge case:** No — regression guard only.

## Integration Tests

### IT1 — end-to-end round-trip: bin/skills advance → pipelineStateWriterFactory → real file write

- **Verifies:** AC1, AC6
- **Components involved:** `bin/skills` CLI entry point → advance subcommand handler → `pipelineStateWriterFactory(repoRoot)` → temp `pipeline-state.json` file
- **Precondition:** Temp `pipeline-state.json` fixture with a known feature/story. `repoRoot` set to the workspace root (or temp directory containing the fixture).
- **Action:** Execute `node bin/skills advance test-feature-cdg3 test-story-cdg3 dorStatus=signed-off reviewStatus=passed` end-to-end without any stubs — uses the real `pipelineStateWriterFactory`.
- **Expected result:** `pipeline-state.json` on disk contains both fields updated. Exit code 0. The file is valid JSON (parseable). No intermediate partially-written state exists (atomic write via rename).
