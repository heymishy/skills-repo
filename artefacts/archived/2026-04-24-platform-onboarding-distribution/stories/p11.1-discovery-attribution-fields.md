## Story: Add required attribution sections to `/discovery` template and SKILL.md

**Epic reference:** `artefacts/2026-04-24-platform-onboarding-distribution/epics/p11-governance-attribution.md`
**Discovery reference:** `artefacts/2026-04-24-platform-onboarding-distribution/discovery.md`
**Benefit-metric reference:** `artefacts/2026-04-24-platform-onboarding-distribution/benefit-metric.md`

## User Story

As a **platform maintainer**,
I want the `/discovery` template to include required `contributors`, `reviewers`, and `approved-by` sections, and the `/discovery` SKILL.md to prompt the operator to populate these before marking a discovery artefact as Approved,
So that every discovery artefact produced after this change has named attribution, making M3 (non-engineering attribution rate) measurable and making the `approved-by` field available for H-GOV evaluation (p11.3).

## Benefit Linkage

**Metric moved:** M3 ‚Äî Non-engineering outer loop attribution rate
**How:** Without the `approved-by` field in the discovery template, non-engineering attribution cannot be required, measured, or enforced. This story adds the structural field to the template and updates the `/discovery` SKILL.md to prompt the operator to populate it ‚Äî making the attribution model concrete rather than advisory.

## Architecture Constraints

- ADR-011 (artefact-first rule): this story is the required artefact chain for the change to `.github/skills/discovery/SKILL.md` and `.github/templates/discovery.md`. Both must be changed in the same PR as this story's test and DoR artefacts.
- SKILL.md and template files are Markdown only ‚Äî no embedded HTML except HTML comments for instructions. No trailing whitespace. Unix line endings.
- The existing discovery contract in `check-skill-contracts.js` must continue to pass after this change. If the contract requires a marker that references an `approved-by` step, add that marker.
- `check-pipeline-artefact-paths.js`: no new output paths are introduced by this change ‚Äî no update to PIPELINE_PATHS needed.

## Dependencies

- **Upstream:** None ‚Äî this is the first story in the feature's governance epic.
- **Downstream:** p11.3 (H-GOV hard block) depends on the `approved-by` field format being defined here. p11.3 must not be started until this story is DoD-complete.

## Acceptance Criteria

**AC1:** Given the updated `.github/templates/discovery.md`, when a new discovery artefact is created from the template, then it contains three new sections: `## Contributors` (list of people who contributed input), `## Reviewers` (list of people who reviewed the draft), and `## Approved By` (named person(s) with role and date), each with instructional HTML comments explaining expected content and the distinction between engineering and non-engineering roles.

**AC2:** Given the updated `.github/skills/discovery/SKILL.md`, when the skill reaches its completion step (the step that sets `Status: Approved`), then the output explicitly instructs the operator to populate `contributors`, `reviewers`, and `approved-by` before marking the artefact Approved ‚Äî and states that leaving `approved-by` empty will result in a H-GOV hard block at DoR.

**AC3:** Given the existing `check-skill-contracts.js` discovery skill contract, when `npm test` is run after this change, then the discovery skill contract passes with no new failures. If the contract does not include a required marker for the `approved-by` completion prompt, add a marker string (e.g. `<!-- approved-by-required -->`) to the SKILL.md and add it to the contract in `check-skill-contracts.js` in the same commit.

**AC4:** Given a discovery artefact produced before this change (without the new sections), when it is opened alongside the updated template, then the template update does not invalidate or break any existing artefact ‚Äî the sections are additive, not structural replacements.

## Out of Scope

- H-GOV hard block enforcement at DoR ‚Äî that is p11.3 and must not be implemented in this story.
- Changes to the `/benefit-metric` template ‚Äî that is p11.2.
- Changes to any skill other than `/discovery` SKILL.md and `.github/templates/discovery.md`.
- Enforcement of reviewer attribution (confirming reviewers actually reviewed) ‚Äî the MVP is structural presence, not verification.
- Backfilling `approved-by` in existing artefacts ‚Äî historical artefacts are unchanged.

## NFRs
