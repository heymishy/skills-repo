## Unit Tests

### T1 — `pin()` creates lockfile at `.github/skills/skill-lockfile.json`

- **Verifies:** AC1
- **Precondition:** Temp dir with `.github/skills/skill-a/SKILL.md` (non-empty content)
- **Action:** Call `pin(tempRoot)`; assert lockfile path `<tempRoot>/.github/skills/skill-lockfile.json` exists on disk
- **Expected result:** File exists
- **Edge case:** No

### T2 — Lockfile has correct top-level schema fields

- **Verifies:** AC1
- **Precondition:** T1 setup
- **Action:** Read the lockfile; parse JSON; assert `schemaVersion === "1.0.0"`; assert `pinnedAt` matches ISO-8601 format; assert `skills` is an Array
- **Expected result:** All three assertions pass
- **Edge case:** No

### T3 — Each entry in `skills` array has `skill`, `path`, and `sha256` fields

- **Verifies:** AC1 (schema), AC5 (sha256 forward-compat)
- **Precondition:** Temp dir with 2 stub SKILL.md files; `pin()` called
- **Action:** Parse lockfile; iterate `skills` array; for each entry assert it has `skill` (string), `path` (string), `sha256` (string matching /^[a-f0-9]{64}$/)
- **Expected result:** All entries pass validation — FAILS if any field missing or sha256 not a 64-char hex string
- **Edge case:** No

### T4 — `verify()` returns `{ status: 'pass', drifted: [], checked: n }` when no changes made

- **Verifies:** AC2
- **Precondition:** Temp dir with 2 SKILL.md files; `pin(tempRoot)` called; no files modified
- **Action:** Call `verify(tempRoot)`; assert `result.status === 'pass'`; assert `result.drifted` is empty array; assert `result.checked === 2`
- **Expected result:** All assertions pass
- **Edge case:** No

### T5 — `verify()` `checked` count matches number of SKILL.md files

- **Verifies:** AC2 (correctness of checked count)
- **Precondition:** Temp dir with exactly 3 SKILL.md files; `pin()` called
- **Action:** Call `verify(tempRoot)`; assert `result.checked === 3`
- **Expected result:** Assertion passes
- **Edge case:** Validates recursive discovery finds all skills

### T6 — `verify()` returns `{ status: 'fail' }` after one file is modified

- **Verifies:** AC3
- **Precondition:** Temp dir with 2 SKILL.md files; `pin(tempRoot)` called; one SKILL.md content changed
- **Action:** Call `verify(tempRoot)`; assert `result.status === 'fail'`
- **Expected result:** Assertion passes
- **Edge case:** No

### T7 — `verify()` `drifted` array contains the modified file entry

- **Verifies:** AC3
- **Precondition:** T6 setup
- **Action:** Assert `result.drifted.length === 1`; assert the entry has `skill`, `path`, `expected` (original hash), `actual` (new hash) fields; assert `expected !== actual`
- **Expected result:** All assertions pass — FAILS if drifted is empty or entry lacks required fields
- **Edge case:** No

### T8 — `verify()` `drifted` entry includes `expected` (pre-modification sha256) and `actual` (post-modification sha256)

- **Verifies:** AC3 (field completeness)
- **Precondition:** T6 setup — known original content and modified content
- **Action:** Compute expected SHA256 of original content; compute expected SHA256 of modified content; assert `drifted[0].expected` matches original; assert `drifted[0].actual` matches modified
- **Expected result:** Both assertions pass — confirms hashes are computed correctly not just present
- **Edge case:** No

### T9 — `pin()` is idempotent: second call with same files produces identical sha256 values

- **Verifies:** AC4
- **Precondition:** Temp dir with 2 SKILL.md files; `pin(tempRoot)` called; read lockfile 1; `pin(tempRoot)` called again; read lockfile 2
- **Action:** For each skill entry, assert `lockfile1.skills[i].sha256 === lockfile2.skills[i].sha256`; assert `lockfile1.skills[i].path === lockfile2.skills[i].path`
- **Expected result:** All assertions pass — only `pinnedAt` timestamp may differ
- **Edge case:** No

### T10 — `docs/skill-lockfile.schema.json` exists in the repo

- **Verifies:** AC5 (schema file existence)
- **Precondition:** Implementation merged
- **Action:** Assert `fs.existsSync(path.join(repoRoot, 'docs', 'skill-lockfile.schema.json'))` returns true
- **Expected result:** Assertion passes — FAILS until schema file is created
- **Edge case:** No

### T11 — Schema JSON is valid and `skills` items contain `sha256` in their definition

- **Verifies:** AC5 (schema structure)
- **Precondition:** `docs/skill-lockfile.schema.json` exists
- **Action:** Read and parse the schema; navigate to the `skills` array items definition; assert the properties object contains a `sha256` key
- **Expected result:** Assertion passes
- **Edge case:** No — forward compatibility hook; schema must remain parseable JSON

## NFR Tests

### NFR1 — `pin()` completes in under 5 seconds for 50 SKILL.md files

- **Verifies:** Performance NFR (< 5s for 50 skills)
- **Precondition:** Temp dir with 50 SKILL.md stub files (each ~500 bytes)
- **Action:** Record `Date.now()` before call; call `pin(tempRoot)`; record `Date.now()` after; assert elapsed < 5000ms
- **Expected result:** Assertion passes
- **Risk if fails:** 🟡 Medium — large skill sets hit degradation

### NFR2 — Lockfile does not contain credentials or secrets

- **Verifies:** Security NFR (no credentials in lockfile)
- **Precondition:** Temp dir SKILL.md files contain a substring `Bearer sk-faketoken123` to simulate a credential that might accidentally be included
- **Action:** `pin(tempRoot)`; read lockfile content; assert the string `sk-faketoken123` does NOT appear in the lockfile
- **Expected result:** Credential is not in the lockfile — `pin()` only stores sha256 hashes and file paths, never file content
- **Edge case:** This is a key security invariant — the hash of content is stored, not the content itself
