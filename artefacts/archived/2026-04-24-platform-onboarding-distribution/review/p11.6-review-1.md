## LOW findings — note for retrospective

- **1-L1** [C — AC quality] — AC2 states "when `/start` is invoked on a repo where `workspace/state.json` exists and records an in-progress pipeline phase, then the output names the current pipeline phase, identifies what skill was last run, and names the next skill to run." The phrase "identifies what skill was last run" is ambiguous: `workspace/state.json` stores `currentPhase` and a `resumeInstruction`, not a "last run skill" field. A test writer would need to infer what "last run skill" means from the state file structure. Consider revising to "reads and surfaces the `currentPhase` value and the `resumeInstruction` from `workspace/state.json`" to match the actual data fields.
