# DoR Contract: Create `/start` SKILL.md with greenfield orientation path

**Story:** `artefacts/2026-04-24-platform-onboarding-distribution/stories/p11.6-start-skill.md`
**Date approved:** 2026-04-28

---

## What will be built

A new `.github/skills/start/SKILL.md` will be created with YAML frontmatter declaring triggers (`"start"`, `"where am I"`, `"what do I do"`, `"help me get started"`, `"I just cloned this"`). The skill will provide a greenfield orientation path: it detects that no prior pipeline state exists, presents a concise menu (what the platform does; how to start a feature; first action), optionally shows the last-run skill if `workspace/state.json` is present, and confirms skill availability. The entire orientation completes within a single AI turn. A `<!-- single-turn-constraint -->` marker will be added to the SKILL.md. The `start` skill will be added to `check-skill-contracts.js` with `single-turn-constraint` as a required string.

## What will NOT be built

- Brownfield routing (Entry A: in-flight stories; Entry B: existing src/ codebase; Entry C: DoD artefacts with retrospective template) — this is p11.7
- Any JavaScript implementation or file-reading logic — the skill is pure instruction text consumed by an AI agent
- Modification of any existing SKILL.md file

## Assumptions

1. The SKILL.md instruction text instructs the AI agent; there is no JavaScript file-reading logic to implement.
2. "Last skill run" (AC2) is obtained by the AI agent reading `workspace/state.json` → `currentPhase` field. If the file does not exist, the agent skips that element.
3. The greenfield detection check (no `pipeline-state.json` AND no `workspace/state.json`) is described in the SKILL.md and executed by the AI agent — it is not validated by the test at runtime (tests assert file content only).
4. This story creates the SKILL.md file from scratch; p11.7 will extend it. The coding agent must not include brownfield routing content preemptively.
