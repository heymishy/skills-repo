## Setup

**Before you start:**
1. Ensure the implementation PR for p11.3 has been merged (or the branch is checked out).
2. Open `.github/skills/definition-of-ready/SKILL.md` in your editor.
3. Have a discovery artefact nearby for the live-agent scenarios (Scenarios 2–5 are manual live-agent tests).

**Reset between scenarios:** Scenarios 1 and 6 are file-inspection only. Scenarios 2–5 are live-agent invocations — no state is modified, but run each in a fresh chat if you want clean output.

### Scenario 1: DoR SKILL.md contains all required H-GOV instruction text

**Covers:** AC1–AC4 (instruction text presence), AC5 (contract marker)

**Steps:**
1. Open `.github/skills/definition-of-ready/SKILL.md`.
2. Search for `H-GOV` to locate the new block.
3. Read the entire H-GOV block carefully. Confirm the following are present:
   a. A pass condition that describes a non-engineering approver causing H-GOV to pass.
   b. A fail condition for **empty** `## Approved By` section — with (i) "H-GOV" named explicitly, (ii) what is missing described, (iii) a resolution step given.
   c. A fail condition for **absent** `## Approved By` section — references the updated discovery template.
   d. A fail condition for **engineer-only** entries — phrased distinctly from empty/absent.
4. Search for `h-gov-block` to confirm the contract marker is present.

**Expected outcome:**
> All four conditions (a–d) are present and clearly described in the H-GOV block. The contract marker `h-gov-block` (typically as `<!-- h-gov-block -->`) is present in the file.

**Result:** [ ] Pass  [ ] Fail
