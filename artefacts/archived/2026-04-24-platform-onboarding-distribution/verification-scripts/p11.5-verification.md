### Scenario 1: `init()` on a fresh directory creates the skills folder and stub lockfile

**Covers:** AC1

**Steps:**
1. In a terminal, start a Node.js REPL: `node`
2. Run the following:
   ```js
   const os = require('os'), fs = require('fs'), path = require('path');
   const { init } = require('./src/enforcement/cli-adapter.js');
   const tmp = fs.mkdtempSync(path.join(os.tmpdir(), 'p11-init-'));
   const result = init(tmp);
   console.log('result:', JSON.stringify(result));
   console.log('skills dir exists:', fs.existsSync(path.join(tmp, '.github', 'skills')));
   const lf = JSON.parse(fs.readFileSync(path.join(tmp, '.github', 'skills', 'skill-lockfile.json'), 'utf8'));
   console.log('lockfile:', JSON.stringify(lf));
   ```

**Expected outcome:**
> `result: {"status":"ok","command":"init","created":true}`. `skills dir exists: true`. Lockfile printed with `schemaVersion: "1.0.0"`, a `pinnedAt` timestamp, and `skills: []` (empty array).

**Covers:** AC4

   const tmp = fs.mkdtempSync(path.join(os.tmpdir(), 'p11-fetch2-'));

With a fake remote: git throws (acceptable); cli-adapter correctly invoked execSync and the remote URL was passed — visible in the error message `git fetch https://github.com/heymishy/...`.
