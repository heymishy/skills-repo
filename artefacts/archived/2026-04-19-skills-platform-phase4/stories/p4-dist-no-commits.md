## Story: Install generates zero commits in consumer repo — CI-verifiable assertion

**Epic reference:** artefacts/2026-04-19-skills-platform-phase4/epics/e2-distribution-model.md
**Discovery reference:** artefacts/2026-04-19-skills-platform-phase4/discovery.md
**Benefit-metric reference:** artefacts/2026-04-19-skills-platform-phase4/benefit-metric.md

## User Story

As a **consumer running skills-repo commands in CI (Craig, Thomas, or any adopter)**,
I want to **have a CI-verifiable assertion that confirms zero commits were generated by any distribution command**,
So that **the zero-commit install property is enforceable in CI and I have an objective, automated signal to include in M1 measurement evidence rather than relying on manual review**.

## Benefit Linkage

**Metric moved:** M1 — Distribution sync
**How:** M1's first target (100% zero-commit install) requires more than the install behaving correctly in development; it requires an automated assertion that catches regressions in CI. Without this story, a future change to `init`, `fetch`, `pin`, or `verify` could accidentally generate a commit and M1's measurement evidence would fail — but no gate would stop it. This story makes M1's zero-commit property machine-verifiable.

## Architecture Constraints

- C1: the zero-commit property is load-bearing for non-fork distribution — any CI assertion failure must block the consumer's CI run; this is not a warning-only check
- MC-CORRECT-02: the CI assertion uses schema-validated command outputs (not screen-scraping of human-readable text); commit count comparison uses `git rev-list --count HEAD` before and after the command
- ADR-004: the commands tested by the assertion are the same commands driven by context.yml — the assertion does not introduce a separate command list; it wraps the existing distribution commands

## Dependencies

- **Upstream:** p4.dist-install must be complete (assertion wraps the install command; no install story means no assertion to validate)
- **Downstream:** None — this story hardens an existing property; no other story is blocked on it

## Acceptance Criteria

**AC1:** Given a fresh consumer repository with no sidecar installed, When `skills-repo init` runs end-to-end in CI, Then a CI assertion fires that compares `git rev-list --count HEAD` before and after `init` — if the count increases, the CI run fails with an error message: "Distribution command generated unexpected commit(s): <count> commit(s) added".

**AC2:** Given the CI assertion is in place and a distribution command (`init`, `fetch`, `pin`, or `verify`) is added or modified, When the consumer's CI suite runs, Then the assertion covers all four commands — each command has its own before/after commit count check registered in the test suite; adding a new distribution command without a corresponding assertion fails the `check-archive.js` or equivalent governance check.

**AC3:** Given the consumer runs `skills-repo verify` against a correct lockfile, When verify completes, Then the assertion confirms that verify is read-only with respect to git state: zero commits generated, zero staged files, zero unstaged modifications to tracked files.

## Out of Scope

- Validating commit format for non-distribution commands — that is p4.dist-commit-format; this assertion checks distribution-command git hygiene only
- Alerting on sidecar directory modification — the sidecar is in `.gitignore`; modifications to it are not git state changes
- Generating a remediation commit when the assertion fails — the assertion is a gate, not a fixer

## NFRs

- **Security:** Assertion output must not leak sidecar contents or lockfile hashes to CI log output (MC-SEC-02)
- **Correctness:** The `git rev-list --count HEAD` baseline must be captured immediately before the command and compared immediately after — no intervening commands between capture and comparison
- **Performance:** Assertion adds at most 2 seconds to CI suite wall time

## Complexity Rating

**Rating:** 1
**Scope stability:** Stable — the assertion pattern is straightforward once p4.dist-install defines the install command
