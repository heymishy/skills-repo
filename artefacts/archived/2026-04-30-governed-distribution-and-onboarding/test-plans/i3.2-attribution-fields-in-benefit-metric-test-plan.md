## Integration Tests

### `benefit-metric-attribution-step-is-first-step-block` — Attribution check appears as the first numbered or structured step in SKILL.md

- **Verifies:** AC1
- **Precondition:** SKILL.md has a structured step list (Step 1, Step 2, etc.)
- **Action:** Read SKILL.md; assert that "Step 1" (or the first item in the step/phase sequence) contains the attribution check — not a later step
- **Expected result:** Attribution check is Step 1 (or equivalent first position)
- **Edge case:** No

### `benefit-metric-warning-tied-to-empty-and-absent-condition` — Warning instruction is triggered by BOTH empty-field and absent-section conditions

- **Verifies:** AC2, AC4
- **Precondition:** SKILL.md contains warning instruction
- **Action:** Read SKILL.md; assert the condition triggering the warning covers both: (a) field present but empty/placeholder, and (b) attribution section absent entirely
- **Expected result:** Both conditions trigger the same warning — neither is a silent pass
- **Edge case:** Yes — two distinct absence modes

### `benefit-metric-note-tied-to-option-b-path` — "Attribution incomplete" note instruction is explicitly conditional on option (b) being chosen

- **Verifies:** AC5
- **Precondition:** SKILL.md has option (b) instruction
- **Action:** Read SKILL.md; assert the note text instruction is positioned within or directly after the option (b) path description — not written as an unconditional instruction that fires for all runs
- **Expected result:** Note only written when option (b) is chosen — not on every benefit-metric run
- **Edge case:** Yes — unconditional note would be a regression

### `benefit-metric-absent-section-equivalent-to-empty` — Absent section and empty field produce identical warning behaviour per SKILL.md instructions

- **Verifies:** AC4
- **Precondition:** SKILL.md has absent-section handling instruction
- **Action:** Read SKILL.md; confirm the instruction treats absent section as "equivalent to empty" — using the same warning message template with no separate branch for "section missing"
- **Expected result:** One warning template handles both cases; no silent divergence
- **Edge case:** No
