## Unit Tests

### orient-entry-b-routing-block-present

- **Verifies:** AC1 prerequisite — Entry B routing logic exists in SKILL.md
- **Precondition:** `.github/skills/orient/SKILL.md` exists
- **Action:** Read file; check for "Entry B" string
- **Expected result:** "Entry B" appears in the file
- **Edge case:** No

### orient-entry-b-names-state-code-first-brownfield

- **Verifies:** AC1(a) — output names state "Entry B — code-first brownfield"
- **Precondition:** SKILL.md exists
- **Action:** Read file; check for "Entry B — code-first brownfield" (or equivalent canonical label)
- **Expected result:** Entry B state is named explicitly
- **Edge case:** No

### orient-entry-b-explains-reverse-engineer

- **Verifies:** AC1(b) — explains that /reverse-engineer extracts existing system's business rules and design
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry B section references `/reverse-engineer` and extraction of business rules or design
- **Expected result:** `/reverse-engineer` named with a description of what it produces
- **Edge case:** No

### orient-entry-b-shows-full-path

- **Verifies:** AC1(c) — lays out full suggested path: /reverse-engineer → /discovery → /benefit-metric → /definition
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry B section contains all four skill names in sequence
- **Expected result:** `/reverse-engineer`, `/discovery`, `/benefit-metric`, and `/definition` all appear in the Entry B routing section
- **Edge case:** No

### orient-entry-b-names-detection-signal

- **Verifies:** AC2 — routing output names specific detection signal (src/ or app/ with application files found; no artefacts/ folder)
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry B section instructs naming the found code directories (e.g. "Found: src/ directory with application files")
- **Expected result:** Entry B routing section includes instruction to state which code directories were found
- **Edge case:** No

### orient-entry-b-signal-allows-operator-correction

- **Verifies:** AC2 — operator can verify and correct the classification
- **Precondition:** SKILL.md exists
- **Action:** Read file; check Entry B uses hedged language ("This looks like Entry B") not an assertion
- **Expected result:** Classification is presented as tentative — "looks like" or equivalent
- **Edge case:** No

### orient-entry-b-entry-a-takes-precedence

- **Verifies:** AC3 — when story artefacts are present, Entry A is selected over Entry B
- **Precondition:** SKILL.md exists
- **Action:** Read file; check routing priority rules — Entry A fires before Entry B when story artefacts are present; Entry B requires absence of pipeline artefacts of any kind
- **Expected result:** Entry B detection in SKILL.md is conditioned on "no pipeline artefacts" — stories present → Entry A, not Entry B
- **Edge case:** Yes — the precedence ordering is a correctness requirement

### orient-entry-b-nfr-config-files-not-app-logic

- **Verifies:** NFR — config-only files (.yml, .json, Dockerfile, Makefile) do NOT trigger Entry B
- **Precondition:** SKILL.md exists
- **Action:** Read file; check that Entry B detection criteria specifies application logic file extensions (.js, .ts, .py, .java, etc.) and not just any files in a code directory
- **Expected result:** Entry B detection criteria explicitly lists or describes application-logic file extensions; config file types are either excluded or the specification is precise enough that config-only repos would not match
- **Edge case:** Yes — this is the primary false-positive risk for Entry B

## Integration Tests

### orient-entry-b-entry-a-priority-order-enforced

- **Verifies:** AC3 integration — routing priority: Entry A before Entry B
- **Components involved:** orient SKILL.md (Entry A section from i2.1 + Entry B section from i2.2)
- **Precondition:** SKILL.md exists with both Entry A and Entry B routing logic
- **Action:** Read file; verify that the routing order or conditions make Entry A and Entry B mutually exclusive — Entry B must require absence of story artefacts
- **Expected result:** Entry A and Entry B routing conditions are explicitly exclusive; Entry B condition includes "no story artefacts" or equivalent
