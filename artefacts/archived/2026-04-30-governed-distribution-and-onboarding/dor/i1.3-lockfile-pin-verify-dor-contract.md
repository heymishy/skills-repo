## What will be built

1. `scripts/platform-pin.js` — Node.js CLI script (built-ins only):
   - Finds all skill files under `.github/skills/` recursively
   - Computes SHA-256 hash of each file's raw content using `crypto.createHash('sha256')`
   - Writes `platform-lock.json` in V1 schema: `{ "version": "1", "pinnedAt": "<ISO-8601>", "platform": "<git-sha>", "skills": { "<relative-path-forward-slash>": "<sha256-hex>" } }`
   - Paths sorted alphabetically; forward slashes regardless of OS

2. `scripts/platform-verify.js` — Node.js CLI script (built-ins only):
   - If no `platform-lock.json`: exits non-zero with "No platform-lock.json found. Run npm run platform:pin first."
   - If all files match pinned hashes: exits 0 with "All N skill files match pinned version [platform-id]. ✓"
   - If any files drifted: exits non-zero with three-part error per drifted file: (a) relative path, (b) expected vs actual SHA-256, (c) "Run npm run platform:pin or npm run platform:fetch"

3. `package.json` — adds two entries to `scripts`:
   - `"platform:pin": "node scripts/platform-pin.js"`
   - `"platform:verify": "node scripts/platform-verify.js"`

## platform-lock.json V1 schema (stable — no breaking changes in this story)

```json
{
  "version": "1",
  "pinnedAt": "<ISO-8601 timestamp>",
  "platform": "<git-sha-or-semver>",

  }
}
```

## Assumptions

- Skill files to hash are under `.github/skills/` (relative paths from repo root)
- `platform` field derived from `git rev-parse HEAD` via `child_process.execSync`
- `platform-lock.json` stored at repo root (same level as `package.json`)
- `platform-lock.json` is committed to VCS — design for diffability (sorted keys, consistent indent)
