# AC Verification Script: AGENTS.md adapter in assembly script (p2.4)

**Story reference:** artefacts/2026-04-11-skills-platform-phase2/stories/p2.4-agents-md-adapter.md
**Date:** 2026-04-11

This script is used by a human reviewer before coding begins (to confirm expected behaviour is understood) and as a smoke-test checklist after merge.

---

## Overview

The story adds VCS-type-aware output path selection to the assembly script. The script now checks `vcs.type` (and an optional `vcs.agent_instructions.format` override) and writes the assembled instructions to either `.github/copilot-instructions.md` (GitHub default) or `AGENTS.md` at the repo root (non-GitHub or explicit override).

---

## Scenarios

### Scenario 1 — GitHub default: output written to .github/copilot-instructions.md

**AC:** AC1

**Steps:**
1. Create or use a test context.yml with `vcs.type: github` (or no `vcs.type` key — see Scenario 5).
2. Run the assembly script against that context.
3. Check the output directory.

**Expected result:** The file `.github/copilot-instructions.md` is present and contains the assembled skill instructions. No `AGENTS.md` file exists at the repo root.

**Pass/Fail:** Pass if `.github/copilot-instructions.md` exists and `AGENTS.md` is absent.

---

### Scenario 2 — Non-GitHub vcs.type: output written to AGENTS.md at repo root

**AC:** AC2

**Steps:**
1. Create or use a test context.yml with `vcs.type: bitbucket` (or `gitlab`, or `azure-devops`).
2. Run the assembly script against that context.
3. Check the output directory.

**Expected result:** The file `AGENTS.md` is present at the repo root and contains the assembled skill instructions. No `.github/copilot-instructions.md` is created.

**Pass/Fail:** Pass if `AGENTS.md` exists and `.github/copilot-instructions.md` is absent.

---

### Scenario 3 — Explicit override: vcs.agent_instructions.format: agents-md forces AGENTS.md even for GitHub

**AC:** AC3

**Steps:**
1. Create a context.yml with `vcs.type: github` AND `vcs.agent_instructions.format: agents-md`.
2. Run the assembly script.
3. Check the output directory.

**Expected result:** `AGENTS.md` is written at the repo root. `.github/copilot-instructions.md` is NOT created. The override key takes precedence over the default.

**Pass/Fail:** Pass if `AGENTS.md` exists and `.github/copilot-instructions.md` is absent.

---

### Scenario 4 — Invalid format value: assembly exits with a clear error

**AC:** AC3 (error branch)

**Steps:**
1. Create a context.yml with `vcs.agent_instructions.format: openai-project` (an unsupported value).
2. Run the assembly script.
3. Check exit code and stderr.

**Expected result:** The script exits with a non-zero code. The error message identifies `openai-project` as an unsupported format value. No output file is written.

**Pass/Fail:** Pass if exit code ≠ 0, error message references the invalid value, and no output file is created.

---

### Scenario 5 — Unconfigured context defaults to .github/copilot-instructions.md

**AC:** AC5

**Steps:**
1. Create a context.yml with no `vcs.type` key and no `vcs.agent_instructions.format` key.
2. Run the assembly script.
3. Check the output directory.

**Expected result:** The file `.github/copilot-instructions.md` is present. `AGENTS.md` is absent. No error is raised.

**Pass/Fail:** Pass if `.github/copilot-instructions.md` exists, `AGENTS.md` is absent, and exit code = 0.

---

### Scenario 6 — npm test passes: check-assembly.js covers both output paths

**AC:** AC6

**Steps:**
1. Ensure the implementation for both paths has been merged.
2. Run `npm test`.
3. Review the `[check-assembly]` output section.

**Expected result:** Both the GitHub-path test case and the non-GitHub-path test case pass. The full npm test suite exits 0.

**Pass/Fail:** Pass if `[check-assembly]` shows both path cases green and total exit code = 0.
