## Contract Proposal

### What will be built

- **Assembly script format branch** in `scripts/assemble-copilot-instructions.sh`: Read `vcs.agent_instructions.format` from `context.yml` (nested under `vcs:` block). If value is `agents-md`, emit `AGENTS.md` at repo root. If value is `copilot-instructions` (or absent/default), emit `.github/copilot-instructions.md`. Explicit format declaration takes precedence over `vcs.type` inference. Unknown values are assembly-time errors.
- **`vcs.type` fallback logic**: If `vcs.agent_instructions.format` is not set, infer from `vcs.type`: non-github values → `AGENTS.md`; github (or absent) → `.github/copilot-instructions.md`.
- **Tests in `check-assembly.js`**: At minimum one test for GitHub path (`.github/copilot-instructions.md` output) and one for non-GitHub path (`AGENTS.md` output). Both pass on clean checkout.
- **Context template schema documentation**: `contexts/` template files (or a schema YAML comment) document `vcs.agent_instructions.format` as a field with permitted values `copilot-instructions` and `agents-md` (MC-CONSIST-01).
- **`stdout` log of output path** during assembly for CI audit trail (AC6 auditability NFR).

### What will NOT be built

- Content transformation by VCS type (content is format-agnostic, byte-for-byte identical — AC4)
- AGENTS.md conflict resolution for stale files from a prior config
- Bitbucket Pipelines CI adapter (p2.10)
- Multiple simultaneous output paths (single path per config)
