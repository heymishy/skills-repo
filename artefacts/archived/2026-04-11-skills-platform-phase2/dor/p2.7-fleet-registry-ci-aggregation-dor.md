## Contract Proposal

### What will be built

- **Per-squad fleet registry file format and schema validation** (`fleet/squads/{id}.json`): Required fields schema (`squadId`, `repoUrl`, `pipelineStateUrl`, `registeredAt`) committed to `pipeline-state.schema.json` or a co-located fleet schema file before any CI job reads or writes fleet files (ADR-003). A pre-commit or CI validation script rejects squad files missing required fields with a descriptive error naming each missing field.
- **CI aggregation job** (`.github/workflows/fleet-aggregation.yml` or equivalent CI script): Reads all `fleet/squads/{id}.json` files where the squad's `context.yml` has `registry_mode: publishing`; fetches or reads each referenced `pipelineStateUrl`; writes the combined output to `fleet-state.json` as a `squads[]` array — one entry per squad included. Job continues on any unreachable `pipelineStateUrl` (graceful degradation).
- **fleet-state.json structure**: Each squad entry contains exactly: `squadId` (string), `stage` (string from squad's pipeline-state.json), `health` (green/amber/red/unknown), `updatedAt` (ISO datetime of aggregation run), `sourceUrl` (the `pipelineStateUrl` that was fetched). Schema defined before first write.
- **registry_mode filtering**: Squads with `registry_mode: publishing` included; `none` or absent excluded. Config read from `context.yml` only (ADR-004).
- **Fleet viz panel in `pipeline-viz.html`**: New fleet panel section renders ≥2 squad cards from `fleet-state.json` — each card shows `squadId`, `stage`, `health` (colour PLUS text label or icon — MC-A11Y-02), `updatedAt`. All interactive elements keyboard-accessible (MC-A11Y-01). File remains self-contained with no external network calls (MC-SELF-01/02, MC-SEC-03).
- **Graceful degradation**: Unreachable `pipelineStateUrl` → entry has `health: "unknown"` and `error` field; aggregation job continues to remaining squads and does not abort.

### What will NOT be built

- Cross-repo or federated aggregation — CI job reads squad files within this repository only
- Automated squad file creation at onboarding — squads create `fleet/squads/{id}.json` manually
- Real-time fleet state (sub-minute refresh) — fleet-state.json updated on CI trigger only
- Historical fleet state trending or archiving — each CI run overwrites fleet-state.json
- Authentication-gated `pipelineStateUrl` retrieval — private repo URLs are a graceful degradation scenario (AC6)
