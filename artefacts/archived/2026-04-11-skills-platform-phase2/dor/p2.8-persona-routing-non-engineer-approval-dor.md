## Contract Proposal

### What will be built

- **Channel hint routing function**: Reads `channel_hints.approval` from `context.yml` and emits a notification to the configured external tool containing story slug, squad ID, approval action description, and fleet registry reference (via `fleet/squads/{id}.json` → `pipelineStateUrl`). Phase 2 implementation supports `approval_channel: github-issue` only.
- **GitHub Issue approval adapter** (`github-issue` channel): A designated approver posts a comment containing `/approve-dor` on the linked GitHub Issue. A GitHub Actions workflow detects the comment and triggers the `pipeline-state.json` write. Channel-specific plumbing lives in an adapter module; the state write contract is channel-agnostic.
- **pipeline-state.json update on sign-off**: Writes `dorStatus: "signed-off"`, `dorApprover` (GitHub username — not full name or email), and `dorChannel: "github-issue"` to the relevant squad's `pipeline-state.json`. These three fields are schema-first (ADR-003). `dorStatus: "signed-off"` is the canonical evidence field — same field as a VS Code sign-off (ADR-002).
- **IDE channel regression guard**: Adding the approval channel does not suppress the existing VS Code IDE notification — both channels receive the notification when both are configured.
- **Unconfigured approval channel fallback**: No `channel_hints.approval` in `context.yml` → IDE-only notification + configuration warning logged to session output; no error thrown, no action blocked.

### What will NOT be built

- Full automated CI pipeline (webhook → write): Phase 2 maintainer may process sign-off manually from the interface record. Full automation is Phase 3.
- Jira, Confluence, Slack, Teams adapters — Phase 2 dogfood uses `github-issue` only. Phase 3 enterprise channels.
- Multi-approver or approval quorum workflows — single approver per story sign-off.
- Bi-directional sync between `pipeline-state.json` and the external tool.
- Personal data (full names, email addresses) stored in `dorApprover` — username/role only.
