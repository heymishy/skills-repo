## Contract Proposal

### What will be built

- **D4 — Per-story incremental write timing change** in `.github/skills/review/SKILL.md`: The /review skill writes each story's findings to `artefacts/[feature]/review/[story-slug]-review-N.md` immediately after completing that story's review — before loading the next story. The file is persisted to disk, not held in memory until session end.
- **Session recovery logic** in `.github/skills/review/SKILL.md`: At session start, the skill checks `artefacts/[feature]/review/` for existing slug-matched artefacts and skips already-reviewed stories (unless operator explicitly requests re-review). Only unreviewed stories are processed in the resumed session.
- **Write completeness guarantee**: Each per-story artefact must contain all required finding fields at minimum: story slug, review date, findings list (may be empty), severity levels, recommended actions. Partial writes are a failure condition.

### What will NOT be built

- Changes to the review finding template structure or severity classification system
- Automated session recovery detection from logs or timestamps
- Re-review triggering logic changes
- Applying the incremental write pattern to other skills
