## What will be built

| Component | File | Notes |
|-----------|------|-------|
| IaC adapter | `src/surface-adapter/adapters/iac.js` | execute("iac", context) → result; IaC findings vocabulary |
| SaaS-API adapter | `src/surface-adapter/adapters/saas-api.js` | execute("saas-api", context) → result; API findings vocabulary |
| IaC POLICY.md stub | `standards/iac/POLICY.md` | 3 mandatory sections + ≥1 IaC-specific criterion |

- SaaS-GUI, M365-admin, manual adapters (p2.5b)
- Live IaC provider or SaaS-API network calls
- SaaS-API contract test execution
- DoD criteria variant generation

## Out-of-scope signals

If the implementing agent touches `src/surface-adapter/adapters/saas-gui.js`, `m365-admin.js`, or `manual.js` (p2.5b scope), that is a scope violation — add a PR comment. If any adapter makes a network call to an external provider, that is an MC-SEC-03 violation — add a PR comment.
