## Category C — AC Quality

**F-1 [MEDIUM]:** AC1 specifies "HTTP GET + app_id" as the EA registry query mechanism, but the AC does not define: (a) the query encoding format (path segment `GET /apps/{app_id}` vs query parameter `GET /apps?id={app_id}`); (b) the expected JSON response schema beyond the `surfaceType` field (other required/optional fields, content-type header); (c) the HTTP auth mechanism — is it an unauthenticated GET, bearer token, API key, or other? ASSUMPTION-01 covers only the field name; the full HTTP contract is open. For a C3/Unstable story with an external dependency, this under-specification is material: it prevents a deterministic test plan from being written for AC1 (the test fixture response format would be guessed). At minimum, document an assumed stub of the HTTP contract in the Architecture Constraints block, even as a placeholder marked "unconfirmed pending DoR H1 validation."
