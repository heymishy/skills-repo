## Category E — Architecture Compliance

**F-1 [MEDIUM]:** AC2 introduces H8-ext (new hard block) and AC5 introduces B1-enforce (new hard block) in the DoR SKILL.md, but neither AC references an eval scenario name to guard against regression. Compare to p2.2 AC4 which explicitly names `s-review-incremental-state`. MC-CONSIST-02 requires gate additions synced with SKILL.md criteria verification — for SKILL.md instruction gates, an eval scenario is the verification mechanism. Without a named scenario for each new block, the gates may go unexercised in the automated suite. Add eval scenario names for H8-ext injection and B1-enforce injection (e.g. `s-dor-schema-depends-hard-block`, `s-dor-nfr-profile-hard-block`) to AC2 and AC5 respectively, or add a standalone AC covering both.

**F-2 [LOW]:** Architecture Constraints cite ADR-003 for the `schemaDepends` field. However, ACs AC1–AC3 describe `schemaDepends` appearing only in the DoR-contract *markdown artefact* (a file in `artefacts/`), not in `pipeline-state.json`. ADR-003 governs schema-first discipline for `pipeline-state.json` fields — it does not govern markdown artefact field names. If `schemaDepends` is never written to `pipeline-state.json`, the ADR-003 citation is inapplicable. Clarify: (a) if `schemaDepends` will be written to `pipeline-state.json`, add an AC for the schema-first step; (b) if it lives only in the markdown artefact, remove the ADR-003 citation and replace with a note that this is an artefact-level convention, not a schema contract.
