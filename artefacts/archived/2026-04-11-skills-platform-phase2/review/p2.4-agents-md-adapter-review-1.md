## Category E — Architecture Compliance

**F-1 [MEDIUM]:** ACs AC2 and AC3 reference `vcs.type` and an "explicit format override" respectively, but neither AC specifies: (a) the exact context.yml key path for the explicit format override (e.g. is it `agent_instructions.format`? `agents.output_format`? `vcs.agent_format`?); (b) the permitted values for the format field (e.g. `"copilot-instructions"`, `"agents-md"`, or some other enum). ADR-004 mandates that all configuration is read from `context.yml`, and ADR-005 mandates that the format is driven by `vcs.type` — but without the explicit key path, the check-assembly.js fixture author and the implementation author may use different key names. This risks a passing CI test against a fixture that doesn't match the format the downstream skills read. Add a concrete context.yml fragment (even as a one-line example) to AC2 or the Architecture Constraints block, specifying the key path and valid values.
