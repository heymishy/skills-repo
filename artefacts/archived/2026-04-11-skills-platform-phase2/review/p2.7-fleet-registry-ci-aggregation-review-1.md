## Category E — Architecture Compliance

**F-1 [MEDIUM]:** AC5 requires the pipeline viz to render a fleet panel with ≥2 squad cards, but the Architecture Constraints do not cite MC-A11Y-01 (interactive elements keyboard-accessible) or MC-A11Y-02 (colour not sole indicator of gate status). Fleet panel squad cards likely display health status using colour coding (green/amber/red) — if colour is the sole health indicator, this violates MC-A11Y-02. Add MC-A11Y-01 and MC-A11Y-02 to Architecture Constraints. Amend AC5 or add a companion AC specifying that each squad card's health status is conveyed by both colour and a text label or icon (per MC-A11Y-02), and that any interactive elements (expand/collapse, links) are keyboard-accessible (per MC-A11Y-01).
