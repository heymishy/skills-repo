## Manual Tests

### AC3 (Manual): SKILL.md update and proposal status update are in the SAME commit (atomicity check)

- **Verifies:** AC3 (single-commit requirement)
- **Why manual:** Git commit content verification requires human inspection; CI can verify file states but not commit atomicity
- **Pre-conditions:** Acceptance workflow run; branch has at least one new commit
- **Steps:**
  1. Run `git log --name-only -1` on the acceptance commit
  2. Confirm the commit message contains the proposal ID (e.g. `improvement: apply [proposal-id] — ...`)
  3. Confirm the diff shows BOTH the target SKILL.md change AND the proposal file status change in the single commit
  4. Confirm no second commit contains a partial change to either file
- **Expected outcome:** One commit with both file changes; commit message references proposal ID; no split commits
- **Pass/Fail:** PASS: atomicity confirmed; FAIL: SKILL.md change in one commit, proposal status in another; commit message missing proposal ID

## NFR Tests

### Human oversight: challenger-result.md reviewer field is a named human, not CI job ID

- **NFR addressed:** Human oversight
- **Measurement method:** Run the result validation test (see AC2 unit test); also: inspect improvement agent SKILL.md for any step that would accept `github-actions` or `copilot` as a valid reviewer identity
- **Pass threshold:** All result file reviewer fields validated to be non-CI identities; SKILL.md procedure documentation explicitly states reviewer must be a named human identity
- **Tool:** AC2 unit test (reviewer validation); SKILL.md grep for human identity requirement

### Traceability: every accepted SKILL.md commit message references the proposal ID

- **NFR addressed:** Traceability
- **Measurement method:** After the manual acceptance test (Scenario 4), run `git log --grep="improvement:"` on the branch; verify commit message format includes `improvement: apply [proposal-id] — ...`; verify proposal-id is resolvable to a file in `workspace/proposals/`
- **Pass threshold:** Zero accepted SKILL.md commits without traceable proposal ID; proposed format used
- **Tool:** git log grep; workspace/proposals/ file existence check

### Immutability of evidence: challenger-result.md content is not overwritten by workflow runs

- **NFR addressed:** Immutability of evidence
- **Measurement method:** After acceptance workflow completes, compute the hash of the original challenger-result.md content; verify that the file content is unchanged post-acceptance (status transitions are written to the outer proposal file, not to challenger-result.md)
- **Pass threshold:** Challenger-result.md sha256 hash identical pre- and post-acceptance; no modification timestamps changed
- **Tool:** sha256 hash check on challenger-result.md (before and after acceptance)
