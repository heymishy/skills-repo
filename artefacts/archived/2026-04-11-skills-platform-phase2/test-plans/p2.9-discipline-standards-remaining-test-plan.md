### Each of 8 new disciplines: POLICY.md file exists at declared path

### P1.7 structural pattern: new discipline files contain all required headings

- **Verifies:** AC2 (structural conformance)
- **Precondition:** All new standard files committed
- **Action:** Read each new core.md and POLICY.md; check for required heading set: `## Overview`, `## Outcomes`, `## Requirements`, `## Out of Scope for this standard`; check that at least one MUST/SHOULD/MAY statement is present and outcome-oriented
- **Expected result:** All 16 new files (8×2) contain all required headings; at least one MUST/SHOULD/MAY statement per file has outcome-oriented phrasing (describes what result is enabled or prevented)
- **Edge case:** No

### POLICY.md files: MUST/SHOULD/MAY statements are outcome-oriented, not process-only

- **Verifies:** AC2 (quality constraint)
- **Precondition:** All new POLICY.md files committed
- **Action:** Sample the first MUST statement from each POLICY.md; check that the statement describes an outcome (what is enabled or prevented), not just a process step (what to do)
- **Expected result:** Each sampled MUST statement contains a result clause, not just a process description (e.g. "MUST: [outcome] so that [benefit]" or "MUST: [claim] to prevent [failure]")
- **Edge case:** No

## NFR Tests

### Consistency: all new POLICY.md files follow identical structure to Phase 1 standards

- **NFR addressed:** Consistency
- **Measurement method:** Diff the heading structure of all 11 new files (8 disciplines × POLICY.md + 3 domain POLICY.mds) against the P1 standard POLICY.md heading structure; check for structural variation (missing or extra headings)
- **Pass threshold:** No structural variation — all files share the same heading set
- **Tool:** Automated heading check script or diff in check-standards-model.js

### Extensibility: standards/index.yml YAML syntax valid after extension

- **NFR addressed:** Extensibility (machine-readable)
- **Measurement method:** Parse `standards/index.yml` with a YAML parser; check for YAML syntax errors; verify all declared paths are strings
- **Pass threshold:** No YAML parse errors; all path values are strings; no null or improperly typed entries
- **Tool:** `yaml.parse()` or JS-YAML in check-standards-model.js

### Maintainability: no cross-references between standard files

- **NFR addressed:** Maintainability (standalone documents)
- **Measurement method:** Scan all new standard files for hyperlinks (`[...](...md)`) or `include:` directives that reference another standard file path
- **Pass threshold:** No internal cross-file references that would require reading another standard file to interpret this one; adjacent-standards annotations are permitted but must be in a clearly marked `## Adjacent standards` section
- **Tool:** Link/reference scan in check script
