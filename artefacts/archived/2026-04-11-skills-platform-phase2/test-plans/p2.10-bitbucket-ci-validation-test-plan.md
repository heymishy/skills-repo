### PCI / sensitivity constraints

Credentials (app password, OAuth consumer, SSH private key) MUST be injected via CI environment secrets. Under no circumstances committed to the repository. `docker-compose.bitbucket-dc.yml` must not contain credential literals — it must reference environment variables for all secret values.

## Unit Tests

### AC1: valid bitbucket-pipelines.yml with assurance gate step passes YAML syntax check

- **Verifies:** AC1
- **Precondition:** Well-formed `bitbucket-pipelines.yml` fixture present
- **Action:** Pass fixture content to the YAML syntax validator function; capture result
- **Expected result:** No parse errors; valid: true; no missing-step error
- **Edge case:** No

### AC1: bitbucket-pipelines.yml with intentional YAML syntax error fails with parse error (negative path)

- **Verifies:** AC1 (confirms validator is active, not always-pass)
- **Precondition:** Malformed YAML fixture (e.g. missing colon, invalid indent)
- **Action:** Pass malformed fixture to YAML syntax validator
- **Expected result:** `valid: false`; error message references YAML syntax; status code / error type distinguishes syntax-error from pipeline-shape-error
- **Edge case:** No

## Integration Tests

### AC3: Bitbucket DC Docker — app-password auth topology test returns no 401/403 (Docker-gated)

- **Verifies:** AC3
- **Components involved:** Bitbucket DC test script, Docker-running DC instance, app-password credentials injected via CI secret
- **Precondition:** DC Docker instance running (`docker compose up` from AC7 fixture); `APP_PASSWORD` and `BB_USER` available as CI secrets
- **Action:** Run DC app-password auth topology test against the local Docker DC instance
- **Expected result:** Request succeeds; HTTP response status is not 401 or 403; test reports auth topology as valid
- **Edge case:** No; Docker-only prerequisite documented in prerequisites section below

### AC4: Bitbucket DC Docker — OAuth consumer token obtained successfully (Docker-gated)

- **Verifies:** AC4
- **Components involved:** Bitbucket DC test script, Docker DC instance, OAuth consumer key+secret
- **Precondition:** DC Docker instance running; `OAUTH_KEY` and `OAUTH_SECRET` available as CI secrets
- **Action:** Run DC OAuth consumer auth topology test
- **Expected result:** Token obtained; no auth error; test reports OAuth topology as valid
- **Edge case:** No

### AC5: Bitbucket DC Docker — SSH key auth; no SSH handshake failure (Docker-gated)

- **Verifies:** AC5
- **Components involved:** Bitbucket DC test script, Docker DC instance, SSH private key
- **Precondition:** DC Docker instance running; `SSH_PRIVATE_KEY` available as CI secret
- **Action:** Run DC SSH key auth topology test
- **Expected result:** SSH handshake completes; no SSH error (ConnReset, AuthFail, or PermissionDenied); test reports SSH key topology as valid
- **Edge case:** No

### AC6: Cloud and DC paths run independently with no shared state (isolation integration)

- **Verifies:** AC6 (independent execution)
- **Components involved:** Both Cloud and DC test runners; shared test harness state
- **Precondition:** Both pathways wired in test harness
- **Action:** Run Cloud-only path; confirm DC-related state is unset; then run DC-only path; confirm Cloud-related state is unset; run full suite; confirm outputs are independent
- **Expected result:** No shared state pollution between runs; each pathway executes cleanly without the other's credentials being required or accidentally consumed
- **Edge case:** No

### AC7: docker compose up starts DC instance and AC3/AC4/AC5 tests execute without external network access (Docker-gated)

- **Verifies:** AC7 (reproducibility)
- **Components involved:** `tests/fixtures/docker-compose.bitbucket-dc.yml`, local Docker daemon, DC auth tests
- **Precondition:** Docker daemon running; no external network access to production Bitbucket DC
- **Action:** Run `docker compose -f tests/fixtures/docker-compose.bitbucket-dc.yml up -d`; run DC auth tests against local DC instance; run `docker compose down`
- **Expected result:** DC instance starts successfully; auth tests execute (may fail on auth depending on local config, but do not fail with "host not found" or "connection refused to external host"); `docker compose up` workflow documented in README or fixture header comment
- **Edge case:** No

## NFR Tests

### Security: no credential literals committed to repository

- **NFR addressed:** Security (no credentials in repo)
- **Measurement method:** Run `git log -p --all -- tests/` and `git grep -n "password\|secret\|private_key\|OAUTH" -- tests/fixtures/`; also scan `docker-compose.bitbucket-dc.yml` for literal credential values
- **Pass threshold:** Zero credential literals found in any committed file; all sensitive values reference environment variables (`${APP_PASSWORD}`, etc.)
- **Tool:** git grep, static scan

### Isolation: Cloud and DC test runs produce no shared state side-effects

- **NFR addressed:** Isolation
- **Measurement method:** After a Cloud-only run, inspect CI environment and test output for DC-specific variables set or consumed; vice versa for DC-only run
- **Pass threshold:** Zero cross-contamination instances; each path exits without setting state the other path reads
- **Tool:** CI output inspection; test harness isolation check

### Reproducibility: Docker Compose workflow reproducible with no external network required

- **NFR addressed:** Reproducibility
- **Measurement method:** Run `docker compose up` without internet connectivity (or with DC external calls blocked at network layer); confirm AC3/AC4/AC5 tests execute (framework-side, not auth-side)
- **Pass threshold:** Tests reach the auth topology evaluation stage without failing at the network-resolution level; local DC instance addresses all DC test targets
- **Tool:** `docker compose up`, network isolation test (e.g. `--network none` or firewall rule)

## Prerequisites

The following prerequisites are required for AC3, AC4, AC5, and AC7 integration tests. If these are unavailable, those tests are blocked and must be marked as skipped with reason `[PREREQ-DOCKER]`:

1. Docker daemon running and `docker compose` available
2. `tests/fixtures/docker-compose.bitbucket-dc.yml` committed (AC7 deliverable)
3. CI secrets configured: `APP_PASSWORD`, `BB_USER`, `OAUTH_KEY`, `OAUTH_SECRET`, `SSH_PRIVATE_KEY`

**These prerequisites are not blocking for AC1, AC2, or AC6 unit tests** — those run without Docker or credentials.
