# Contract Proposal — Implement versioned skill distribution and progressive skill disclosure

**Contract status:** Approved — 2026-04-10

## What will be built

- An assembly mechanism / script that produces a `copilot-instructions.md` in consuming squad repos, composed from three declared layers: (1) core platform layer from the versioned skills-repo package, (2) domain layer (optional, absent-marker in header if not present), (3) squad layer (optional)
- A composition header in the assembled file recording: layer sources, platform version reference (commit hash or tag), and declared layer order
- A progressive disclosure configuration within the assembled file: outer loop skills (discovery, benefit-metric, definition, review, test-plan, definition-of-ready, workflow, decisions) loaded at session start; inner loop skills (tdd, implementation-plan, subagent-execution, verify-completion, branch-setup, branch-complete) deferred until inner loop phase declared
- A `/load [skill-name]` handler documented in the assembled file that makes phase-deferred skills available for the remainder of the session without restart
- An M1 acceptance test record structure in `benefit-metric.md` under the M1 evidence section, with fields: `changeReference`, `distributionMechanism`, `elapsedTime`, `changePresentInAssembledContext`, `squadMergeActionRequired`

## What will NOT be built

- Push infrastructure CI scheduler / pull automation — the push vs pull implementation decision is made at implementation start and logged in `decisions.md`; this contract covers the format and mechanism contract, not the automation layer
- EA registry integration (Path A surface type resolution) — Phase 2; explicitly out of scope
- Squad override conflict resolution tooling — override model is documented but no tooling built
- Automated drift detection — Phase 3
- Token telemetry for the progressive disclosure pattern — out of scope per story

## How each AC will be verified

| AC | Test approach | Type |
|----|---------------|------|
| AC1 | Open session in squad repo using assembled file; invoke `/load discovery`; confirm no SKILL.md files copied into squad repo | Integration + Manual |
| AC2a | Commit controlled change to skills-repo; observe presence in assembled file after one distribution cycle (≤72h push / one session open pull) | Integration + Manual |
| AC2b | Inspect squad git log — confirm no squad-authored merge commits during distribution period | Manual |
| AC2c | Confirm assembled file commit was authored by distribution mechanism, not squad member | Manual |
| AC3 | Parse assembled fixture: assert outer loop skills present, inner loop skills absent; assert `/tdd` / `tdd/SKILL.md` not present in outer loop context | Unit (×2) + Manual |
| AC4 | In a session with progressive disclosure active, invoke `/load tdd`; confirm skill available without session restart | Integration + Manual |
| AC5 | Parse assembled file header: assert three-layer references in declared order; assert composition metadata (version reference) present | Unit (×2) + Manual |
| AC6 | Assert M1 acceptance test record contains all five required fields with correct values | Unit + Integration |

## Assumptions

- The push vs pull decision will be made at implementation start and logged in `decisions.md` before any code is written
- A consuming squad repo is available for the AC1/AC2/AC4 acceptance tests
- The assembled `copilot-instructions.md` header is a structured Markdown comment block (not a separate metadata file)
- SESSION START and SKILL loading behaviour is governed by Markdown content in the assembled file — no runtime component outside of standard Copilot Chat context loading
