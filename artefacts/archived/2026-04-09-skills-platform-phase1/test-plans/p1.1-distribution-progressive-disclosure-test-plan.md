## Unit Tests

### `progressive-disclosure-outer-loop-skills-only-at-discovery` (AC3)

- **Verifies:** AC3 — only outer loop skills are loaded at session start; inner loop skills absent
- **Precondition:** Assembled `copilot-instructions.md` fixture in outer loop (discovery) state
- **Action:** Parse the loaded skills list from the fixture; assert that outer loop skills are present (discovery, benefit-metric, definition, review, test-plan, definition-of-ready); assert that inner loop skills are absent (tdd, implementation-plan, subagent-execution, verify-completion, branch-setup, branch-complete)
- **Expected result:** All six outer loop skills found; all six inner loop skills not found
- **Edge case:** No — a future skill added to either list should update this test, not break it silently

### `progressive-disclosure-inner-loop-absent-at-outer-phase` (AC3)

- **Verifies:** AC3 — a specific inner loop skill name is not present in the outer loop context
- **Precondition:** Same fixture as above
- **Action:** Assert that the string `/tdd` or `tdd/SKILL.md` does not appear in the loaded context at session start
- **Expected result:** Zero occurrences
- **Edge case:** No

### `assembled-file-has-three-layer-composition` (AC5)

- **Verifies:** AC5 — assembled file contains evidence of three-layer composition in the header
- **Precondition:** Assembled `copilot-instructions.md` from the distribution mechanism
- **Action:** Read the file header (first 50 lines); assert presence of: (1) reference to the core platform layer or versioned skills-repo package, (2) a domain layer marker or absent-marker, (3) a squad layer marker; assert they appear in declared order
- **Expected result:** All three layer references present in order; if domain layer is absent, the header notes its absence explicitly
- **Edge case:** No

### `assembled-file-composition-order-documented` (AC5)

- **Verifies:** AC5 — composition order is documented in the file header (human-readable)
- **Precondition:** Same assembled file
- **Action:** Assert the header contains a comment or section describing the composition order and the platform version/hash it was assembled from
- **Expected result:** Header contains assembly metadata — version reference or commit hash, layer order
- **Edge case:** No

### `m1-acceptance-test-record-contains-all-fields` (AC6)

- **Verifies:** AC6 — the M1 acceptance test record contains all required fields
- **Precondition:** A completed M1 acceptance test output (fixture or real)
- **Action:** Assert the record contains: `changeReference` (non-empty), `distributionMechanism` (push or pull), `elapsedTime` (numeric), `changePresentInAssembledContext` (boolean true), `squadMergeActionRequired` (boolean false)
- **Expected result:** All five fields present with the correct values for a passing M1 test

## Integration Tests

### `skills-available-without-squad-file-copy` (AC1)

- **Verifies:** AC1 — core platform skills accessible via `/load` in a session without the skills-repo being copied into the squad repo
- **Components involved:** Assembled `copilot-instructions.md`, distribution mechanism, session loading
- **Precondition:** A consuming squad repo with no skills-repo files copied; assembled `copilot-instructions.md` generated by the distribution mechanism
- **Action:** Open a session using the assembled file; invoke `/load discovery`; confirm the discovery skill is available
- **Expected result:** Discovery skill loads and is available for use; the squad repo's file tree contains no copied skills-repo SKILL.md files

### `distribution-cycle-delivers-change-to-squad-context` (AC2a)

- **Verifies:** AC2a — a skills-repo change reaches the squad's assembled context within one distribution cycle
- **Components involved:** Skills-repo, distribution mechanism, consuming squad repo, assembled `copilot-instructions.md`
- **Precondition:** A controlled change committed to skills-repo (add or modify a single identifiable string)
- **Action:** Wait one distribution cycle (≤72h push or one session open pull); inspect the squad's assembled `copilot-instructions.md` for the change
- **Expected result:** The controlled change is present in the assembled file

### `load-command-loads-non-phase-skill` (AC4)

- **Verifies:** AC4 — `/load [skill-name]` loads a skill not in the current phase context
- **Components involved:** Progressive disclosure mechanism, session context, `/load` command
- **Precondition:** A session in outer loop state; `tdd` skill not loaded
- **Action:** Invoke `/load tdd`; confirm the TDD skill is available for the rest of the session
- **Expected result:** TDD skill is loaded and available; no session restart required

### `m1-acceptance-test-full-cycle` (AC6)

- **Verifies:** AC6 — the full M1 acceptance test records all required fields from an end-to-end distribution cycle
- **Components involved:** Skills-repo, distribution mechanism, consuming squad repo, M1 measurement record
- **Precondition:** Live distribution cycle (see test data note above)
- **Action:** Commit a controlled change to skills-repo; observe distribution; record all five required M1 fields; confirm change present and no squad merge required
- **Expected result:** All five M1 fields recorded; `changePresentInAssembledContext = true`; `squadMergeActionRequired = false`

## NFR Tests

### `nfr-assembled-file-under-8000-tokens`

- **Verifies:** NFR — Performance: assembled `copilot-instructions.md` base layer must not exceed 8,000 tokens at session-start state
- **Precondition:** Assembled `copilot-instructions.md` in outer loop (session-start) state, with only outer loop skills loaded
- **Action:** Tokenise the file content using the same tokeniser as the target model (or a proxy); assert token count ≤ 8,000
- **Expected result:** Token count at or below 8,000

### `nfr-no-credential-values-in-distribution-config`

- **Verifies:** NFR — Security: distribution config must contain only credential name references, no values
- **Precondition:** The distribution mechanism configuration file(s) committed to the repo
- **Action:** Scan all distribution config files for credential-value patterns (`ghp_`, `Bearer [A-Za-z0-9]{20,}`, `token: [^$]`, `password: [^$]`)
- **Expected result:** Zero matches

### `nfr-assembled-file-header-contains-version-info`

- **Verifies:** NFR — Audit: assembly composition order and source versions readable from file header
- **Precondition:** Assembled `copilot-instructions.md`
- **Action:** Read file header; assert it contains a platform version reference (commit hash, tag, or equivalent) that a reviewer could use to identify which platform version the session was running against
- **Expected result:** Version reference present and non-empty in the header
