## Scenario 5 — AC4: merged scenario cannot be silently removed or skipped

**Given:** A scenario has been accepted and merged into `workspace/suite.json`
**When:** Any subsequent gate run executes the suite
**Then:**
- The scenario appears in the active evaluation set for every gate run
- Attempting to remove the scenario without a `retiredAt` and `retiredReason` returns a silent-removal error
- A scenario with both `retiredAt` and `retiredReason` populated is classified as explicitly retired and is excluded (not silently skipped — the retirement is auditable)

**Automated:** Tests U6, U7, I4
**Status:** 🔴 Not yet passing

## Scenario 6 — AC5: M4 controlled acceptance test (dependency-blocked)

**Given:** The complete suite exists, `p1.4-watermark-gate` is DoD-complete, and `p1.3-assurance-agent-ci-gate` is running
**When:** The controlled M4 acceptance test is executed:
1. Add a failing scenario to `workspace/suite.json` (a scenario whose `expectedOutcome` will not be present in the next gate run output)
2. Run the assurance gate
3. Observe the gate verdict
4. Remove the failing scenario
5. Run the assurance gate again
6. Observe the restored verdict
**Then:**
- Step 3: gate returns `blocked` verdict
- Step 6: gate returns `pass` verdict
- The regression guarantee holds: the failing scenario triggered the block; removing it restored the pass

**Automated:** Acceptance test (requires p1.4 and p1.3 DoD-complete — dependency-blocked; scheduled after both are delivered)
**Status:** ⏸ Dependency-blocked (p1.4 + p1.3 prerequisite)
