## Unit Tests

### AC1 — Sign-off commit construction

**T1.1** `buildSignOffPayload(markdown, user, timestamp) appends ## Approved by section`
- **AC:** AC1
- **Precondition:** `user = { name: 'Test Stakeholder' }`; `timestamp = '2026-05-02T10:00:00Z'`; markdown from `discovery-unsigned.md`
- **Action:** `const updated = buildSignOffPayload(markdown, user, timestamp)`
- **Expected:** Returned string contains `## Approved by\n\nTest Stakeholder — 2026-05-02T10:00:00Z`; original markdown content unchanged before the appended section
- **Fails before implementation:** Yes

**T1.2** `buildSignOffPayload timestamp is ISO 8601 formatted`
- **AC:** AC1
- **Precondition:** `timestamp` generated by `new Date().toISOString()`
- **Action:** `buildSignOffPayload(markdown, user, timestamp)`
- **Expected:** The timestamp in the appended section matches ISO 8601 pattern `YYYY-MM-DDTHH:mm:ssZ`
- **Fails before implementation:** Yes

**T1.3** `commitSignOff(artefactPath, signOffPayload, token) calls Contents API with correct method and auth`
- **AC:** AC1
- **Precondition:** Mock GitHub Contents API PUT endpoint; access token from `oauth-token-exchange-success.json`
- **Action:** `await commitSignOff('artefacts/2026-01-01-example-feature/discovery.md', payload, 'gho_test_fixture_token_wuce1')`
- **Expected:** `fetch` called with PUT method; URL contains correct artefact path; Authorization header `Bearer gho_test_fixture_token_wuce1`; request body contains updated file content and commit message
- **Fails before implementation:** Yes

### AC3 — Commit author/committer identity

**T3.1** `commitSignOff sets author and committer to authenticated user's GitHub identity`
- **AC:** AC3
- **Precondition:** Authenticated user from `user-identity.json` (`login: 'test-stakeholder'`, `email: 'stakeholder@example.com'`); mock Contents API
- **Action:** Capture the request body from `commitSignOff` call
- **Expected:** Request body `committer.name === 'Test Stakeholder'`; `committer.email === 'stakeholder@example.com'`; `author.name === 'Test Stakeholder'`; `author.email === 'stakeholder@example.com'`; no reference to a service account or bot identity
- **Fails before implementation:** Yes

### AC4 — Path traversal prevention

**T4.1** `validateArtefactPath returns false for path containing ../`
- **AC:** AC4
- **Precondition:** None
- **Action:** `validateArtefactPath('../etc/passwd')`
- **Expected:** Returns `false`
- **Fails before implementation:** Yes

**T4.2** `validateArtefactPath returns false for absolute path not starting with artefacts/`
- **AC:** AC4
- **Precondition:** None
- **Action:** `validateArtefactPath('/root/secret.md')`
- **Expected:** Returns `false`
- **Fails before implementation:** Yes

### AC5 — Contents API conflict handling

**T5.1** `commitSignOff throws SignOffConflictError when Contents API returns 409`
- **AC:** AC5
- **Precondition:** Mock Contents API to return `contents-api-conflict.json` with status 409
- **Action:** `await commitSignOff(path, payload, token)`
- **Expected:** Throws `SignOffConflictError`
- **Fails before implementation:** Yes

### AC6 — Already signed off detection

**T6.1** `detectExistingSignOff(markdown) returns true when ## Approved by section is present`
- **AC:** AC6
- **Precondition:** Markdown from `discovery-signed.md` containing `## Approved by` section
- **Action:** `detectExistingSignOff(markdown)`
- **Expected:** Returns `true`
- **Fails before implementation:** Yes

**T6.2** `detectExistingSignOff returns the approver name and timestamp from the existing section`
- **AC:** AC6
- **Precondition:** `discovery-signed.md` with `Test Stakeholder — 2026-04-28T14:32:00Z`
- **Action:** `const result = detectExistingSignOff(markdown)`
- **Expected:** `result.approver === 'Test Stakeholder'`; `result.date === '2026-04-28T14:32:00Z'`
- **Fails before implementation:** Yes

## Integration Tests

**IT1** `POST /sign-off with valid artefact path and no existing sign-off commits and returns 200`
- **ACs:** AC1, AC3
- **Precondition:** Authenticated session; `discovery-unsigned.md` fetched; mock Contents API PUT returns `contents-api-write-success.json`
- **Action:** `POST /sign-off` with body `{ artefactPath: 'artefacts/2026-01-01-example-feature/discovery.md' }`
- **Expected:** Status 200; response body contains success confirmation; mock Contents API PUT was called once with correct author/committer identity
- **Fails before implementation:** Yes

**IT2** `GET /artefact/:slug/discovery after sign-off commit shows ## Approved by section`
- **ACs:** AC2
- **Precondition:** Mock Contents API to return the post-commit version of the artefact (signed markdown)
- **Action:** `GET /artefact/2026-01-01-example-feature/discovery`
- **Expected:** Response body contains `Approved by` text (rendered from the signed markdown)
- **Fails before implementation:** Yes

**IT3** `POST /sign-off with path traversal returns 400 and does not call Contents API`
- **ACs:** AC4
- **Precondition:** Authenticated session; mock Contents API (to detect if called)
- **Action:** `POST /sign-off` with body `{ artefactPath: '../../etc/passwd' }`
- **Expected:** Status 400; Contents API mock not called
- **Fails before implementation:** Yes

**IT4** `POST /sign-off when Contents API returns 409 returns 409 with reload instruction`
- **ACs:** AC5
- **Precondition:** Authenticated session; mock Contents API to return 409
- **Action:** `POST /sign-off` with valid artefact path
- **Expected:** Status 409; response body contains "Artefact was updated" or "please reload" message
- **Fails before implementation:** Yes

**IT5** `POST /sign-off when artefact already has ## Approved by returns 409 with existing approver info`
- **ACs:** AC6
- **Precondition:** Authenticated session; mock fetch returns `discovery-signed.md` content; mock Contents API (should not be called)
- **Action:** `POST /sign-off` with path pointing to already-signed artefact
- **Expected:** Status 409; response body contains approver name and date; Contents API PUT not called
- **Fails before implementation:** Yes

## NFR Tests

**NFR1** `sign-off endpoint rate-limits at 10 attempts per user per minute`
- **NFR:** Security
- **Precondition:** Make 11 sign-off POST requests in sequence with the same user session
- **Action:** Check status of the 11th request
- **Expected:** 11th request returns 429 (Too Many Requests)
- **Fails before implementation:** Yes

**NFR2** `sign-off submission event is logged with user ID, artefact path, and timestamp`
- **NFR:** Audit
- **Precondition:** Mock logger; authenticated session with user ID 99001; successful sign-off
- **Action:** Call sign-off route handler
- **Expected:** Logger called with `event: 'signoff_submitted'`, `userId: 99001`, `artefactPath: 'artefacts/2026-01-01-example-feature/discovery.md'`, ISO timestamp
- **Fails before implementation:** Yes

**NFR3** `server does not have write access in its own right — no server-level GitHub token used for commits`
- **NFR:** Security
- **Precondition:** Verify environment configuration does not include a server-side `GITHUB_PERSONAL_ACCESS_TOKEN` used for write operations
- **Action:** Inspect `commitSignOff` adapter implementation — token parameter must come from the session (user's OAuth token), not from an env var
- **Expected:** The write-path adapter accepts `token` as a parameter; no write operation falls back to an env-var token
- **Fails before implementation:** Yes
