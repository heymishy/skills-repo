## What will be built

1. **`run-assurance-gate.js`** (or the gate-logic script called by `tests/check-assurance-gate.js` — confirm exact path) — extended with 5 substantive content checks:
   - AC2: `state.json` contains the expected `version` and `features` fields (schema field presence + non-empty)
   - AC3: `state.json` passes schema validation (reads `pipeline-state.schema.json` and validates with `ajv` or equivalent)
   - AC4: At least one story reaches `stage: "definition-of-ready"` (semantic content assertion)
   - AC5: `workspace/state.json` includes a `currentPhase` block with a non-empty `lastUpdated` field
   - AC6: Gate execution runtime < 30s (measured and reported; exits 1 if exceeded)

2. **`pipeline-state.schema.json`** — modified only if new trace output fields (`checksRun`, `failedChecks`) are added to the gate's written output. Per ADR-003: any new field in the output trace must be declared in the schema in the same commit. If no new output fields are added, this file is unchanged.

---

## What will NOT be built

- No external service calls or network IO in the gate
- No changes to `npm test` command line (gate continues to run as part of existing test suite)
- No changes to `.github/skills/` files
- No new npm packages — plain Node.js CommonJS only
- `pipeline-state.schema.json` is changed only if new trace fields are written — not as a speculative change
