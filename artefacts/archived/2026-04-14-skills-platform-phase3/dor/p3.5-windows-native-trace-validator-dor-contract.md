## What will be built

1. **`scripts/validate-trace.ps1`** — new PowerShell script. Parallel implementation of `scripts/validate-trace.sh`. Must:
   - Accept `--ci` flag with identical semantics to the bash version
   - Run every check present in `validate-trace.sh` under the same check names
   - Exit 0 on validation pass; exit non-zero on hard failure
   - Print per-check output in the same format as the bash version
   - Use only PowerShell 5.1+ built-in cmdlets — no external module dependencies
   - Complete within 10 seconds on any Windows PowerShell 5.1+ environment

---

## What will NOT be built

- No modification to `scripts/validate-trace.sh`
- No PowerShell equivalents for any other script in `scripts/`
- No changes to npm test or pre-commit hooks
- No new external PowerShell module dependencies

## AC verification mapping

| AC | Test | Type | Spec |
|----|------|------|------|
| AC1 | `scripts/validate-trace.ps1` exists; runs with `--ci` on valid trace; exits 0 | Unit + Integration | File present; invoke with valid fixture; assert exit code 0 |
| AC2 | Hard failure trace → exit non-zero + specific field in output | Unit | Invoke with fixture missing required field; assert exit code != 0 and failed field name in output |
| AC3 | Every check in `.sh` is present in `.ps1` (by check name) | Integration / parity | Extract check names from both files; assert sets are equal |
| AC4 | `pwsh -File scripts/validate-trace.ps1 --ci` exits with same code as `bash scripts/validate-trace.sh --ci` on same input | Integration | Run both on same fixture; compare exit codes |
| AC5 | Running `validate-trace.sh` on Linux produces unchanged output (PS1 file does not interfere) | Integration | Confirm .sh execution path is unmodified by .ps1 presence |
| NFR | validate-trace.ps1 --ci completes within 10 seconds | NFR / timing | Time execution on standard fixture; assert < 10s |

## Assumptions

- `pwsh` or `powershell.exe` is available on the CI runner for integration test execution.
- A valid trace fixture and a failing trace fixture both exist or will be created as test data.
- The test for AC5 can be a static code check (confirm .sh does not import or call .ps1).
