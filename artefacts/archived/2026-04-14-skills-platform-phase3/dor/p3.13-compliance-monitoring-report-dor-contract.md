# DoR Contract — Produce automated compliance monitoring report from cross-team trace registry

**Story reference:** p3.13-compliance-monitoring-report.md
**Date:** 2026-04-18
**Status:** BLOCKED — awaiting p3.7 DoD-complete AND p3.2a DoD-complete

---

## What will be built

An audit sampling module for the improvement agent that:
- Calls `getTraces({ filter: { period: 'last-30-days' } })` across all participating squads
- Generates `workspace/compliance-reports/compliance-[YYYY-MM].md` on a monthly schedule
- Reports per-squad: squad ID, trace count sampled, gate verdicts present (yes/no), T3M1 fields populated status (yes/no + missing count), gap count, overall compliance status (PASS/FAIL)
- Marks squads exceeding 10% gap threshold as FAIL with specific missing field names and trace slugs
- Includes self-contained header: policy threshold, sample size, period covered, registry path reference
- Optional tamper-evidence verification if p3.2b is DoD-complete when implementation runs

A GitHub Actions workflow (`compliance-report.yml`) with:
- Monthly schedule trigger (`0 0 1 * *`)
- Read-only registry token (ADR-009 compliant)
- No delivery pipeline trigger

---

## What will NOT be built

- Automated remediation on FAIL status — report is advisory only
- Real-time or on-demand compliance monitoring — monthly cadence is Phase 3 target
- External governance platform integration (ServiceNow, Archer) — Phase 4
- Marking non-registered squads as FAIL — listed as "not registered" only
- Any write access to the trace registry

## Assumptions

- p3.7 DoD-complete: `getTraces()` API available and stable before implementation starts
- p3.2a DoD-complete: T3M1 fields (standardsInjected, watermarkResult, stalenessFlag, sessionIdentity) defined and present in trace data before implementation starts
- p3.2b may or may not be complete when implementation runs — report handles both cases
- Policy threshold of 10% is established by this story (no prior definition exists); can be made configurable via environment variable if needed
- Monthly compliance reports are committed to the delivery repo (not platform-infrastructure); no overwrite guarantee from timestamp in filename
