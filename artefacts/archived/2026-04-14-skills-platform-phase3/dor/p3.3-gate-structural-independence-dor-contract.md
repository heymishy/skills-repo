## What will be built

1. **`platform-infrastructure` repository** (external) — contains `run-assurance-gate.js` and all scripts it invokes. Delivery agent CI identity has no push, create-branch, or delete access. Gate script releases are published manually by the platform maintainer.

2. **Thin shim script (delivery repo)** — replaces the current gate script entry point in the delivery repo. Contains no governance logic; its only function is to invoke the downloaded gate. Name TBD at implementation time.

3. **Gate-download CI workflow step (delivery repo)** — added to the assurance gate CI workflow. Downloads the gate script from a pinned immutable ref in `platform-infrastructure` (full SHA or version tag). Computes SHA-256 checksum of downloaded file. Compares against known-good value committed in the delivery repo. Exits 1 with "Gate script checksum mismatch — aborting" if mismatch. Uses HTTPS. Uses OIDC or read-only GitHub App token — no PAT.

4. **`pipeline-state.schema.json`** — modified in the same commit as the CI workflow to add:
   - `gateScriptRef` — the platform-infrastructure ref (SHA or tag) used to download the gate
   - `gateChecksumVerified` — boolean, `true` when checksum matched

5. **`pipeline-state.json`** — updated at gate-execution time to record `gateScriptRef` and `gateChecksumVerified: true` per story in the stories[n] object (not in JSONL trace file — per review finding 1-L1 clarification).

---

## What will NOT be built

- No automated release publishing to `platform-infrastructure` (Phase 4)
- No movement of SKILL.md files — only executable gate scripts are relocated
- No migration of other CI scripts (e.g. `scripts/validate-trace.sh`)

- The platform maintainer creates the `platform-infrastructure` repository manually; the coding agent cannot create repositories.
- The known-good checksum value is committed to the delivery repo before the first gate download run.
- Complexity 3 / scope Unstable: implementation details for the repository creation step may be refined at implementation time to match the confirmed Git host model.
