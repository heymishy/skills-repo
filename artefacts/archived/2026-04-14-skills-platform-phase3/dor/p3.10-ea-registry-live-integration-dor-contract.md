## What will be built

1. **`.github/skills/discovery/SKILL.md`** — modified to add a two-path integration block:
   - Path A block (active when `context.yml` → `architecture.ea_registry_authoritative: true`): describes calling `getBlastRadius(systemId)` to surface affected systems count, downstream consumers list, and domain owner inline in the discovery session. Includes a comment referencing ADR-007 (surface-type mapping table).
   - Path B block (active when `architecture.ea_registry_authoritative: false` or absent): confirms existing discovery behaviour is unchanged — no blast-radius section, no error.
   - Missing registry entry case: surface "no EA registry entry found for [systemId]" and continue.
   - All existing content is preserved — blocks are additions only.

---

## What will NOT be built

- No write operations to the EA registry from any path
- No embedding of the full registry content in discovery artefacts — only blast-radius summary
- No Phase 4 skills integration (EA registry contribution from definition/review skills)
- No runtime code modules — SKILL.md update only
