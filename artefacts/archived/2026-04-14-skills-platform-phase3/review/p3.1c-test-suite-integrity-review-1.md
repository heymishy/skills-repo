## Findings

**1-L1 (LOW):** AC2 specifies "the floor value is documented in the source file as a named constant." The testable outcome is that `watermark-gate.js` throws or exits 1 when passRate is below 0.70 — the named-constant storage requirement is an implementation prescription that constrains how the floor is written, not what it does. This is a coding-standards concern rather than an acceptance criterion. No block. Recommendation: remove the named-constant clause from AC2, or move it to the Architecture Constraints section as a coding standard note.
