## Test Data Strategy

**Source:** Synthetic `estimation-norms.md` fixtures created in test setup:
- `fixtures/estimation-norms-trigger.md` — 3 features all >30% underrun: p1 (E=4h, A=6h=50%), p2 (E=8h, A=11h=37.5%), p3 (E=3h, A=4.2h=40%)
- `fixtures/estimation-norms-below-count.md` — only 2 features >30% underrun
- `fixtures/estimation-norms-below-rate.md` — 3 features but max underrun is 28% (below threshold)
- `fixtures/estimation-norms-mixed.md` — 4 features: 2 >30%, 1 exactly 30%, 1 <30% → total triggering count is 2 (boundary case — no trigger)

**PCI/sensitivity:** No — estimation data is internal; no customer data
**External network:** None — module reads local fixture files

## Unit Tests

### calibration-detects-three-features-above-threshold

- **Verifies:** AC1 (basic trigger: 3 features all >30%)
- **Test file:** `src/improvement-agent/calibration.test.js`
- **Precondition:** Calibration module exists at `src/improvement-agent/calibration.js`; `fixtures/estimation-norms-trigger.md` prepared
- **Action:** Run the calibration detection dimension with `fixtures/estimation-norms-trigger.md` as input; check the output
- **Expected result:** A proposal file is generated at `workspace/proposals/estimation-calibration-[today-date].md` (or equivalent output — test can check returned proposal object before disk write)
- **RED step:** Module does not exist; no proposal generated

### proposal-file-contains-required-fields

- **Verifies:** AC2 (proposal content completeness)
- **Test file:** `src/improvement-agent/calibration.test.js`
- **Precondition:** Calibration module exists; trigger fixture used
- **Action:** Run calibration detection with trigger fixture; capture the proposal content; check for all required fields: (a) underestimation rate as percentage, (b) list of contributing feature slugs, (c) extract of current affected E1/E2 guidance, (d) proposed updated guideline, (e) rationale
- **Expected result:** All five fields present and non-empty in the proposal content
- **RED step:** Proposal is generated but is missing one or more required fields

### calibration-silent-for-fewer-than-three-features

- **Verifies:** AC3 (below count threshold)
- **Test file:** `src/improvement-agent/calibration.test.js`
- **Precondition:** Calibration module exists; `fixtures/estimation-norms-below-count.md` prepared (2 features >30%)
- **Action:** Run calibration detection with below-count fixture
- **Expected result:** No proposal generated (module returns null or undefined, no file written); no error or warning thrown
- **RED step:** Module generates a proposal even with only 2 triggering features

### calibration-silent-for-rate-below-threshold

- **Verifies:** AC3 (below rate threshold)
- **Test file:** `src/improvement-agent/calibration.test.js`
- **Precondition:** Calibration module exists; `fixtures/estimation-norms-below-rate.md` prepared (3 features but ≤30% underrun)
- **Action:** Run calibration detection with below-rate fixture
- **Expected result:** No proposal generated; no error
- **RED step:** Module uses an incorrect threshold comparison (e.g. ≥30 instead of >30)

### calibration-does-not-write-to-skill-files

- **Verifies:** AC4 (no auto-apply)
- **Test file:** `src/improvement-agent/calibration.test.js`
- **Precondition:** Calibration module exists; trigger fixture used; file system is watched or mocks are in place
- **Action:** Run calibration detection; after execution, verify no changes were made to `.github/skills/` directory, `workspace/estimation-norms.md`, or `context.yml`
- **Expected result:** Only the proposal file (in `workspace/proposals/`) is written; no SKILL.md, no norms file, no context.yml changes
- **RED step:** Module auto-applies the calibration by writing to SKILL.md

### constant-estimation-underrun-threshold-is-0-30

- **Verifies:** AC5 (named constant value)
- **Test file:** `src/improvement-agent/calibration.test.js` (or a dedicated constants test)
- **Precondition:** Calibration module exists
- **Action:** Import and inspect `ESTIMATION_UNDERRUN_THRESHOLD` from the calibration module
- **Expected result:** `ESTIMATION_UNDERRUN_THRESHOLD === 0.30`
- **RED step:** Constant not exported; or value differs; or it is an inline magic number

### constant-estimation-min-feature-count-is-3

- **Verifies:** AC5 (named constant value)
- **Test file:** `src/improvement-agent/calibration.test.js`
- **Precondition:** Calibration module exists
- **Action:** Import and inspect `ESTIMATION_MIN_FEATURE_COUNT` from the calibration module
- **Expected result:** `ESTIMATION_MIN_FEATURE_COUNT === 3`
- **RED step:** Constant not exported; or value is not exactly 3

## Integration Tests

### calibration-module-wired-into-improvement-agent

- **Verifies:** AC1 (module integration — calibration is a dimension of the improvement agent)
- **Test file:** `tests/check-improvement-agent.js` (extend)
- **Precondition:** Improvement agent entry point exists; calibration module exists
- **Action:** Import or inspect the improvement agent entry module; verify that the calibration detection function is invoked as part of a standard improvement agent run (i.e. it is registered as a dimension, not just an ad-hoc call)
- **Expected result:** Calibration detection is listed/wired in the improvement agent's dimension registry or run loop
- **RED step:** Calibration module exists but is not wired into the improvement agent; operator must call it manually
