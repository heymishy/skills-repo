## Tests

### M1 — Step 0 heading exists

**Type:** Manual
**ACs covered:** AC1

**Steps:**
1. Open `.github/skills/verify-completion/SKILL.md`
2. Search for `## Step 0`

**Expected result:** The heading `## Step 0 — Fresh-session check` is present in the file.

**Fails if:** The heading is absent or named differently.

---

### M2 — Step 0 is positioned before Step 1

**Type:** Manual
**ACs covered:** AC1

**Steps:**
1. Open `.github/skills/verify-completion/SKILL.md`
2. Note the line number of `## Step 0`
3. Note the line number of `## Step 1 — Run the full test suite`

**Expected result:** The Step 0 heading appears at a lower line number than Step 1.

**Fails if:** Step 0 appears after Step 1, or Step 1 is absent.

---

### M3 — Trigger condition text present

**Type:** Manual
**ACs covered:** AC2

**Steps:**
1. Open `.github/skills/verify-completion/SKILL.md`
2. Read the Step 0 section content

**Expected result:** The section contains text that identifies when it applies — specifically: resuming from a previous session, when state.json shows in-progress work, or when a prior session has made a completion claim.

**Fails if:** The step contains instructions but no trigger condition (i.e., no "when this applies" language).

---

### M4 — Independent verification instructions present

**Type:** Manual
**ACs covered:** AC3

**Steps:**
1. Open `.github/skills/verify-completion/SKILL.md`
2. Read the Step 0 section instructions

**Expected result:** The instructions include all four components:
- Do NOT inherit the previous session's completion claim
- Read state.json / workspace artefacts to understand what was claimed
- Run Step 1 fresh regardless of prior claims
- If prior claim and fresh result disagree, surface the discrepancy before proceeding

**Fails if:** Any of the four components is absent.

## AC verification script (plain language)

### AC1 verification
Open `.github/skills/verify-completion/SKILL.md` and locate the headings. Confirm `## Step 0 — Fresh-session check` appears before `## Step 1 — Run the full test suite`. Pass if both headings exist in that order.

### AC2 verification
Read the Step 0 section. Confirm it says when it applies (e.g. "when resuming from a previous session"). Pass if trigger language is present.

### AC3 verification
Read the Step 0 instructions. Confirm they say: (1) do not inherit prior claim, (2) check state.json / artefacts first, (3) run Step 1 fresh anyway, (4) surface any discrepancy. Pass if all four points are present.
