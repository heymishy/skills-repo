## Unit Tests

### issue-dispatch-skill-md-contains-closes-keyword-in-dispatch-output-template

- **Verifies:** AC1
- **Precondition:** `.github/skills/issue-dispatch/SKILL.md` exists and has been updated
- **Action:** Read the file content; search for a line matching `Closes #` in the dispatch output template section
- **Expected result:** File contains at least one occurrence of `Closes #[issueNumber]` (or `Closes #` followed by a placeholder) within the PR description guidance section of the dispatch output template
- **Edge case:** No

### definition-of-ready-skill-md-contains-forward-pointer-to-issue-dispatch

- **Verifies:** AC2
- **Precondition:** `.github/skills/definition-of-ready/SKILL.md` exists and has been updated
- **Action:** Read the file content; search for a reference to `/issue-dispatch` and text indicating the Closes guidance is generated automatically
- **Expected result:** File contains a forward-pointer section at or near the DoR sign-off block that (a) references `/issue-dispatch` and (b) states that the Closes keyword guidance is generated by that skill — operator does not need to add it manually
- **Edge case:** No

### pr-diff-contains-only-two-skill-md-files

- **Verifies:** AC4 (PAT-07 compliance)
- **Precondition:** The PR implementing this story is open
- **Action:** Inspect the PR diff; list all changed files
- **Expected result:** The diff contains changes to exactly two files: `.github/skills/issue-dispatch/SKILL.md` and `.github/skills/definition-of-ready/SKILL.md`; no other files are modified
- **Edge case:** No
