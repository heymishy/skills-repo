### M2 — AC1b: Paragraph is positioned after the Completion expectation block

**Command:**
```powershell
$content = Get-Content ".github/copilot-instructions.md" -Raw
$completion = $content.IndexOf("**Completion expectation:**")
$writeSafety = $content.IndexOf("**State write safety:**")
Write-Host "Completion at $completion, WriteSafety at $writeSafety, Safety after completion: $($writeSafety -gt $completion)"
```

**Expected (FAIL before impl):** WriteSafety index returns -1.
**Expected (PASS after impl):** WriteSafety index > Completion index; output shows `Safety after completion: True`.

---

### M3 — AC2: Safe-write pattern elements all present

**Command:**
```powershell
$content = Get-Content ".github/copilot-instructions.md" -Raw
$hasComplete = $content -match "complete JSON"
$hasTemp = $content -match "state\.json\.tmp"
$hasVerify = $content -match "valid JSON"
$hasRename = $content -match "rename"
Write-Host "CompleteJSON: $hasComplete, TempFile: $hasTemp, VerifyJSON: $hasVerify, Rename: $hasRename"
```

**Expected (FAIL before impl):** All return False.
**Expected (PASS after impl):** All four return True.

---

### M4 — AC3: Failure mode and check script explicitly named

**Command:**
```powershell
$content = Get-Content ".github/copilot-instructions.md" -Raw
$hasErrorString = $content -match "Unexpected non-whitespace character after JSON"
$hasCheckScript = $content -match "check-workspace-state\.js"
Write-Host "ErrorString: $hasErrorString, CheckScript: $hasCheckScript"
```

**Expected (FAIL before impl):** Both return False (paragraph absent).
**Expected (PASS after impl):** Both return True.
