## Unit Tests

### ps1-exits-0-on-valid-trace-with-ci-flag

- **Verifies:** AC1
- **Precondition:** `scripts/validate-trace.ps1` exists; valid fixture trace file present at `tests/fixtures/valid-trace.yaml`
- **Action:** Run `pwsh -File scripts/validate-trace.ps1 --ci tests/fixtures/valid-trace.yaml`; capture exit code and stdout
- **Expected result:** Exit code 0; stdout lists each check name in the same order and format as `validate-trace.sh --ci` on the same input
- **Edge case:** No

### ps1-exits-0-prints-all-check-names

- **Verifies:** AC1 (output format parity)
- **Precondition:** Valid trace fixture; both PS1 and SH present
- **Action:** Capture stdout of both scripts on the same trace; compare check names line by line
- **Expected result:** Check names in PS1 output match check names in SH output exactly (case-insensitive on check name text; whitespace differences acceptable)
- **Edge case:** No

### ps1-exits-nonzero-on-missing-required-field

- **Verifies:** AC2
- **Precondition:** Fixture with a missing required field (e.g. `storySlug` absent)
- **Action:** Run `pwsh -File scripts/validate-trace.ps1 --ci tests/fixtures/missing-field-trace.yaml`
- **Expected result:** Exit code non-zero (1); stderr or stdout names the missing field
- **Edge case:** No

### ps1-and-sh-produce-same-exit-code-on-invalid-trace

- **Verifies:** AC2, AC4
- **Precondition:** Missing-field fixture; both PS1 and SH present
- **Action:** Run both scripts on the same invalid fixture; capture exit codes
- **Expected result:** Exit codes are equal (both non-zero)
- **Edge case:** No

### sh-script-unmodified-when-ps1-present

- **Verifies:** AC5
- **Precondition:** `scripts/validate-trace.sh` exists; PS1 has been added
- **Action:** Read the SHA-256 hash of `validate-trace.sh` before and after adding PS1; or compare file contents to the last known version
- **Expected result:** `validate-trace.sh` file contents are byte-for-byte identical to the version that passed all prior tests
- **Edge case:** No

## Integration Tests

### ps1-check-set-matches-sh-check-set-enumerated

- **Verifies:** AC3
- **Components:** `scripts/validate-trace.ps1`, `scripts/validate-trace.sh`
- **Precondition:** Both scripts present in the repository
- **Action:** Extract the list of check names from both scripts (by parsing output on a valid trace or by text inspection); produce two sorted arrays; compare
- **Expected result:** Arrays are equal — every check name in SH is present in PS1; no check is absent or renamed; counts match
- **Edge case:** No

### pwsh-ci-exit-code-matches-bash-on-same-input-valid

- **Verifies:** AC4 (valid trace)
- **Components:** `scripts/validate-trace.ps1`, `scripts/validate-trace.sh`, CI runner environment
- **Precondition:** Valid fixture; PowerShell 5.1+ available; bash available
- **Action:** Run both scripts with `--ci` on the valid fixture; capture exit codes
- **Expected result:** Both exit 0
- **Edge case:** No

### pwsh-ci-exit-code-matches-bash-on-same-input-invalid

- **Verifies:** AC4 (invalid trace)
- **Components:** Same scripts, invalid fixture
- **Action:** Run both scripts with `--ci` on the missing-field fixture
- **Expected result:** Both exit non-zero; exit codes match
- **Edge case:** No
