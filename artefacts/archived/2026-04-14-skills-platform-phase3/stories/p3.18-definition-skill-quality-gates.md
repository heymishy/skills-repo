# Story: Add platform-availability and prerequisite cross-codebase validation gates to /definition skill

**Epic reference:** artefacts/2026-04-14-skills-platform-phase3/epics/e1-governance-chain-integrity.md
**Discovery reference:** artefacts/2026-04-14-skills-platform-phase3/discovery.md
**Benefit-metric reference:** artefacts/2026-04-14-skills-platform-phase3/benefit-metric.md
**Improvement source:** workspace/learnings.md — two entries surfaced by /improvement run 2026-04-16

## User Story

As a **platform maintainer**,
I want the /definition skill to actively gate against untestable platform-dependent ACs and against prerequisite stories rooted in mismatched codebases,
So that the artefacts produced are verifiable at DoD and prerequisite dependencies do not produce void story chains.

## Benefit Linkage

**Metric moved:** MM2 — Outer loop self-sufficiency: Phase 3 delivery without documentation lookups
**How:** Embedding these gates directly in the /definition skill instruction prevents the operator from needing to recall or look up the validation steps externally — the skill enforces them at the right point in the workflow, reducing the chance of defects that must be corrected at review or DoR.

## Architecture Constraints

- Modifications are confined to `.github/skills/definition/SKILL.md` — no code or schema changes.
- The D2-platform gate must use mandatory language (not advisory), consistent with the existing testability-filter structure in D2.
- The D1-prereq block must follow the existing D1 dependency chain validation block in the same section.
- Language and heading style must match the existing SKILL.md convention to avoid confusion during future edits.

## Dependencies

- **Upstream:** None.
- **Downstream:** None — this story adds defensive gates to an existing skill; no downstream story depends on it.

## Acceptance Criteria

**AC1 — Platform-availability gate (D2-platform) present and positioned correctly**
Given `.github/skills/definition/SKILL.md` is read,
When the reader locates the D2 testability filter section,
Then a `**Platform-availability gate (D2-platform — mandatory, applied before writing any AC or Architecture Constraint):**` block is present immediately after the `**This filter is advisory only**` paragraph, containing: a "verify availability" instruction, a "defer / log / add revisit trigger" branch for unavailable platforms, a "do not embed unverifiable ACs" directive, and an explicit statement that this gate is mandatory (not advisory).

**AC2 — Prerequisite cross-codebase validation (D1-prereq) present and positioned correctly**
Given `.github/skills/definition/SKILL.md` is read,
When the reader locates the D1 dependency chain validation section,
Then a `**Prerequisite cross-codebase validation (D1-prereq — run before writing any prerequisite story flagged from a RISK-ACCEPT):**` block is present immediately after the existing D1 dependency chain validation block, containing: a three-point verification checklist (same repo, traces to a metric, correct delivery target), and a "surface to operator before writing the story" instruction triggered when any check fails.

**AC3 — Gate language is mandatory and consistent with existing SKILL.md style**
Given both new blocks are present,
When the text of each block is read,
Then neither block uses advisory language (e.g. "consider", "may", "should optionally") — both use imperative style ("verify", "defer", "do not", "flag to operator") consistent with other mandatory steps in the same SKILL.md file.

## Out of Scope

- Changes to any other skill file, governance check, schema, or source file.
- Adding automated tests that validate the SKILL.md text at CI time (this is a documentation-only change; verification is manual via the AC verification script).
- Changing the advisory status of the existing D2 testability filter — only the new D2-platform gate is mandatory.

## NFRs

None specific to this story — SKILL.md instruction text addition only.
