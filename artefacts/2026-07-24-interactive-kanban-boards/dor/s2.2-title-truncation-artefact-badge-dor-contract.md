# Contract Proposal: Port title truncation and artefact-count badge to the live kanban boards

**What will be built:**
`_renderKanbanColumns`'s card renderer wired to call the existing `truncateTitle()` function (already defined in `kanban-view.js`, currently only used by the legacy `featureCard()`). First implementation step: investigate whether `getArtefactsForJourney()` can be feasibly wired into the board's journey query without unbounded per-card query cost; if feasible, add the artefact-count badge using the existing badge markup/logic from `featureCard()`; if not, ship title truncation alone and document the gap.

**What will NOT be built:**
Any schema migration to pre-compute artefact counts. Any change to `getArtefactsForJourney()` itself.

**How each AC will be verified:**
| AC | Test approach | Type |
|----|---------------|------|
| AC1 | Unit test with a >48-char title | Unit |
| AC2 | Unit test with a ≤48-char title | Unit |
| AC3 | Integration test checking full title in a hover-accessible attribute | Integration |
| AC4 | Unit + Integration test, conditional on investigation outcome | Unit + Integration |
| AC5 | Integration test confirming truncation ships independent of AC4's outcome | Integration |

**Assumptions:**
The artefact-count data-availability investigation (AC4) is a genuine first implementation step, not pre-resolved — this is the RISK-ACCEPT logged in `decisions.md`.

**Estimated touch points:**
Files: `src/web-ui/views/kanban-view.js`, possibly `src/web-ui/routes/products.js` (if the journey query is extended for artefact counts).
Services: journey store, possibly `journey-store-pg.js`'s `getArtefactsForJourney`.
APIs: None new (extends existing board data endpoints if AC4 is pursued).
