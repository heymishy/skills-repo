# Contract Proposal: Redesign the product/org/tenant kanban board CSS to use the shared design-system tokens

**What will be built:**
Replace every raw hex colour value in `_renderKanbanColumns`'s inline `<style>` block (`kanban-view.js`) with the equivalent `var(--token-name)` from `html-shell.js`'s existing root/dark-theme token set. No new tokens invented unless a genuine gap is found (documented in `decisions.md` if so).

**What will NOT be built:**
Any change to the legacy `renderKanban({features,ideas})` path (dead code). Any new interactive elements' styling (those are Epic 1/3's own stories).

**How each AC will be verified:**
| AC | Test approach | Type |
|----|---------------|------|
| AC1 | Unit test regex-scanning the CSS output for hex patterns | Unit |
| AC2 | Manual side-by-side comparison (RISK-ACCEPT, no visual-regression tooling) | Manual |
| AC3 | Unit test (static override-block presence) + manual visual legibility check | Unit + Manual |
| AC4 | Integration test running existing psh-s6/psh-s7 specs unmodified | Integration |

**Assumptions:**
`html-shell.js`'s existing token set is sufficient for this board's needs — no board-specific new token is anticipated, but if found necessary, it will be added to `html-shell.js`'s root block (platform-wide), not locally.

**Estimated touch points:**
Files: `src/web-ui/views/kanban-view.js` only (CSS block).
Services: None.
APIs: None.
