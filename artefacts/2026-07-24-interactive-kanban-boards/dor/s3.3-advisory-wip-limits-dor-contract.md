# Contract Proposal: Show an advisory (soft) WIP-limit warning per column

**What will be built:**
Port `kanban-view.js`'s existing `lane()` function's WIP-badge logic (`kb-wip`/`kb-wip--over` classes, count/limit display) into `_renderKanbanColumns`, with a sensible default limit-per-stage configuration (hardcoded, informed by the legacy defaults' spirit — flagging known-bottleneck stages like `review`/`test-plan`).

**What will NOT be built:**
Any hard/enforced blocking behaviour. A settings UI for configuring limits.

**How each AC will be verified:**
| AC | Test approach | Type |
|----|---------------|------|
| AC1 | Unit test: over-limit column shows distinct badge | Unit |
| AC2 | Unit test: at/under-limit column shows plain badge | Unit |
| AC3 | Integration test: advance/drag into over-limit column still succeeds | Integration |
| AC4 | Unit test: limit convention consistent across board scopes | Unit |

**Assumptions:**
None beyond the sensible-default approach already specified in the story.

**Estimated touch points:**
Files: `src/web-ui/views/kanban-view.js` only.
Services: None new.
APIs: None new.
