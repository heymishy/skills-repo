## Unit Tests

### cardIsRealLinkWithCorrectIdentifier

- **Verifies:** AC1
- **Precondition:** The coding agent has confirmed the destination route and identifier mapping (documented in `decisions.md`)
- **Action:** Render a card via `_renderKanbanColumns`
- **Expected result:** The card is a real `<a>` element (not a `<div>`), `href` pointing to the confirmed destination with the correct identifier
- **Edge case:** No

---

## Integration Tests

### destinationShowsUsefulState

- **Verifies:** AC2
- **Components involved:** Whichever destination route is confirmed (e.g. `features.js`'s `/features/:slug` handler, or a journey-specific equivalent)
- **Precondition:** A card's journey with known current stage and artefact state
- **Action:** Request the destination page for this journey/feature
- **Expected result:** Response includes the current stage and a link to artefact files — matching `kfd1`'s existing detail-page content if that's the confirmed destination

### clearWayBackToBoard

- **Verifies:** AC3
- **Components involved:** Destination page rendering
- **Precondition:** Same as above
- **Action:** Parse the destination page's rendered HTML
- **Expected result:** A back-link/breadcrumb to the originating board is present

### crossTenantDetailAccessDenied404

- **Verifies:** AC4
- **Components involved:** Destination route, tenant-ownership check
- **Precondition:** A journey belonging to tenant B; request authenticated as tenant A
- **Action:** Request the detail view for tenant B's journey, as tenant A
- **Expected result:** HTTP 404 (matching this repo's existing FORBIDDEN-vs-NOT_FOUND policy, same as S1.1's AC4)
