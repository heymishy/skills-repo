## Unit Tests

### longTitleTruncatesAt48Chars

- **Verifies:** AC1
- **Precondition:** A card object with a 60-character title
- **Action:** Render the card via `_renderKanbanColumns`
- **Expected result:** Displayed title is 48 characters + ellipsis, using the existing `truncateTitle()` function (not a new implementation)
- **Edge case:** No

### shortTitleDisplaysUnmodified

- **Verifies:** AC2
- **Precondition:** A card object with a 30-character title
- **Action:** Render the card
- **Expected result:** Full 30-character title displayed, no truncation applied
- **Edge case:** No

### artefactBadgeShown_IF_dataAvailable

- **Verifies:** AC4 (conditional — only runs/asserts if the coding agent's investigation confirms data is obtainable)
- **Precondition:** A card with a known artefact count (e.g. 3)
- **Action:** Render the card
- **Expected result:** Badge text "3 artefacts" shown, matching `kfd1`'s existing badge format
- **Edge case:** 0-artefact case shows "no artefacts yet" (matching `kfd1`'s existing empty-state text)

## Integration Tests

### fullTitleAvailableViaHoverAttribute

- **Verifies:** AC3
- **Components involved:** Card rendering
- **Precondition:** A card with a truncated (>48 char) title
- **Action:** Parse the rendered HTML for a `title` attribute (or equivalent) on the card element
- **Expected result:** The full, untruncated title is present in that attribute

### artefactCountDataFetchIntegration

- **Verifies:** AC4 (conditional)
- **Components involved:** Whichever data-fetch mechanism the coding agent's investigation confirms (batched query or bounded per-journey calls)
- **Precondition:** A set of test journeys with varying artefact counts
- **Action:** Fetch board data for these journeys
- **Expected result:** Each card's artefact count matches its real, actual artefact count in the store

### titleTruncationShipsIndependentOfBadgeOutcome

- **Verifies:** AC5
- **Components involved:** Card rendering
- **Precondition:** Simulate the "artefact-count NOT obtainable" branch (e.g. stub the data-fetch to return null/unavailable)
- **Action:** Render a card with a long title under this condition
- **Expected result:** Title truncation (AC1) still applies correctly — this story's core value ships even if AC4's harder half doesn't
