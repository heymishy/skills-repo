## Unit Tests

### notReadyIndicatorHasTextLabelNotColourOnly

- **Verifies:** AC1
- **Precondition:** A not-ready card's rendered HTML
- **Action:** Parse the rendered markup for the not-ready indicator
- **Expected result:** A text label (not just a CSS class/colour) is present in the markup — e.g. "Discovery turn in progress", not merely a red border with no text
- **Edge case:** No

### readyCardShowsNoNotReadyIndicator

- **Verifies:** AC3
- **Precondition:** A ready card's rendered HTML
- **Action:** Parse the rendered markup
- **Expected result:** No not-ready indicator markup present; the Advance action (S1.1) IS present and visually distinguishable (different CSS class)
- **Edge case:** No

### validationFailureStyledDistinctlyFromNotReady

- **Verifies:** AC4
- **Precondition:** A card that just failed a real gate-confirm validation attempt (S1.1 AC5's scenario)
- **Action:** Compare the rendered failure-state markup/class names against the not-ready-state markup/class names
- **Expected result:** Different CSS classes / different text content — the two states are not visually or textually identical
- **Edge case:** No

---

## Integration Tests

### hoverRevealsDetailedReason

- **Verifies:** AC2
- **Components involved:** Card rendering, tooltip/detail-expansion mechanism
- **Precondition:** A not-ready card
- **Action:** Trigger the hover/focus detail mechanism (server-rendered `title` attribute or client-side expand, per implementation choice)
- **Expected result:** Additional detail text is available (e.g. via the element's `title` attribute content, or an expandable detail element's content) beyond the always-visible short label

---

## E2E Tests

### notReadyIndicatorVisibleAndKeyboardFocusable

- **Verifies:** AC1, AC2 (keyboard-accessibility of the detail mechanism)
- **Precondition:** A not-ready card on a real rendered board
- **Action:** Tab (keyboard) to the not-ready card in a real browser (Playwright)
- **Expected result:** The card receives visible keyboard focus, and the detail mechanism (tooltip/expansion) is reachable without a mouse
- **Edge case:** No

---

## NFR Tests

### notReadyIndicatorAccessibility

- **NFR addressed:** Accessibility
- **Measurement method:** Same as Unit Test `notReadyIndicatorHasTextLabelNotColourOnly` (text label present, not colour-only) plus the E2E keyboard-focus test above
- **Pass threshold:** Text label present in markup; detail mechanism keyboard-reachable
- **Tool:** Node test runner + Playwright
