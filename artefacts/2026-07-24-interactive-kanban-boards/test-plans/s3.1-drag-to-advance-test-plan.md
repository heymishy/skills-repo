## E2E / browser-layout detection (Step 3a)

**All of AC1-AC4 flagged:** drag-and-drop is explicitly named in this skill's trigger patterns ("which element is the drop target depends on CSS layout"). A DOM-simulation environment (jsdom) cannot compute real drop-target resolution from pointer coordinates.

**Handling decision:** E2E browser test (Playwright, already configured per ADR-018 — `tests/e2e/`, local `NODE_ENV=test` harness) — option 1 from Step 3a, not manual-only, since the required tooling already exists in this repo. AC5 (keyboard-only alternative remains available) is NOT layout-dependent — it's a simple presence/functional check, testable at Integration level.

## Integration Tests

### clickAlternativeStillWorksAlongsideDrag

- **Verifies:** AC5
- **Components involved:** S1.1's Advance action endpoint (reused, unmodified)
- **Precondition:** A ready card, drag-and-drop feature deployed
- **Action:** Call S1.1's click-action endpoint directly (bypassing drag entirely)
- **Expected result:** Identical success outcome to S1.1's own AC1 test — this story's addition doesn't remove or alter the click path

---

## E2E Tests

### dragReadyCardToValidNextColumnAdvances

- **Verifies:** AC1
- **Precondition:** A ready card, its single valid next-stage column visible on the same board
- **Action:** Real browser drag-and-drop (Playwright's `dragTo()` or manual pointer down/move/up sequence) from the card's current column to its valid next column
- **Expected result:** After the drop, the card's real stage has advanced (confirmed via a follow-up API check, not just visual position) — matches S1.1's AC1 outcome exactly
- **Edge case:** No

### dragToInvalidColumnRejectedClientSide

- **Verifies:** AC2
- **Precondition:** A ready card, a column that is NOT its valid next stage
- **Action:** Drag the card onto the invalid column and drop
- **Expected result:** No server call is made (verified via network request interception — zero requests to the advance endpoint); the card visually reverts to its original column
- **Edge case:** No

### dragNotReadyCardRejectedWithExplanation

- **Verifies:** AC3
- **Precondition:** A not-ready card (S1.1's fixture)
- **Action:** Attempt to drag the card to its would-be next column
- **Expected result:** Either the drag doesn't initiate, or it initiates and is rejected on drop with S1.2's not-ready explanation shown — card reverts
- **Edge case:** No

### dragValidationFailureRevertsWithRealReason

- **Verifies:** AC4
- **Precondition:** A ready card whose artefact will fail validation (S1.1's AC5 fixture)
- **Action:** Drag the card to its valid next column and drop
- **Expected result:** Server call is attempted (unlike AC2/AC3), fails with the real validation reason (matching S1.1's AC5), card reverts, real error text shown — not a generic "drop failed" message
- **Edge case:** No

- **Tool:** Node test runner
