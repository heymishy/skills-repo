# Decisions: Ideate Canvas Turn-2 Render Fix

## DECISION — Root cause is a client-side auto-continuation heuristic mismatched to /ideate's turn shape, not a canvas dedup bug or a fixture-authoring gap (2026-07-23)

**Context:** `tests/e2e/a3-product-feature-ideate-canvas.spec.js`'s AC3 ("2 ideation turns against the deterministic mock fixture render and update the canvas") failed on real `wuce-staging` in PR #568's "Scenario A E2E (staging)" CI gate (run 29996127983, job 89170114731): `Expected: > 30, Received: 30` — turn 1 alone produced 30 canvas blocks, and turn 2's genuine submission produced no further growth within the 20-second poll window. This story was dispatched to find the real root cause among three hypotheses: (a) a fixture/test-authoring gap (fixture doesn't vary per turn), (b) a mock-gateway wiring gap (wrong fixture/scenario selected per turn), or (c) a genuine canvas-rendering/dedup defect that would also affect real, non-mocked usage.

**Investigation finding:** None of (a)/(b)/(c) exactly, though the effect matches (c)'s severity. Read `tests/e2e/fixtures/llm-gateway/ideate.success.json` (confirmed genuinely static/deterministic by design — `mock-llm-gateway.js`'s own doc comment: "identical (stage, model, scenarioName) always returns an equal response object (AC1)", which is correct, intentional behaviour, not a bug). Read `parseCanvasBlock` and `appendCanvasBlock`/`renderCanvasBlock` (`src/web-ui/routes/skills.js`, `chat-view.js`) — confirmed NO de-duplication logic exists anywhere in the canvas-rendering pipeline; every parsed `---CANVAS-JSON:---` marker is unconditionally pushed and appended as a new `.canvas-block` element. This is correct, intentional design (canvas blocks are meant to accumulate across turns) — not the defect.

The actual defect: `_renderChatPage`'s inline client script (`sendTurn` function) auto-fires a hidden "continue" turn whenever a turn's streamed text has no literal "?" character AND no `---ARTEFACT-START---`/`---ARTEFACT-END---` marker (added in commit `a10b32a3`, 2026-05-06, "artefact generation — no hold turns + auto-nudge client when model announces without asking"). Confirmed via `git log -S` that this heuristic was purpose-built for artefact-generating skills whose system prompt (`buildSystemPrompt`'s "ARTEFACT GENERATION" instruction, same commit) guarantees every turn ends in either the artefact or a literal question — and confirmed via `server.js`'s own test-mode comment that "discovery/design/definition/etc. success fixtures all contain `---ARTEFACT-START---`/`---ARTEFACT-END---` markers", i.e. `ideate.success.json` is the ONLY fixture that legitimately lacks one, because `/ideate` is the only skill that is conversational/canvas-based rather than artefact-producing per-turn.

Applying the heuristic to ideate anyway fires an unbounded chain of hidden "continue" turns against the identical static fixture content: each iteration re-parses the same `---CANVAS-JSON:---` marker and pushes another duplicate `.canvas-block` (correct dedup-free design, misapplied against runaway repetition), while the submit button stays disabled indefinitely (`submitBtn.disabled` is only reset by the "done" branch or the "text contains '?'" branch — the ideate fixture satisfies neither). This is confirmed to be the root cause by a jsdom-driven regression test (`tests/check-icv-s1-ideate-canvas-turn2-render-fix.js`) that reproduces the exact pattern pre-fix (5 executor calls / 5 duplicate canvas blocks within a 600ms settle window — the same shape as the real CI log's "30", just a shorter observation window) and eliminates it post-fix (exactly 1 call/block per genuine turn).

**Decision:** Gate the `a10b32a3` auto-continue nudge behind `!IS_IDEATE` — a one-line conditional change to an existing, already-in-scope, server-computed flag:
```js
} else if (!IS_IDEATE && streamText && streamText.indexOf("?") === -1) {
  _continuationPending = true;
  setTimeout(function(){ _continuationPending = false; sendTurn("continue", true); }, 100);
} else {
  // ... existing re-enable-input branch, now also ideate's normal path ...
}
```
No change to `mock-llm-gateway.js`, `skill-turn-executor.js`, `ideate.success.json`, or any canvas-rendering/parsing code — all of that was already correct.

**Rationale:**
1. **`IS_IDEATE` is already computed and in scope in the exact function that needs it** (used elsewhere in the same script for `updateDraftPanel`'s ideate-specific branch) — this is additive, minimal-surface-area gating on an existing signal, not a new mechanism.
2. **The `a10b32a3` nudge remains fully intact for the skills it was built for.** Verified by a contrast-case test (AC4/T4): a `discovery`-shaped turn with no "?" and no artefact marker still auto-fires exactly one hidden continuation, which still correctly completes the session when the continuation produces the artefact. This fix is scoped, not a blanket regression.
3. **TDD discipline followed**: the new test was run against the pre-fix code FIRST (via `git stash` isolating the one-line fix), confirming RED for the exact reason the CI failure describes (executor call count / canvas block count both >1 for a single logical turn), THEN the fix was applied and the same test confirmed GREEN.
4. **This is a genuine production defect, not merely a test-environment artifact.** Any real, non-mocked `/ideate` turn whose response happens not to end in a literal "?" character — plausible, ordinary phrasing for a conversational, suggestion-ending turn — would trigger the identical unbounded chain against the real Anthropic API in production: burning real turn-credits and tokens with no bound, and leaving the real user's chat input disabled indefinitely. The mocked E2E fixture did not create this bug; it correctly modeled realistic ideate output and, in doing so, surfaced a latent bug that real usage was already exposed to (masked in casual testing only by real model output usually — not reliably — ending turns with "?").

**Testing note (harness pitfall found and corrected):** The AC4 contrast case necessarily produces a real `---ARTEFACT-START---`/`---ARTEFACT-END---` turn, which the real handler auto-saves to disk at `artefacts/<derived-slug>/discovery.md`. Without an explicit `---SLUG---` override, the handler's fallback slug (`<today's-date>-discovery`) collided with — and overwrote — a real, already-committed artefact directory (`artefacts/2026-07-23-discovery/discovery.md`) during initial test development. Caught immediately via `git diff`/`git status`, and the file's prior committed content was unaffected once `git checkout --` was run (confirmed clean). Fixed by pinning an explicit, obviously-disposable test slug (`zzz-test-icv-s1-disposable-artefact`) in the test's stubbed artefact text, with a `finally`-block cleanup (`fs.rmSync`) removing that directory after each run. Flagging this pattern for any future test that stubs an artefact-producing turn: always pin a `---SLUG---` override rather than relying on the handler's date-based fallback, which is not test-isolated by construction.

**Consequence:** `/ideate` turns now correctly complete as single, self-terminating exchanges — matching the feature's actual conversational design — with no hidden continuation chain, no unbounded canvas duplication, and no risk of the input staying disabled after a normal turn. This closes the AC3 CI-blocking failure and very likely (though not independently confirmed by this story) also explains `a4-ideate-session-resume.spec.js`'s co-failing AC2/AC3 in the same CI run, since a4's own `afterTurn1Count` baseline is built from the same runaway-loop-affected turn-1 canvas count.

**Source:** `icv-s1` investigation and implementation, 2026-07-23; corroborating investigation: `src/web-ui/routes/skills.js` (`_renderChatPage`'s inline script, `handlePostTurnStreamHtml`, `parseCanvasBlock`), `src/web-ui/modules/mock-llm-gateway.js`, `src/modules/skill-turn-executor.js`, `tests/e2e/fixtures/llm-gateway/ideate.success.json`, `git log -S'indexOf("?") === -1'` (commit `a10b32a3`), real CI log via `gh run view 29996127983 --job 89170114731 --log-failed`, new regression test `tests/check-icv-s1-ideate-canvas-turn2-render-fix.js`.
