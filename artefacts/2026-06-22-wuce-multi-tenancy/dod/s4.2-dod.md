## Operator decision — Solo mode for initial beta

`TENANT_ORG_ALLOWLIST` is intentionally left empty for the initial beta. In solo mode, each GitHub user who authenticates becomes their own tenant (`tenantId = req.session.login`). No org-based access control is applied at this stage. This is a deliberate operator choice, not a deferred item — the allowlist mechanism is code-complete and can be activated by setting the secret when org-based beta access is required.

ACs that require an org allowlist configured (AC1–AC4) are marked N/A with this rationale.

## When to activate org-based mode

Set `TENANT_ORG_ALLOWLIST=<org-name>` as a Fly.io secret and create `$WUCE_TENANT_ROOT_BASE/<org-name>/` directory via `fly ssh console` when the first org-based beta user is onboarded. No code changes required — the mechanism is already live.
