### 8-L2 — LOW | Category C: AC8 references an unresolved N for disk write point count

**Location:** `artefacts/2026-06-22-wuce-multi-tenancy/stories/p5.1.md`, AC8.

AC8 states: "Given the Phase 2 implementation (p2.1) introduced N distinct disk write operations whose path is derived from tenantId, When the adversarial test suite runs, Then each of the N write points has at least one adversarial test case." The value of N is not known at definition time — it is determined by the p2.1 implementation.

This is structurally sound (the story cannot know N before p2.1 is implemented), but it means AC8 cannot be independently verified from the story alone at DoR time. The DoR for p5.1 must include a pre-condition: "The p2.1 implementation plan or its post-implementation notes have enumerated the N distinct disk-write-from-tenantId points and that list is available to the implementer."

**Recommended action:** Add to the Dependencies section: "The list of N disk write points derived from `tenantId` must be enumerated in the p2.1 implementation notes or DoD artefact before p5.1's DoR is signed off." Non-blocking for /test-plan.
