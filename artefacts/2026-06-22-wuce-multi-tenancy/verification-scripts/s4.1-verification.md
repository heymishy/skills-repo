## Setup

**Before you start:**
1. Set `APP=<your-fly-app-name>` in your shell.
2. Confirm fly CLI is authenticated: `fly auth whoami`.
3. Confirm all secrets are set (run Scenario 1 first before proceeding).
4. Have a GitHub account that is a member of an allowlisted org ready for OAuth login.

**Quick automated run:** `APP=<name> bash scripts/smoke-test-production.sh` covers Scenarios 1, 2, and 3 automatically and provides guided prompts for the rest.

**Reset between scenarios:** Scenarios 4–6 share a live session — log out between scenarios if re-running.

### Scenario 4: Journey created via UI persists to Neon (AC4)

**Covers:** AC4

**Steps:**
1. Open `https://$APP.fly.dev` in a browser and log in with a GitHub account from an allowlisted org.
2. Create a new journey via the UI (click the button to start a new journey — use any feature name).
3. Note the journey ID shown in the URL or UI.
4. Open the Neon SQL editor at console.neon.tech.
5. Run: `SELECT journey_id, tenant_id FROM journeys ORDER BY created_at DESC LIMIT 1;`

**Expected outcome:**
> A row appears in the Neon SQL result. The `journey_id` matches the journey you just created. The `tenant_id` matches the GitHub org login of your account.
