## Test Data Strategy

**Source:** Synthetic in-line — `makeReq()` factory with 3-field session shape; in-memory journey store seeded per test
**PCI/sensitivity in scope:** No
**Availability:** Available now — handlers are pure functions called with synthesised req/res objects
**Owner:** Self-contained

### Data requirements per AC

| AC | Data needed | Source | Sensitive fields | Notes |
|----|-------------|--------|-----------------|-------|
| AC1–AC9 | Two users (`alice`, `bob`); one journey owned by `bob`; `alice`'s session (3-field: `accessToken`, `userId`, `login`) | Inline synthetic | None | Session shape: `{ accessToken: 'tok-alice', userId: '1', login: 'alice' }` — no tenantId (Phase 0) |
| AC10–AC11 | Same two-user setup | Inline synthetic | None | These are mutating handlers — assert 403 not 404 |
| AC12 | Journey owned by `alice`; `alice`'s session | Inline synthetic | None | Assert response is the same as current behaviour (not 4xx) |
| AC13 | Journey with `ownerId: null`; any authenticated session | Inline synthetic | None | Assert no 4xx — passes through |

### Test fixture pattern

Each handler test uses `freshRequire()` for module isolation and a seeded journey store:
```
const makeReq = (overrides) => ({ session: { accessToken: 'tok', userId: '1', login: 'alice' }, params: {}, body: {}, ...overrides });
const makeRes = () => { let status, body; return { status(s){ status=s; return this; }, json(b){ body=b; }, redirect(s){ status=s; }, statusCode: null, _get: () => ({ status, body }) }; };
```

## Integration Tests

### handleGetJourneyState — cross-user call returns 404

- **Verifies:** AC1
- **Components involved:** `journey.js` handler, `journey-access.js` guard, in-memory journey store
- **Precondition:** Journey `{ journeyId: 'j1', ownerId: 'bob' }` in store; `alice`'s 3-field session
- **Action:** Call handler with `makeReq({ params: { journeyId: 'j1' }, session: { ...alice } })`
- **Expected result:** `res.status` called with `404`; response body contains no journey state data

## NFR Tests

### No latency regression for authorized callers (synchronous guard, no I/O)

- **NFR addressed:** Performance — "No performance regression for authorized callers — the guard is a synchronous check with no I/O"
- **Measurement method:** Assert `requireJourneyAccess` is not async (does not return a Promise); confirm guard completes within the synchronous call frame
- **Pass threshold:** No `await` in guard code path; handler response time unchanged
- **Tool:** Code inspection + plain `assert` (runtime timing not required at unit level)

## Out of Scope for This Test Plan

- Routes outside `src/web-ui/routes/journey.js` (skills.js, auth.js) — AC out-of-scope
- Tenant-level isolation (`isSameTenant` real enforcement) — Phase 1; `isSameTenant` returns true throughout Phase 0
- Response body format for authorized callers — ADR-024 shape is validated separately; this plan asserts only the 4xx/redirect for cross-user callers
- Path-traversal guards in journey.js — must not be removed; verified by code review at DoR, not this test plan
