## Unit Tests

### isSameTenant — returns false when tenantIds differ

- **Verifies:** AC2
- **Precondition:** `journey = { tenantId: 'org-a' }`, `session = { tenantId: 'org-b' }`
- **Action:** `assert.strictEqual(isSameTenant(journey, session), false)`
- **Expected result:** `false`
- **Edge case:** No

### isSameTenant — returns true when tenantIds match

- **Verifies:** AC3
- **Precondition:** `journey = { tenantId: 'org-a' }`, `session = { tenantId: 'org-a' }`
- **Action:** `assert.strictEqual(isSameTenant(journey, session), true)`
- **Expected result:** `true`
- **Edge case:** No

### isSameTenant — returns true when journey.tenantId is null (Phase 0 passthrough)

- **Verifies:** AC4
- **Precondition:** `journey = { tenantId: null }`, `session = { tenantId: 'org-a' }`
- **Action:** `assert.strictEqual(isSameTenant(journey, session), true)`
- **Expected result:** `true`
- **Edge case:** Yes — null on journey side

### isSameTenant — returns true when session.tenantId is undefined (Phase 0 passthrough)

- **Verifies:** AC4
- **Precondition:** `journey = { tenantId: 'org-a' }`, `session = { tenantId: undefined }`
- **Action:** `assert.strictEqual(isSameTenant(journey, session), true)`
- **Expected result:** `true`
- **Edge case:** Yes — undefined on session side

### isSameTenant — returns true when both tenantIds are absent

- **Verifies:** AC4
- **Precondition:** `journey = {}`, `session = {}`
- **Action:** `assert.strictEqual(isSameTenant(journey, session), true)`
- **Expected result:** `true`
- **Edge case:** Yes — fully absent (single-user mode)

## Integration Tests

### createJourney — stores tenantId from session on new journey

- **Verifies:** AC1
- **Components involved:** `createJourney()` function, in-memory journey store
- **Precondition:** Session `{ accessToken: 'tok', userId: '1', login: 'alice', tenantId: 'my-org' }` (4-field shape)
- **Action:** Call `createJourney({ session, featureSlug: 'test-feature' })`
- **Expected result:** Returned journey object has `tenantId === 'my-org'`; journey stored in the store also has `tenantId`
- **Expected result (disk, if disk adapter active):** `journey.tenantId` written to disk; `accessToken` not in disk file (NFR-sec-no-accesstoken-disk preserved)

### Cross-tenant read returns 404 — isSameTenant real enforcement

- **Verifies:** AC6
- **Components involved:** `handleGetJourneyState` (or any read handler from p0.2), `requireJourneyAccess`, `isSameTenant`
- **Precondition:** Journey `{ journeyId: 'j1', ownerId: 'bob', tenantId: 'org-b' }` in store; session `{ accessToken: 'tok', userId: '1', login: 'alice', tenantId: 'org-a' }` (4-field)
- **Action:** Call read handler with alice's 4-field session for bob's journey
- **Expected result:** HTTP 404 — `isSameTenant` returns `false` → FORBIDDEN → TENANT policy → 404
- **Edge case:** Also assert same-tenant access still works: session `tenantId = 'org-b'` for the same journey → no 4xx

### Pre-existing test suite regression check

- **Verifies:** AC7
- **Components involved:** All `tests/check-*.js` files
- **Precondition:** p1.2 implementation complete; only Phase-1-relevant test files updated to 4-field shape
- **Action:** Run `node tests/check-*.js` (or equivalent glob runner)
- **Expected result:** All tests that passed before p1.2 still pass; exit code 0

## NFR Tests

### accessToken not written to disk when tenantId added to session format

- **NFR addressed:** Security — NFR-sec-no-accesstoken-disk ("strip-before-write invariant must be preserved when tenantId is added to the disk session format")
- **Measurement method:** After creating a journey with a 4-field session, read the disk session file; assert `accessToken` key is absent
- **Pass threshold:** `JSON.parse(fs.readFileSync(sessionFilePath)).accessToken === undefined`
- **Tool:** Plain Node.js `assert` + `fs.readFileSync`

### Session cookie shape unchanged

- **NFR addressed:** Backward compatibility — "no change to session cookie shape or HttpOnly/SameSite attributes"
- **Measurement method:** Code inspection at DoR — verify no changes to cookie middleware configuration
- **Pass threshold:** Cookie attributes unchanged in `server.js` / session middleware config
- **Tool:** DoR code review checklist item
