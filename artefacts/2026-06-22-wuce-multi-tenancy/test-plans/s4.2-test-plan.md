## NFR Tests

### NFR-SEC-1 — TENANT_ORG_ALLOWLIST value not disclosed in rejection message

- **NFR addressed:** Access control (non-disclosure of allowlist contents)
- **Measurement method:** AC2 scenario — inspect the 403 response body for the exact message; confirm it does not contain org names from the allowlist
- **Pass threshold:** 403 response body contains exactly "You are not a member of an authorised organisation." and does not contain any org name from the allowlist
- **Tool:** curl or browser dev tools (manual)

### NFR-SEC-2 — TENANT_ORG_ALLOWLIST not committed

- **NFR addressed:** Security
- **Measurement method:** `git grep "TENANT_ORG_ALLOWLIST=" -- ':!artefacts/' ':!workspace/'` returns zero results containing an actual org name value
- **Pass threshold:** Zero grep hits
- **Tool:** git grep (manual)

## Out of Scope for This Test Plan

- Self-service tenant registration
- Automated tenant directory creation
- Per-tenant feature flag configuration beyond journey cap
- Testing with more than 2 concurrent tenants (beta scale)
