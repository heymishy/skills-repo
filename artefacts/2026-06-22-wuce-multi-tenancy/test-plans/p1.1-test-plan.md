## Unit Tests

### OAuth scope includes read:org when TENANT_ORG_ALLOWLIST is set

- **Verifies:** AC1
- **Precondition:** `process.env.TENANT_ORG_ALLOWLIST = 'org-a'`
- **Action:** Call the function that constructs the OAuth authorize URL
- **Expected result:** Returned URL string includes `read:org` in the scope parameter alongside `repo,read:user`
- **Edge case:** Also test that `read:org` is absent when `TENANT_ORG_ALLOWLIST` is not set (AC6 overlap)

### Single-page org resolution — first allowlist match wins by allowlist order

- **Verifies:** AC2
- **Precondition:** `setFetchOrgs(async () => [{ login: 'org-b' }, { login: 'org-a' }])`; allowlist = `'org-a,org-b'`
- **Action:** Run the OAuth callback org-resolution function
- **Expected result:** `session.tenantId === 'org-a'` (first in allowlist, even though org-b appeared first in the orgs response)
- **Edge case:** No

### Multi-page pagination — match on page 2 is found

- **Verifies:** AC3
- **Precondition:** `setFetchOrgs(async (page) => page === 1 ? { orgs: [{ login: 'org-b' }], nextPage: 2 } : { orgs: [{ login: 'org-a' }], nextPage: null })`; allowlist = `'org-a'`
- **Action:** Run org resolution
- **Expected result:** All pages fetched; `session.tenantId === 'org-a'`
- **Edge case:** Yes — match is only on the second page; ensures full pagination is implemented

### Zero-match — session not created; 4xx returned

- **Verifies:** AC4
- **Precondition:** `setFetchOrgs(async () => [{ login: 'org-x' }])`; allowlist = `'org-a'`
- **Action:** Run org resolution; capture response
- **Expected result:** Response status is 4xx (HTTP 400 or 403); session not set; error message indicates the user is not a member of an allowed organisation
- **Edge case:** No

### Zero-match — error message does not expose allowlist contents

- **Verifies:** AC4 + NFR (information disclosure)
- **Precondition:** Same as above; allowlist = `'org-a,org-b'`
- **Action:** Capture the error response body
- **Expected result:** Body does not contain `'org-a'` or `'org-b'` — allowlist names are not disclosed
- **Edge case:** Yes — security edge case

### Multi-match — first in allowlist order wins over orgs response order

- **Verifies:** AC5
- **Precondition:** `setFetchOrgs(async () => [{ login: 'org-b' }, { login: 'org-a' }])`; allowlist = `'org-a,org-b'` (org-a first)
- **Action:** Run org resolution
- **Expected result:** `session.tenantId === 'org-a'` not `'org-b'`
- **Edge case:** Yes — disambiguation rule

### Absent TENANT_ORG_ALLOWLIST — no fetch, tenantId not set, backward-compatible

- **Verifies:** AC6
- **Precondition:** `delete process.env.TENANT_ORG_ALLOWLIST`; org-fetch spy to detect calls
- **Action:** Run the OAuth callback
- **Expected result:** Org-fetch spy not called; `req.session.tenantId` not set; login flow proceeds normally
- **Edge case:** Also test empty string value (`TENANT_ORG_ALLOWLIST = ''`) — must behave same as absent

### Default fetchOrgs stub throws when not wired (D37)

- **Verifies:** AC7
- **Precondition:** Module freshly required without calling `setFetchOrgs()`
- **Action:** Call the internal `_fetchOrgs()` directly (via module internals or by triggering a code path that calls it with TENANT_ORG_ALLOWLIST set)
- **Expected result:** Throws `Error('Adapter not wired: fetchOrgs. Call setFetchOrgs() with a real implementation before use.')` — exact message match
- **Edge case:** No

## NFR Tests

### Org fetch uses async/await — does not block event loop

- **NFR addressed:** Performance — "must not block the event loop with synchronous waits"
- **Measurement method:** Assert that the org-resolution function returns a Promise (is async); assert `await` is used internally (code inspection at DoR)
- **Pass threshold:** Function is async; no synchronous `while` or polling loop
- **Tool:** Code inspection at DoR + `assert.ok(result instanceof Promise)` at call site

### Total org fetch completes within 5 seconds

- **NFR addressed:** Performance — "total org fetch must complete within 5 seconds"
- **Measurement method:** In the test, use a mock that resolves after a simulated delay; verify no timeout error for a 3-page response
- **Pass threshold:** Resolves within 5000ms for ≤10 pages of synthetic responses
- **Tool:** Plain `assert` with `Promise.race([orgFetch(), timeout(5000)])`

### Warning logged if fetch exceeds 3 seconds

- **NFR addressed:** Observability — "log a warning if it exceeds 3 seconds"
- **Measurement method:** Inject a mock that resolves after 3100ms; capture log output; assert warning is emitted
- **Pass threshold:** Warning log entry present; no error thrown
- **Tool:** Log spy in test
