None. Secret values are never read or logged — only their presence in `fly secrets list` output is checked.

### Gaps

All manual ACs (AC1 partial, AC2, AC4, AC5, AC6) require a live Fly.io deployment with real credentials. No local automation is possible. `scripts/smoke-test-production.sh` automates the secrets presence check (AC1) and health check (AC3) and provides guided prompts for all manual steps.

## Integration Tests

> **TDD note:** T1–T3 are implemented by `scripts/smoke-test-production.sh`. They fail (or cannot run) until the app is deployed. The "failing" state is: app not yet deployed / env vars not set.

### T1 — Required secrets present in fly secrets list

- **Verifies:** AC1
- **Components involved:** smoke-test-production.sh → `fly secrets list`
- **Precondition:** `APP=<fly-app-name>` set; fly CLI authenticated
- **Action:** `APP=<name> bash scripts/smoke-test-production.sh` (AC1 section)
- **Expected result:** All 7 secrets (DATABASE_URL, UPSTASH_REDIS_REST_URL, UPSTASH_REDIS_REST_TOKEN, TENANT_ORG_ALLOWLIST, MAX_JOURNEYS_PER_TENANT, GITHUB_CLIENT_ID, GITHUB_CLIENT_SECRET) appear as `[PASS]` lines

### T2 — No error lines in startup logs

- **Verifies:** AC2
- **Components involved:** smoke-test-production.sh → `fly logs` last 60 lines
- **Precondition:** App deployed and running; T1 passed
- **Action:** AC2 section of smoke-test-production.sh
- **Expected result:** `[PASS] No startup error log lines found` — no lines matching `PG write error|loadAllFromPg failed|Redis load error`

### T3 — GET /health returns 200

- **Verifies:** AC3
- **Components involved:** smoke-test-production.sh → curl → Fly.io app /health endpoint
- **Precondition:** App deployed and running
- **Action:** AC3 section of smoke-test-production.sh
- **Expected result:** `[PASS] GET /health returned 200`

## Out of Scope for This Test Plan

- Load testing or concurrent-user testing
- Automated redeploy CI/CD pipeline
- Rollback procedure beyond `fly machine restart`
