# Story s0.1 — Resume guard + ownerId/tenantId disk persistence

**Feature:** wuce-multi-tenancy
**Epic:** sprint-0-tenant-fixes
**Status:** Complete
**Complexity:** 2
**Scope stability:** Stable

## User story

As an operator,
I want `handleGetJourneyResume` to enforce tenant/owner access control and for ownerId + tenantId to survive a server restart,
So that cross-tenant users cannot access each other's journeys via the resume endpoint, and the isolation holds even after a process restart.

## Metric linkage

- **M1** (Authorization coverage rate): s0.1 closes the resume-endpoint gap, bringing cross-endpoint coverage to 100%.
- **T3-M1** (KNOWN BUG closure): cross-tenant resume bypass is the outstanding bug this story fixes.

## Acceptance criteria

**AC1** — Owner can resume their own journey
Given a journey with `ownerId: 'alice'` and `tenantId: 'alice'` exists on disk,
When alice (session: `login: 'alice'`, `tenantId: 'alice'`) calls `GET /journey/:slug/resume`,
Then the handler does not return 404 or 403.

**AC2** — Cross-tenant user is blocked
Given a journey with `ownerId: 'alice'` and `tenantId: 'alice'` exists on disk,
When bob (session: `login: 'bob'`, `tenantId: 'bob'`) calls `GET /journey/:slug/resume`,
Then the handler returns 404.

**AC3** — ownerId and tenantId are saved to disk
Given `journeyStore.createJourney()` is called followed by `setJourneyFields({ ownerId, tenantId })`,
When `journeyDisk.loadJourney()` is called on the resulting file,
Then `diskJourney.ownerId` and `diskJourney.tenantId` are present and correct.

**AC4** — loadAllFromDisk restores ownerId and tenantId
Given journeys with `ownerId` and `tenantId` exist on disk,
When the server restarts (`journeyStore._clear()` + `loadAllFromDisk()`),
Then the in-memory journey has correct `ownerId` and `tenantId`.

**AC5** — Unauthenticated resume redirects
Given no `accessToken` in session,
When `GET /journey/:slug/resume` is called,
Then the handler returns 302.

## Implementation touchpoints

- `src/web-ui/routes/journey.js`: `handleGetJourneyResume` — access guard added after disk load; `handlePostJourney` — `setJourneyFields` used instead of direct mutation to persist ownerId/tenantId
- `src/web-ui/modules/journey-store.js`: `loadAllFromDisk` — restores `ownerId` and `tenantId` from disk format
- `tests/check-s0.1-resume-guard.js`: verification script for all 5 ACs
