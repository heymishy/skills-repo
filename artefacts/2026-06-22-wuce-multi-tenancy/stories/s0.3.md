# Story s0.3 — Journey list filtered by tenantId

**Feature:** wuce-multi-tenancy
**Epic:** sprint-0-tenant-fixes
**Status:** Complete
**Complexity:** 1
**Scope stability:** Stable

## User story

As an operator,
I want the journey list (`GET /journey`) to show only journeys belonging to my tenant,
So that cross-tenant journeys are never visible in the UI, and existing pre-s0.1 journeys remain accessible during the upgrade.

## Metric linkage

- **M2** (Tenant isolation correctness): s0.3 closes the journey-list cross-tenant leak found in the p0.2 PR review (deferred then as "Option A — defence in depth").

## Acceptance criteria

**AC1** — alice sees only her own journeys
Given journeys for tenantId=alice, tenantId=bob, and tenantId=myorg exist,
When alice (session tenantId=alice) calls `GET /journey`,
Then only alice's journey appears; bob's and org's do not.

**AC2** — bob sees only his own journeys
Given the same three journeys,
When bob (session tenantId=bob) calls `GET /journey`,
Then only bob's journey appears.

**AC3** — org user sees only org journeys
Given the same three journeys,
When an org user (session tenantId=myorg) calls `GET /journey`,
Then only the org journey appears.

**AC4** — No tenantId in session: all journeys shown (backward compat)
Given a session with no tenantId (created before s0.2 was deployed),
When `GET /journey` is called,
Then all journeys are shown (no filter applied — backward compat for pre-s0.2 sessions).

**AC5** — Unauthenticated: 302 redirect
Given no `accessToken` in session,
When `GET /journey` is called,
Then the handler returns 302.

## Migration grace

For sessions with a tenantId (all sessions after s0.2 deploy), the filter also passes null-tenantId journeys whose `ownerId` matches the current login, or where `ownerId` is also null. This ensures pre-s0.1 journeys on disk (which have neither field) remain accessible to their original creator after the upgrade.

## Implementation touchpoints

- `src/web-ui/routes/journey.js`: `handleGetJourney` — tenant filter added with migration-grace logic for null-tenantId journeys
- `tests/check-s0.3-journey-list-filter.js`: verification script for all 5 ACs
