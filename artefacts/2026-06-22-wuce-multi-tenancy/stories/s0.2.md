# Story s0.2 — tenantId = session.login fallback for individual users

**Feature:** wuce-multi-tenancy
**Epic:** sprint-0-tenant-fixes
**Status:** Complete
**Complexity:** 1
**Scope stability:** Stable

## User story

As a solo operator or sporadic individual beta user (not part of a GitHub org),
I want my session to be assigned a tenantId equal to my GitHub login when no org allowlist is configured,
So that my journeys are isolated from other users' journeys without requiring GitHub org membership.

## Metric linkage

- **M2** (Tenant isolation correctness): s0.2 ensures every authenticated session has a tenantId, enabling s0.3's filter to work correctly for all sessions.

## Acceptance criteria

**AC1** — No TENANT_ORG_ALLOWLIST: tenantId set to user.login
Given `TENANT_ORG_ALLOWLIST` is not set (or empty),
When a user completes the OAuth callback,
Then `session.tenantId` is set to `user.login` and `session.login` is set to `user.login`.

**AC2** — TENANT_ORG_ALLOWLIST set + user in org: tenantId = org slug (unchanged)
Given `TENANT_ORG_ALLOWLIST` is set and the user is a member of a listed org,
When a user completes the OAuth callback,
Then `session.tenantId` is set to the org login (not `user.login`).

**AC3** — TENANT_ORG_ALLOWLIST set + user not in org: 403 (unchanged)
Given `TENANT_ORG_ALLOWLIST` is set and the user is not a member of any listed org,
When a user completes the OAuth callback,
Then the handler returns 403 and `session.tenantId` is not set.

## Migration-safety note

The `tenantId = user.login` assignment is in the `else` branch of the `if (_allowlist.trim())` conditional. Org-based tenant assignment (in the `if` branch) is entirely unaffected. Deployments with `TENANT_ORG_ALLOWLIST` set continue to use org-based tenantIds as before. Deployments without it (individual operators) now get per-user isolation automatically.

## Implementation touchpoints

- `src/web-ui/routes/auth.js`: `handleAuthCallback` — changed to use module reference pattern (`_oauthAdapter.*`) instead of destructuring, enabling test monkeypatching; added `else { req.session.tenantId = user.login; }` branch
- `tests/check-s0.2-tenant-login-fallback.js`: verification script for all 3 ACs
