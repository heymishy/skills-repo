# Story s4.3 — Default executor provider to Anthropic; copilot remains opt-in

**Feature:** wuce-multi-tenancy
**Epic:** sprint-4-beta-deployment
**Discovery:** artefacts/2026-06-22-wuce-multi-tenancy/discovery.md
**Benefit-metric:** artefacts/2026-06-22-wuce-multi-tenancy/benefit-metric.md
**Status:** Definition-of-Done (retroactive — implemented as production hotfix during s4.1 execution)
**Complexity:** 1
**Scope stability:** Stable

## Context

Discovered during s4.1 beta deployment: with `SKILL_EXECUTOR_PROVIDER` unset, the code defaulted to `copilot`, which returned HTTP 400 `model_not_supported` for `claude-sonnet-4-6`. The Copilot Chat Completions API does not support Claude 4.x model IDs. Anthropic direct is the correct default — it enforces `budget_tokens` correctly (noted in `.env` comments; GHCP does not), supports the full Claude model family, and is already wired in production via `ANTHROPIC_API_KEY`.

## User story

As the operator,
I want the skill executor to default to the Anthropic API rather than the GitHub Copilot proxy,
So that the app works out of the box without requiring Copilot configuration, and thinking budgets are enforced correctly on all Claude models.

## Metric linkage

- **M1** (Authorization coverage rate, benefit-metric.md §M1): Skills must produce quality output for auth coverage to be measurable. A model error on every journey start blocks M1 from being measured at all.
- **M2** (Cross-tenant journey data leakage prevention, benefit-metric.md §M2): Journey persistence (Neon write, Redis session) only fires after the model responds. A model error at turn 0 prevents AC4/AC5 of s4.1 from being verified.

## Acceptance criteria

**AC1** — Default provider is Anthropic when env var is unset
Given `SKILL_EXECUTOR_PROVIDER` is not set in the environment,
When `skillTurnExecutor` or `skillTurnExecutorStream` is called,
Then the executor routes to the Anthropic Messages API.

**AC2** — Copilot provider works when explicitly opted in
Given `SKILL_EXECUTOR_PROVIDER=copilot` is set,
When `skillTurnExecutor` is called,
Then the executor routes to `api.githubcopilot.com/chat/completions` (existing behaviour unchanged).

**AC3** — `.env.example` documents the new default
Given a new operator copies `.env.example` to set up the app,
Then `SKILL_EXECUTOR_PROVIDER=anthropic` appears as the documented default with a note that Copilot is opt-in.

**AC4** — Tests that test the copilot path pin the provider explicitly
Given T9 and T10 in `check-wuce26-per-answer-model-response.js` test copilot-specific request format,
Then each test sets `process.env.SKILL_EXECUTOR_PROVIDER = 'copilot'` before the call and restores it in `finally`.

## Out of scope

- Removing the copilot provider entirely — kept as opt-in for teams with Copilot licences
- Adding a new provider (mock is already separate; no new providers here)
- Changing which model is used — `WUCE_TURN_MODEL` is unchanged
