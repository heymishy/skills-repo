### Task 2 — Implement journey-access.js (GREEN phase)

**File:** `src/web-ui/middleware/journey-access.js`
**Action:** Create the module with four named exports.

**Export: `POLICY`**
```js
const POLICY = { TENANT: 'TENANT', OWNER: 'OWNER' };
```

**Export: `isSameTenant(journey, session)`**
- Returns `true` if `journey.tenantId` is null or undefined (Phase 0: either side absent → permissive)
- Returns `true` if `session.tenantId` is null or undefined
- Returns `false` only if BOTH sides have tenantId AND they differ
- No I/O, synchronous

**Export: `requireJourneyAccess(journey, session, policy)`**
Check order (must be evaluated in this order):
1. If `journey` is null or undefined → throw `{ code: 'NOT_FOUND' }`
2. If `!session || !session.accessToken` → throw `{ code: 'UNAUTHENTICATED' }`
3. If `journey.ownerId` is null or undefined → return (legacy passthrough, AC1)
4. If `session.login === journey.ownerId` → return (owner, AC2)
5. If `isSameTenant(journey, session)` returns false → throw `{ code: 'FORBIDDEN' }` (AC3)
6. Otherwise → return (same tenant, passthrough)

Note: In Phase 0 `isSameTenant` always returns true (step 5 never fires) but the code structure is correct for Phase 1 without modification.

**Export: `asHttpResponse(err, policy)`**
Returns a plain integer status code:
- `err.code === 'UNAUTHENTICATED'` → 401 (both policies)
- `err.code === 'NOT_FOUND'` → 404 (both policies)
- `err.code === 'FORBIDDEN'` + `policy === POLICY.OWNER` → 403
- `err.code === 'FORBIDDEN'` + `policy === POLICY.TENANT` → 404 (existence leak prevention)
