# DoR Contract — p2.1: Tenant-parameterised repoRoot + session store namespacing + featureSlug collision guard

**Date:** 2026-06-24
**Story:** p2.1
**Feature:** wuce-multi-tenancy

---

## What will be built

1. `src/web-ui/adapters/repo-root.js` (new) — `slugifyTenantId(id)` and `getRepoRoot(req)`. Synchronous. Falls back to `_repoRoot || CLAUDE_REPO_PATH || COPILOT_REPO_PATH || path.resolve('.')` when `WUCE_TENANT_ROOT_BASE` or `tenantId` is absent.
2. `src/web-ui/adapters/session-store.js` (modify) — `writeSession` and `readSession` use tenant-namespaced subdirectory when `tenantId` present. `accessToken` strip-before-write invariant unchanged.
3. Path-traversal guard on all disk write points derived from `getRepoRoot(req)` — HTTP 400, no raw path logged.
4. `tests/check-p2.1-tenant-storage.js` (new) — 11 tests covering all ACs.

## What will NOT be built

- In-memory `_journeys` Map namespacing — Phase 3
- Tenant directory auto-provisioning — operator-manual
- Adversarial injection test suite — Phase 5 (p5.1)
- Any change to journeyId (UUID) generation or lookup keys
- `server.js` wiring changes — no new injectable adapters

## How each AC will be verified

| AC | Test approach | Type |
|----|--------------|------|
| AC1 — getRepoRoot tenant-scoped path | Unit: set `WUCE_TENANT_ROOT_BASE`, assert `path.resolve('/data/tenants/my-org')` returned | Unit |
| AC2 — fallback when tenantId absent | Unit: no tenantId in session, assert env-var fallback path returned | Unit |
| AC3 — path-traversal guard: HTTP 400 + no file | Integration: crafted featureSlug `../../etc/passwd`, assert 400 AND `!fs.existsSync` | Integration |
| AC4 — tenantId slugified | Unit: 6 cases (uppercase, spaces, `%2f`, path separators, 100-char, leading/trailing hyphens) | Unit |
| AC5 — session written to namespaced path | Integration: write session, assert file at `SESSION_STORE_PATH/my-org/sess-123.json` | Integration |
| AC6 — cross-namespace read isolation | Integration: two tenant files, assert read returns correct org's file only | Integration |
| AC7 — featureSlug collision guard | Integration: same featureSlug, two tenants, assert separate disk paths | Integration |
| AC8 — accessToken stripped at namespaced path | Integration: write with accessToken, read file, assert key absent | Integration |
| AC9 — test suite passes | `node tests/check-p2.1-tenant-storage.js` → exit 0 | Integration |

## Assumptions

- `_slugify()` in `journey.js` is the canonical slugification reference; `slugifyTenantId` reuses identical logic
- `SESSION_STORE_PATH` env var already in codebase; write/read functions in `src/web-ui/adapters/session-store.js`
- `setRepoRoot()` / `_repoRoot` remains as test-helper only; `getRepoRoot(req)` is the sole production path (5-M2 resolution)
- Disk write points in `journey.js` will be enumerated as Task 1 of the implementation plan before any code is written
