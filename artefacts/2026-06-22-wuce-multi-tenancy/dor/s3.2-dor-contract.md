# DoR Contract — s3.2: Upstash Redis provisioning and session persistence activation

**Feature:** wuce-multi-tenancy
**Story:** s3.2
**Date:** 2026-06-29

---

## What will be built

The operator will: (1) create an Upstash Redis database at console.upstash.com (free tier, single-region) and retrieve `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN`; (2) run `node scripts/smoke-test-redis.js` to confirm round-trip write/read/delete with `accessToken` strip verification and TTL range check; (3) set both credentials as Fly.io secrets.

## What will NOT be built

- Multi-region Upstash read replicas (single-region default per Decision 9)
- Session encryption beyond Upstash's default TLS
- Per-tenant session TTL configuration
- Automated credential rotation

## How each AC will be verified

| AC | Test approach | Type |
|----|---------------|------|
| AC1 — Upstash REST credentials available | Operator retrieves from Upstash dashboard; format validated by smoke test | Manual |
| AC2 — Round-trip smoke test passes | `UPSTASH_REDIS_REST_URL=<url> UPSTASH_REDIS_REST_TOKEN=<token> node scripts/smoke-test-redis.js` — exits 0 | Integration |
| AC3 — accessToken absent from Redis writes | Smoke test reads stored value; asserts accessToken field absent and login present | Integration |
| AC4 — TTL of 86400 confirmed on session keys | Smoke test checks TTL immediately after write; asserts value in [86390, 86400] | Integration |
| AC5 — Credentials in Fly.io secrets only | `fly secrets list` shows both; git grep confirms no committed values | Manual |

## Assumptions

- Upstash free tier provides sufficient throughput for 5–10 beta users (per Decision 9).
- `scripts/smoke-test-redis.js` is already merged (PR #419).
- `UPSTASH_REDIS_REST_URL` will be of the form `https://*.upstash.io`.
