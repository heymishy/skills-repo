# DoR Contract — s4.2: Tenant onboarding configuration

**Feature:** wuce-multi-tenancy
**Story:** s4.2
**Date:** 2026-06-29

---

## What will be built

The operator will: (1) set `TENANT_ORG_ALLOWLIST` as a Fly.io secret with the intended beta org login(s); (2) set `WUCE_TENANT_ROOT_BASE` as a Fly.io secret (e.g. `/app/tenants`); (3) create tenant root directories via `fly ssh console`; (4) set `MAX_JOURNEYS_PER_TENANT` (or create `tenant-caps.json`) with the intended beta cap; (5) verify with both an allowlisted and a non-allowlisted GitHub account login flow. No `src/` code changes.

## What will NOT be built

- Self-service tenant registration (operator-manual per Decision 3)
- Automated directory creation (Decision 3: deliberate operator friction)
- Per-tenant feature flags beyond journey cap

## How each AC will be verified

| AC | Test approach | Type |
|----|---------------|------|
| AC1 — Allowlisted user logs in with correct tenantId | OAuth flow with allowlisted GitHub account; Fly.io logs confirm `[auth] login { tenantId: "<org-login>" }` | Manual (E2E) |
| AC2 — Non-allowlisted user receives 403 | OAuth flow with non-allowlisted account; response body is exactly "You are not a member of an authorised organisation." with no org names disclosed | Manual (E2E) |
| AC3 — Tenant root directories exist | `fly ssh console -C "ls $WUCE_TENANT_ROOT_BASE"` lists subdirectory per allowlisted org | Manual |
| AC4 — Journey stored under correct tenant tree | Create journey as allowlisted user; `fly ssh console` confirms artefact in `$WUCE_TENANT_ROOT_BASE/<tenantId>/artefacts/` | Manual (E2E) |
| AC5 — Per-user journey cap configured | `fly secrets list` shows MAX_JOURNEYS_PER_TENANT; cap tested by creating journeys up to limit | Manual |
| AC6 — Solo-mode works for operator | Start app locally without TENANT_ORG_ALLOWLIST; login succeeds; journey paths use GitHub username | Manual |

## Assumptions

- s4.1 (production deployment) is complete before this story runs.
- The beta org logins are known and available before running this story.
- `WUCE_TENANT_ROOT_BASE` maps to a persistent directory on the Fly.io machine (or volume); ephemeral filesystem is acceptable for beta with awareness that directories reset on full redeploy.
