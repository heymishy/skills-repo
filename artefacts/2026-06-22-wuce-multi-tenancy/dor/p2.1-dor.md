### What to build

1. **`src/web-ui/adapters/repo-root.js`** (new module)
   - Export `slugifyTenantId(id)`: lowercase, replace non-alphanumeric with `-`, strip leading/trailing `-`, max 48 chars. Same logic as `_slugify()` in `journey.js`.
   - Export `getRepoRoot(req)`: if `process.env.WUCE_TENANT_ROOT_BASE` and `req.session.tenantId` are both set, return `path.resolve(WUCE_TENANT_ROOT_BASE, slugifyTenantId(req.session.tenantId))`. Otherwise fall back to `_repoRoot || process.env.CLAUDE_REPO_PATH || process.env.COPILOT_REPO_PATH || path.resolve('.')`. Function must be synchronous — no I/O.
   - `getRepoRoot` falls back to `_repoRoot` (the existing injectable in `journey.js`) when `WUCE_TENANT_ROOT_BASE` is absent, maintaining single-tenant backward compatibility.

2. **`src/web-ui/adapters/session-store.js`** (modify)
   - Import `slugifyTenantId` from `repo-root.js`.
   - `writeSession(session)`: if `session.tenantId` is present, write to `SESSION_STORE_PATH/${slugifyTenantId(session.tenantId)}/${session.sessionId}.json`. Otherwise write to the existing flat path. Strip `accessToken` before write — invariant unchanged.
   - `readSession(sessionId, tenantId)`: if `tenantId` present, read from tenant-namespaced path. Otherwise read from flat path.

3. **Path-traversal guard (existing handlers)**
   - Task 1 of the implementation plan must enumerate every handler in `journey.js` (and other routes) that computes a disk write path. Each must assert: `const resolved = path.resolve(derived); if (!resolved.startsWith(tenantRoot + path.sep)) return res.status(400).end()`. Do not log the raw derived path on failure.

4. **`tests/check-p2.1-tenant-storage.js`** (new — TDD RED first)
   - 11 tests: AC1 (1 unit), AC2 (1 unit), AC3 (1 integration), AC4 (5 unit), AC5 (1 integration), AC6 (1 integration), AC7 (1 integration), AC8 (1 integration). Plus NFR tests for sync/< 1ms, async write, no-raw-path-in-log.
   - Use `os.tmpdir()` + `fs.mkdtempSync()` for temp dirs; clean up in a `finally` block.

### What NOT to build

- In-memory `_journeys` Map namespacing — explicitly out of scope
- Tenant directory auto-provisioning
- Any change to journeyId generation or UUID lookup
- Any adversarial injection test suite (Phase 5)

### Architecture constraints (MUST enforce)

- **Path-traversal guard (CLAUDE.md ougl):** `path.resolve(derived).startsWith(tenantRoot + path.sep)` before every disk write whose path includes a tenant-derived component. HTTP 400 on failure. No raw path in log output.
- **`req.session.accessToken` canonical field:** Never use `req.session.token`. All session reads use `req.session.accessToken`.
- **NFR-sec-no-accesstoken-disk:** `accessToken` must not appear in any persisted session file. The strip-before-write invariant in `session-store.js` is mandatory.
- **`slugifyTenantId` before all path construction:** Never use raw `session.tenantId` directly in a file path.
- **`getRepoRoot` is synchronous:** No `await`, no `fs.*Sync`, no network calls.
- **D37 (injectable adapter rule):** No new injectable adapters introduced in this story — `getRepoRoot` is a plain function.

### Dependency resolution (5-M2)

`getRepoRoot(req)` is the sole production path for repoRoot resolution. `setRepoRoot()` / `_repoRoot` in `journey.js` remains for test-helper use only. `getRepoRoot(req)` falls back to `_repoRoot` when `WUCE_TENANT_ROOT_BASE` is absent and tenantId is present — this satisfies AC2's backward-compatible fallback while preserving single-tenant mode.
