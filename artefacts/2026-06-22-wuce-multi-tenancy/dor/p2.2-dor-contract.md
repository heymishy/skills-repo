# DoR Contract — p2.2: Tenant-isolation test suite

**Date:** 2026-06-24
**Story:** p2.2
**Feature:** wuce-multi-tenancy

## What will be built

1. `tests/check-p2.2-tenant-isolation.js` (new) — plain Node.js `assert` test file following the `check-*.js` convention. Proves tenant isolation at HTTP and filesystem layers using a two-tenant fixture (alice/org-a, bob/org-b).

## What will NOT be built

- No production code changes (journey.js, session-store.js, repo-root.js, or any other source file is out of scope)
- No concurrent/stress testing (Phase 3, p3.3)
- No adversarial tenantId injection (Phase 5, p5.1)
- No Postgres/Redis isolation tests (Phase 3)

## How each AC will be verified

| AC | Test approach | Type |
|----|--------------|------|
| AC1 — 9 cross-tenant 404s | 9 individual named handler calls with bob's session / alice's journeyId; assert HTTP 404 each | Integration |
| AC2 — 2 cross-tenant 403s | `handlePostJourneyRecommit` + `handlePostJourneyStageCommit` with bob's session; assert 403 | Integration |
| AC3 — filesystem isolation | Write files under org-a/org-b tmpdirs; recursive scan; assert zero cross-paths in each tree | Integration |
| AC4 — session store isolation | `readSession('sess-id', 'org-b')` for session only under org-a; assert null returned | Integration |
| AC5 — same-tenant not broken | alice calling alice's journey on all 9 read routes; assert no 4xx | Integration |
| AC6 — suite passes | `node tests/check-p2.2-tenant-isolation.js` → exit 0 | Integration |
| AC7 — no regression | All pre-existing `check-*.js` suites continue to pass | Integration |

## Assumptions

- p2.1 is complete and merged (dodStatus: complete, prStatus: merged — confirmed 2026-06-24)
- `isSameTenant(journey, session)` enforces the org-a/org-b boundary — populated at journey create time (p1.2) and wired through getRepoRoot(req) (p2.1)
- All 9 KNOWN BUG route handlers are exported from `journey.js` and callable without HTTP harness
- 4-field makeReq shape (`{ accessToken, userId, login, tenantId }`) is current convention (updated in p1.2)
- `WUCE_TENANT_ROOT_BASE` and `SESSION_STORE_PATH` can be overridden via process.env per test
