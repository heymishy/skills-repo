**W2 — buildCacheKey integration point decided (RISK-ACCEPT)**
The Anthropic ephemeral cache_control mechanism hashes content, not a named key. To scope cache entries per tenant, the implementation will embed `<!-- cache-scope: ${tenantId}-${sessionId} -->` as the first line of the system prompt in `wrapSystemPromptForCaching` (`src/modules/skill-turn-executor.js`). This makes the content hash unique per tenant+session. `buildCacheKey(session)` is a pure helper function in `src/web-ui/adapters/cache-key.js` that constructs the scope string. Operator acknowledged.
