## What will be built

1. `src/web-ui/auth/oauth-adapter.js` — add `read:org` to the OAuth scope string when `TENANT_ORG_ALLOWLIST` is set and non-empty; unchanged when absent or empty
2. OAuth callback handler (in `src/web-ui/routes/auth.js` — confirm by reading) — async tenant resolution after token exchange: parse `TENANT_ORG_ALLOWLIST`, call `_fetchOrgs(accessToken)` across all pages (pagination via GitHub Link header), find first match in allowlist order, set `req.session.tenantId`; zero-match returns HTTP 403 with generic error, no session created; absent/empty allowlist skips org fetch entirely
3. Injectable adapter in callback module: `let _fetchOrgs = () => { throw new Error('Adapter not wired: fetchOrgs. Call setFetchOrgs() with a real implementation before use.') }; function setFetchOrgs(fn) { _fetchOrgs = fn; }`
4. `src/web-ui/server.js` — wire `setFetchOrgs()` with a real implementation calling `GET /user/orgs` using the OAuth access token; wiring call before OAuth routes are registered
5. `tests/check-p1.1-oauth-tenant-resolution.js` — 7 unit tests + 3 NFR tests; all use injectable mock; no real GitHub API calls

## What will NOT be built

- Writing `tenantId` to the journey object — that is p1.2
- `isSameTenant()` real enforcement — that is p1.2
- Test session fixture shape update (3-field → 4-field) — that is p1.2
- Multi-org membership disambiguation beyond first-allowlist-match — deferred (Discovery OQ5)
- GHE token URL fix — explicitly out of scope for this feature

## Assumptions

- OAuth callback handler is in `src/web-ui/routes/auth.js` — confirm on first read; if different, use actual location
- GitHub `GET /user/orgs` returns `{ login: string }[]` per page; `Link: <url>; rel="next"` header signals more pages
- Injectable adapter is co-located in the OAuth callback module (not in a separate adapter file and not inline in server.js)
- The test mock uses a simplified page-number interface; production uses Link header parsing — pagination contract must be internally consistent
