## What will be built

1. `src/modules/skill-turn-executor.js` (MODIFY) — `skillTurnExecutor()` and `skillTurnExecutorStream()` extract a `session` object (`{ tenantId, sessionId }`) from their existing `meta`/`options` parameter and pass it to `_callAnthropic()` / `_callAnthropicStream()` respectively, which pass it to the already-existing `_anthropicSystem(systemPrompt, session)` call. No change to `_anthropicSystem()`'s own body.
2. `src/web-ui/routes/skills.js` (MODIFY) — `htmlSubmitTurn()` gains an additional optional `tenantId` parameter and includes `tenantId`/`sessionId` in the `_turnMeta` object it already builds and passes to `_skillTurnExecutor()`. `handlePostTurnHtml()` passes `req.session.tenantId` at its existing call to `htmlSubmitTurn()`. `handlePostTurnStreamHtml()` adds `tenantId`/`sessionId` to the `_turnOptions` object it already builds and passes to `_skillTurnExecutorStream()`.
3. `tests/check-s6.1-cache-scope-session-threading.js` (NEW) — integration-style tests intercepting `https.request` at the module boundary, covering AC1–AC4 and AC7 (per test plan).

## What will NOT be built

- No change to `src/web-ui/adapters/cache-key.js` (`buildCacheKey()`)
- No change to `_anthropicSystem()`'s function body in `src/modules/skill-turn-executor.js`
- No change to `htmlRecordAnswer()` / `handlePostAnswerHtml()` (legacy wizard flow — superseded per `docs/web-ui-skill-session-as-built.md`)
- No change to `_callCopilot()` / `_callCopilotStream()` — Copilot has no cache_control mechanism in this codebase
- No change to `src/web-ui/middleware/rate-limiter.js` — already tenant-scoped by p4.1
