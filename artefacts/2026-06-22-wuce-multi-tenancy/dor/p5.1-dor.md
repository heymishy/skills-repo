**AC7 interpretation note:** `getRepoRoot(req)` calls `slugifyTenantId(tenantId)` before path construction, so a traversal in `session.tenantId` produces a safe slug — the path guard never fires for tenantId-based traversal. The correct security outcome is that the derived path is always within `WUCE_TENANT_ROOT_BASE`. AC7 tests verify this property (path-containment, not HTTP 400). This aligns with the story intent: "a maliciously crafted tenantId cannot escape the tenant directory."
