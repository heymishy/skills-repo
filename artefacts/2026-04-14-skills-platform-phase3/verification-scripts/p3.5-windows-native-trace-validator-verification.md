## Setup

**Before you start:**
1. Open a PowerShell terminal in the root of the `skills-repo` repository.
2. Confirm PowerShell is available: run `$PSVersionTable.PSVersion` — version 5.1 or higher.
3. After the story is implemented, `scripts/validate-trace.ps1` should exist alongside `scripts/validate-trace.sh`.
4. The test fixtures (`tests/fixtures/valid-trace.yaml` and `tests/fixtures/missing-field-trace.yaml`) are created by the coding agent as part of the test suite.

**Reset between scenarios:** Each scenario is independent. No shared state.

## Scenarios

---

### Scenario 1: PS1 passes on a valid trace

**Covers:** AC1

**Steps:**
1. In your PowerShell terminal, run:
   `pwsh -File scripts/validate-trace.ps1 --ci tests/fixtures/valid-trace.yaml`
2. Observe the output and the exit code.
3. Run `echo $LASTEXITCODE` to confirm the exit code.

**Expected outcome:**
> The script prints the name of each check it runs (e.g. "Checking storySlug... OK"). The final exit code is 0. The check names should match the check names you see when running `bash scripts/validate-trace.sh --ci tests/fixtures/valid-trace.yaml` (if you have bash available to compare).

### Scenario 2: PS1 fails and names the problem field on an invalid trace

**Covers:** AC2

**Steps:**
1. Run:
   `pwsh -File scripts/validate-trace.ps1 --ci tests/fixtures/missing-field-trace.yaml`
2. Observe the output and the exit code.
3. Run `echo $LASTEXITCODE`.

**Expected outcome:**
> The script prints a message naming the specific missing or invalid field (e.g. "FAIL: storySlug is missing"). The exit code is non-zero (not 0). The same invalid fixture run through `validate-trace.sh --ci` also returns a non-zero exit code.

### Scenario 3: Every check in the SH script is also in the PS1 script

**Covers:** AC3

**Steps:**
1. Run `bash scripts/validate-trace.sh --ci tests/fixtures/valid-trace.yaml` (if bash is available) and note the check names printed.
2. Run `pwsh -File scripts/validate-trace.ps1 --ci tests/fixtures/valid-trace.yaml` and note the check names printed.
3. Compare the two lists.

**Expected outcome:**
> Every check name that appears in the SH output also appears in the PS1 output. The number of checks is the same. No check is missing or has a slightly different name.
> If bash is not available on your machine, this scenario can be verified by reviewing the automated test report, which compares the parsed check lists programmatically.

**Result:** [ ] Pass  [ ] Fail
