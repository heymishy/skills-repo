### Scenario 2: Watermark creation is blocked when fewer than 70% of scenarios pass

**Covers:** AC2

**Steps:**
1. In `npm test` output, find the test group for `watermark-gate`.
2. Confirm a test exists that inputs a scenario set where only 60% pass (e.g. 3 of 5) and asserts the result is an error / exit 1.
3. Confirm a boundary test exists for exactly 70% (e.g. 7 of 10) — this should pass normally.

**Expected outcome:**
> The 60%-passing test produces an error and no baseline is written. The 70%-passing test does not produce an error. Both tests are green in `npm test`.

### Scenario 4: Adding one check and removing one check is rejected by the anti-overfitting gate

**Covers:** AC3

**Steps:**
1. In `npm test` output, find the test group for `failure-detector`.
2. Confirm a test exists that evaluates a proposal with +1 added check and −1 removed check.
3. Confirm the test asserts: exit code 1, and output contains separate labels for `added`, `removed`, and `pass` counts.

**Expected outcome:**
> The test passes. The gate rejects the "add one, remove one" proposal and the output clearly shows three separate counters — the reviewer can see exactly how many checks were added, removed, and passed, not just a net total.

**Result:** [ ] Pass  [ ] Fail
