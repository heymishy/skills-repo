## Setup

**Before you start:**
1. Open a terminal in the root of the `skills-repo` repository.
2. Ensure `node` is available: run `node --version` — any v18+ is fine.
3. The `check-trace-commit.js` script should exist at `tests/check-trace-commit.js` (or the path registered in `package.json`) after the story is implemented.
4. For post-merge verification: run `npm test` first to confirm the suite is green, then work through these scenarios manually.

**Reset between scenarios:** No shared state — each scenario is independent.

### Scenario 2: Script reports failure when trace record is older than 24 hours

**Covers:** AC2

**Steps:**
1. In a terminal, run the check with a stale-trace fixture — the coding agent will provide the exact command in the test runner output, but manually you can use:
   `node tests/check-trace-commit.js --traces-dir tests/fixtures/stale-traces`
   (or the equivalent path documented in the test plan)
2. Observe the exit code and output.

**Expected outcome:**
> The script prints a message to the terminal that includes the timestamp of the last recorded commit trace. The message should make clear that the trace is outdated. The process exits with code 1 (you can confirm by running `echo $LASTEXITCODE` in PowerShell or `echo $?` in bash — value will be `1`).

### Scenario 3: Script reports success when trace record is current

**Covers:** AC3

**Expected outcome:**
> The process exits with code 0 (confirm: `echo $LASTEXITCODE` = `0`). No error message appears. Output may include a confirmation message such as "trace is current" or similar, but the key indicator is exit code 0.

### Scenario 4: Script reports failure when branch has no trace record at all

**Covers:** AC4

**Steps:**
1. Run the check with a fixture directory that contains no trace file for the tested branch:
   `node tests/check-trace-commit.js --traces-dir tests/fixtures/empty-traces`
2. Observe the exit code and output.

**Expected outcome:**
> The process exits with code 1. The output indicates that no trace record was found for the branch — it should not exit 0 or silently succeed. The message distinguishes "absent" from "stale" (both exit 1 but for different reasons).
