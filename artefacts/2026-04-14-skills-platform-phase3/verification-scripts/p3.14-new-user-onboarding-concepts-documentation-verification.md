## Scenario 1: All required concept files exist with exact names

**Covers:** AC1

**Steps:**
1. In the terminal, run: `ls docs/concepts/`
2. Confirm the output shows exactly: `README.md`, `building-blocks/`, `primitives/`, `principles/` — nothing else.
3. Run: `ls docs/concepts/principles/`
4. Confirm exactly these 6 files appear (in any order): `governance-by-demonstration.md`, `the-subset-is-the-on-ramp.md`, `adapter-isolated-surface-concerns.md`, `spec-immutability.md`, `human-approval-at-every-gate.md`, `self-improving-harness.md`. Count them — there should be exactly 6.

6. Confirm exactly these 6 files appear: `skill.md`, `surface-adapter.md`, `assurance-gate.md`, `pipeline-state.md`, `eval-suite.md`, `learnings-log.md`. Count — exactly 6.

## Scenario 7: Each concept file is self-contained — no assumed prior reading

**Covers:** AC7 — Manual gap scenario 🔴

**Steps:**
1. Pick any two concept files at random — from different subdirectories (e.g. one principle, one building block).
2. Read each file from start to finish without opening any other file.
3. After reading, answer: "Could I give a correct one-paragraph description of this concept to a colleague, based only on what I just read?" If yes, the file passes.
4. Look through each file for any link or reference to another document. Confirm every such reference is labelled as "Optional further reading:" — not as a required step or dependency.

**Expected outcome:**
> Each file, read in isolation, is sufficient to understand the concept it describes. No file says things like "as we described in..." or "you should have read X first." All cross-references are clearly labelled "Optional further reading:".

**Result:** [ ] Pass  [ ] Fail
