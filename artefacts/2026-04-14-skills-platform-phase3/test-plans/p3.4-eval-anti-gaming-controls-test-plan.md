## Test Data Strategy

**Source:** Synthetic — fixture suite.json entries created in test setup:
- `fixtures/entry-missing-traceId.json` — new entry: has `failurePattern` but no `traceId`
- `fixtures/entry-missing-failurePattern.json` — new entry: has `traceId` but no `failurePattern`
- `fixtures/entry-missing-both.json` — new entry: neither field present
- `fixtures/entry-valid-new.json` — new entry: has valid `traceId` (file exists) + `failurePattern`
- `fixtures/entry-valid-traceId-missing-file.json` — new entry: has `traceId` but the referenced trace file does not exist
- `fixtures/entry-grandfathered.json` — simulated pre-existing entry marked with `_grandfathered: true` or added to the existing entry list before the story's merge marker

**PCI/sensitivity:** No
**External network:** None

## Unit Tests

### validation-fails-on-missing-trace-id

- **Verifies:** AC1 (missing traceId)
- **Test file:** `tests/check-assurance-gate.js` (extend) or `src/suite-validation/suite-validator.test.js`
- **Precondition:** Validation script/module exists
- **Action:** Pass `fixtures/entry-missing-traceId.json` as a candidate new entry to the validator
- **Expected result:** Validator exits 1; error message names `traceId` as the missing field
- **RED step:** Validator does not check for traceId; exits 0 on missing field

### validation-fails-on-missing-failure-pattern

- **Verifies:** AC1 (missing failurePattern)
- **Test file:** `src/suite-validation/suite-validator.test.js`
- **Precondition:** Validation script/module exists
- **Action:** Pass `fixtures/entry-missing-failurePattern.json` to validator
- **Expected result:** Validator exits 1; error message names `failurePattern` as the missing field
- **RED step:** Validator does not check failurePattern

### validation-fails-on-trace-id-file-not-found

- **Verifies:** AC2 (traceId file resolution)
- **Test file:** `src/suite-validation/suite-validator.test.js`
- **Precondition:** Validation module exists; `fixtures/entry-valid-traceId-missing-file.json` has a traceId pointing to a path that does not exist in fixtures
- **Action:** Run validator; traceId file resolution should fail
- **Expected result:** Validator exits 1 with a message indicating the referenced trace file does not exist; names the path
- **RED step:** Validator accepts traceId without checking file existence

### validation-passes-with-valid-trace-id-and-file

- **Verifies:** AC2 (traceId file resolution — happy path)
- **Test file:** `src/suite-validation/suite-validator.test.js`
- **Precondition:** Validation module exists; a fixture trace file at the expected path is created
- **Action:** Pass `fixtures/entry-valid-new.json` to validator; the referenced trace file exists at `workspace/traces/[traceId]`
- **Expected result:** Validator passes (exits 0)
- **RED step:** Validator rejects valid entry

### improvement-agent-proposal-written-to-proposals-not-suite

- **Verifies:** AC3 (proposal staging path)
- **Test file:** `src/improvement-agent/suite-proposal.test.js`
- **Precondition:** Improvement agent suite proposal module exists
- **Action:** Trigger the improvement agent to generate a new scenario proposal; check where the output is written
- **Expected result:** Proposal is written to `workspace/proposals/suite-additions/[proposal-id].json`; NOT written to `workspace/suite.json` or `platform/suite.json`
- **RED step:** Agent writes directly to suite.json

### improvement-agent-proposal-contains-required-fields

- **Verifies:** AC3 (proposal content — traceId, failurePattern, justification)
- **Test file:** `src/improvement-agent/suite-proposal.test.js`
- **Precondition:** Proposal generation module exists; proposal generated by previous test
- **Action:** Read the generated proposal JSON; verify it contains `traceId`, `failurePattern`, and `justification` fields, all non-null/non-empty
- **Expected result:** All three fields present and populated
- **RED step:** Proposal lacks one or more required fields

### grandfathered-entries-pass-validation

- **Verifies:** AC5 (grandfathered entries pass)
- **Test file:** `src/suite-validation/suite-validator.test.js`
- **Precondition:** Validation module exists; `fixtures/entry-grandfathered.json` simulates a pre-existing entry without traceId/failurePattern
- **Action:** Run validator in "grandfathered" mode or add the entry to the existing entries list (before the story merge marker); validate the full suite
- **Expected result:** Existing entries without traceId/failurePattern pass without error; only new entries are checked
- **RED step:** Validator retroactively rejects pre-existing entries

## Manual Tests

### no-automated-proposal-promotion-path-exists

- **Verifies:** AC4 (human-only promotion)
- **Instruction:** Review the source code of the improvement agent and the validation pipeline; confirm there is no code path that automatically moves a file from `workspace/proposals/suite-additions/` to `workspace/suite.json` or `platform/suite.json`. The only path is a human opening a PR to the platform-infrastructure repo.
- **Expected outcome:** No auto-promotion code path found; proposal staging is terminal unless human intervenes
