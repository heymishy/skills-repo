## NFR Tests

### trace-fields-produce-deterministic-output-for-same-input

- **NFR addressed:** Audit (deterministic field values for given input)
- **Measurement method:** Run gate against the same regulated trace fixture twice in sequence; compare the written trace outputs
- **Pass threshold:** Both output trace files are byte-identical (or field-identical if serialisation order varies — all four field values must match)
- **Tool:** Node.js test (Jest or equivalent); file comparison with `assert.deepStrictEqual`

### sessionIdentity-sessionId-does-not-contain-PII

- **NFR addressed:** Security (sessionId must not contain PII)
- **Measurement method:** Run gate with a fixture that uses a realistic sessionId value; assert the written sessionId is not a raw email address, user display name, or UUID that can be directly traced to a human identity. The test must confirm the value is either a hash (hex string ≥ 32 chars) or a truncated/opaque identifier.
- **Pass threshold:** Written `sessionIdentity.sessionId` matches the pattern `/^[a-f0-9]{32,}$/i` (hex hash) OR is an opaque non-personal identifier (test asserts it does NOT match email pattern `/\S+@\S+/`)
- **Tool:** Jest — regex assertion on the written trace output
