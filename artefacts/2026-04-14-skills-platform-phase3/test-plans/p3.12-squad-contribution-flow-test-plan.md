## Unit Tests

### contribution-guide-exists

- **Verifies:** AC1 (file existence)
- **Test file:** `tests/check-p3.14-concepts-docs.js` (extend) or `tests/check-squad-contribution.js`
- **Precondition:** Story is implemented
- **Action:** Check that `docs/squad-contribution-guide.md` exists
- **Expected result:** File exists
- **RED step:** File not yet created

### contribution-guide-has-proposal-flow-section

- **Verifies:** AC1 (section 1: propose via feature branch)
- **Test file:** `tests/check-squad-contribution.js`
- **Precondition:** `docs/squad-contribution-guide.md` exists
- **Action:** Read file; verify it contains a section describing the contribution workflow starting with a feature branch proposal step
- **Expected result:** "feature branch" and "propose" (or equivalent) language present in the flow section
- **RED step:** Guide exists but the flow steps are absent or incomplete

### contribution-guide-has-required-pr-contents-section

- **Verifies:** AC1 (section 2: required PR contents — SKILL.md + EVAL.md + performance evidence)
- **Test file:** `tests/check-squad-contribution.js`
- **Precondition:** `docs/squad-contribution-guide.md` exists
- **Action:** Read file; verify it names all three required PR artefacts: SKILL.md, EVAL.md, and performance evidence (story reference or metric movement)
- **Expected result:** All three artefact types named in required PR contents section
- **RED step:** Guide omits one or more required artefact types

### contribution-guide-has-approval-gate-section

- **Verifies:** AC1 (section 4: human approval gate — designated platform reviewer sign-off)
- **Test file:** `tests/check-squad-contribution.js`
- **Precondition:** `docs/squad-contribution-guide.md` exists
- **Action:** Read file; verify it describes the required approval gate: explicit sign-off from a designated platform reviewer (named role, team alias, or named individual)
- **Expected result:** Approval gate described with platform reviewer requirement
- **RED step:** Guide says "any reviewer" or omits the designated platform reviewer requirement

### contribution-guide-has-merge-path-section

- **Verifies:** AC1 (section 5: merge path — platform team, not contributing squad)
- **Test file:** `tests/check-squad-contribution.js`
- **Precondition:** `docs/squad-contribution-guide.md` exists
- **Action:** Read file; verify it explicitly states that the contributing squad does not merge — the platform team merges the contribution PR
- **Expected result:** "platform team" and "merge" stated; contributing squad excluded from merge action
- **RED step:** Guide does not distinguish who merges

### contribution-guide-eval-md-references-trace-id-and-failure-pattern

- **Verifies:** AC2 (EVAL.md requirements reference p3.4 fields)
- **Test file:** `tests/check-squad-contribution.js`
- **Precondition:** `docs/squad-contribution-guide.md` exists; p3.4 is DoD-complete
- **Action:** Read EVAL.md requirements section of the guide; verify both `traceId` and `failurePattern` are named as mandatory fields; verify the reference to p3.4 requirements is present (either by naming the story, or by referencing the `workspace/suite.json` validation requirements)
- **Expected result:** Both fields named; reference to p3.4 requirement context present
- **RED step:** EVAL.md section does not mention traceId or failurePattern

### contribution-guide-contains-pr-description-template

- **Verifies:** AC3 (PR description template)
- **Test file:** `tests/check-squad-contribution.js`
- **Precondition:** `docs/squad-contribution-guide.md` exists
- **Action:** Read file; verify it contains a PR description template block with at minimum: skill name field, use case rationale field, delivery story reference field, a minimum 3 EVAL.md test cases note, and platform reviewer name field
- **Expected result:** Template block present with all 5 required fields
- **RED step:** Guide describes the PR requirements in prose only — no template block

### contributing-md-section-and-link-to-guide

- **Verifies:** AC4 (CONTRIBUTING.md section with link)
- **Test file:** `tests/check-squad-contribution.js`
- **Precondition:** `CONTRIBUTING.md` exists or is created by this story
- **Action:** Read `CONTRIBUTING.md`; verify it contains a section heading "Contributing to the Skills Platform" (exact or semantically equivalent); verify the section contains a markdown link to `docs/squad-contribution-guide.md`; verify it clarifies that delivery repo changes (stories, artefacts, workspace) do not require this process
- **Expected result:** All three elements present in CONTRIBUTING.md
- **RED step:** CONTRIBUTING.md lacks the section or the link

### pr-diff-contains-only-documentation-files

- **Verifies:** AC5 (no new scripts, permissions, or CI config)
- **Test file:** `tests/check-squad-contribution.js` (or PR diff assertion)
- **Precondition:** Story implementation PR is open
- **Action:** Inspect the PR diff; verify it contains only `docs/squad-contribution-guide.md` and `CONTRIBUTING.md` (with possible `workspace/` state updates); confirm no `.github/workflows/`, no `scripts/`, no CODEOWNERS changes
- **Expected result:** Only documentation files in PR diff
- **RED step:** PR includes an automation script or CI configuration change
