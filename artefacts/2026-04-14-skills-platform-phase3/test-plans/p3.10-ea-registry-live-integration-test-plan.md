## Unit Tests

### skill-md-path-a-block-references-get-blast-radius

- **Verifies:** AC1 + AC5 (Path A documented with config condition)
- **Test file:** `tests/check-p3.14-concepts-docs.js` (extend) or `tests/check-discovery-skill.js`
- **Precondition:** Updated discovery SKILL.md exists
- **Action:** Read `.github/skills/discovery/SKILL.md`; verify it contains: (a) a section or conditional block referencing `ea_registry_authoritative: true`, (b) a call or instruction to invoke `getBlastRadius()` or equivalent EA registry query, (c) output instructions for presenting the blast-radius summary inline
- **Expected result:** All three elements present in SKILL.md
- **RED step:** SKILL.md has not yet been updated; no Path A block present

### skill-md-path-b-block-no-blast-radius

- **Verifies:** AC2 + AC5 (Path B documented with config condition)
- **Test file:** `tests/check-discovery-skill.js`
- **Precondition:** Updated discovery SKILL.md exists
- **Action:** Read `.github/skills/discovery/SKILL.md`; verify it explicitly describes Path B behaviour when `ea_registry_authoritative` is `false` or absent — confirming no blast-radius section is surfaced and no error occurs
- **Expected result:** Path B block present with "false or absent" condition and "behaves as before" or equivalent no-op language
- **RED step:** SKILL.md does not describe Path B explicitly

### skill-md-missing-entry-message

- **Verifies:** AC3 (missing registry entry graceful message)
- **Test file:** `tests/check-discovery-skill.js`
- **Precondition:** Updated discovery SKILL.md exists
- **Action:** Read the SKILL.md; verify it instructs the agent to surface `"no EA registry entry found for [systemId]"` (or semantically equivalent phrasing) when the registry query returns no result, and confirms the discovery session continues without error
- **Expected result:** Message template present in SKILL.md; "continues" or "not a blocker" language present
- **RED step:** SKILL.md does not handle the missing-entry case

### skill-md-read-only-confirmed

- **Verifies:** AC4 (read-only integration)
- **Test file:** `tests/check-discovery-skill.js`
- **Precondition:** Updated discovery SKILL.md exists
- **Action:** Read the SKILL.md; verify there is no write instruction to the EA registry — no text that would instruct the agent to call `registerSystem()`, `updateRegistry()`, or any write-equivalent; confirm text explicitly states read-only
- **Expected result:** No write operation instructions present; read-only stated explicitly
- **RED step:** SKILL.md contains accidental write instruction

### both-paths-have-configuration-condition-documented

- **Verifies:** AC5 (configuration condition explicit for both paths)
- **Test file:** `tests/check-discovery-skill.js`
- **Precondition:** Updated discovery SKILL.md exists
- **Action:** Read SKILL.md; confirm that the condition `architecture.ea_registry_authoritative: true` is named explicitly for Path A, and `false` or "not set" is named for Path B
- **Expected result:** Both conditions are explicit enough that a squad lead can determine their path by reading `context.yml` alone
- **RED step:** Conditions are implicit or missing
