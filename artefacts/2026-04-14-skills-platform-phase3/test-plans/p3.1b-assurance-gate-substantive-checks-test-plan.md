## Unit Tests

### gate-fails-when-state-currentPhase-absent

- **Verifies:** AC1
- **Precondition:** Fixture `state.json` with `currentPhase` field missing
- **Action:** Run `run-assurance-gate.js` against fixture; capture exit code
- **Expected result:** Exit code 1; output references missing `currentPhase`
- **Edge case:** No

### gate-fails-when-state-currentPhase-empty-string

- **Verifies:** AC1
- **Precondition:** Fixture `state.json` with `currentPhase: ""`
- **Action:** Run `run-assurance-gate.js` against fixture
- **Expected result:** Exit code 1
- **Edge case:** Yes — empty string is not null but must still fail

### gate-fails-when-state-lastUpdated-invalid-format

- **Verifies:** AC1
- **Precondition:** Fixture `state.json` with `lastUpdated: "not-a-date"`
- **Action:** Run gate
- **Expected result:** Exit code 1; output references invalid `lastUpdated` format
- **Edge case:** Yes — present but malformed

### gate-fails-when-completedAt-null-and-status-completed

- **Verifies:** AC2
- **Precondition:** Fixture `state.json` with a cycle block containing `status: "completed"` and `completedAt: null`
- **Action:** Run gate
- **Expected result:** Exit code 1; output identifies the specific cycle block by name/key
- **Edge case:** No

### gate-passes-when-completedAt-null-and-status-not-completed

- **Verifies:** AC2 (negative case)
- **Precondition:** Fixture `state.json` with `status: "in-progress"` and `completedAt: null`
- **Action:** Run gate
- **Expected result:** Exit code 0 for this check (no inconsistency)
- **Edge case:** No

### gate-fails-when-pipeline-state-version-absent

- **Verifies:** AC3
- **Precondition:** Fixture `pipeline-state.json` with no `version` field
- **Action:** Run gate
- **Expected result:** Exit code 1
- **Edge case:** No

### gate-fails-when-pipeline-state-features-null

- **Verifies:** AC3
- **Precondition:** Fixture `pipeline-state.json` with `features: null`
- **Action:** Run gate
- **Expected result:** Exit code 1 — null is not an array
- **Edge case:** Yes — null vs absent

### gate-fails-when-artefacts-dir-empty

### gate-fails-when-governance-gates-yml-missing-gates-key

### gate-fails-when-governance-gates-yml-gates-array-empty

## Integration Tests

### gate-identifies-specific-cycle-block-in-inconsistency-report

- **Verifies:** AC2
- **Components involved:** `run-assurance-gate.js`, `check-workspace-state.js`, fixture state.json
- **Precondition:** Fixture `state.json` has two cycle blocks — one valid, one with `status: "completed"` + `completedAt: null`
- **Action:** Run full gate; capture output
- **Expected result:** Output names the specific inconsistent cycle block (e.g. the key name or phase name of the block with the inconsistency), not just "an error was found"

### gate-timing-integration-with-full-check-suite

- **Verifies:** AC6
- **Components involved:** All five new checks running together, gate timing logic
- **Precondition:** Fixture set with all five checks passing
- **Action:** Run full gate; capture start/completedAt timestamps from gate output trace
- **Expected result:** completedAt − startedAt ≥ 5ms; all five checks report pass in trace output

## NFR Tests

### full-gate-completes-within-30-seconds

- **NFR addressed:** Performance
- **Measurement method:** Time full `run-assurance-gate.js` execution including all five new content checks; assert wall-clock < 30000ms
- **Pass threshold:** < 30 seconds
- **Tool:** Node.js `performance.now()` or Jest timer

### gate-makes-no-write-calls-to-checked-files

- **NFR addressed:** Security (read-only)
- **Measurement method:** Patch `fs.writeFile`, `fs.writeFileSync`, `fs.appendFile` with mocks before running gate; assert mocks not called on any of the five checked file paths
- **Pass threshold:** Zero write calls on `state.json`, `pipeline-state.json`, `governance-gates.yml`, or `artefacts/` directory
- **Tool:** Jest mock
