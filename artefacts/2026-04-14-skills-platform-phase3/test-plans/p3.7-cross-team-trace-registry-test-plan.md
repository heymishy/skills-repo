- `fixtures/traces/squad-alpha/story-feature-x.jsonl` — valid trace conforming to trace.schema.json; 3 entries

## Unit Tests

### get-traces-returns-matching-squad

- **Verifies:** AC2 (happy path — filter by squadId)
- **Test file:** `src/trace-registry/getTraces.test.js`
- **Precondition:** Registry is seeded with fixture traces for `squad-alpha` and `squad-beta`
- **Action:** Call `getTraces({ squadId: 'squad-alpha' })`
- **Expected result:** Returns an array containing only trace entries for `squad-alpha`; no `squad-beta` entries in result
- **RED step:** Module does not exist; function not exported

### get-traces-filter-by-date-range

- **Verifies:** AC2 (filter — date range)
- **Test file:** `src/trace-registry/getTraces.test.js`
- **Precondition:** Registry seeded with traces from 2026-03-15 and 2026-04-10
- **Action:** Call `getTraces({ filter: { dateFrom: '2026-04-01' } })`
- **Expected result:** Only traces with `aggregatedAt` or `storyCompletedAt` on or after 2026-04-01 are returned; the March entry is excluded
- **RED step:** Filter logic not yet implemented

### get-traces-filter-by-pass-rate

- **Verifies:** AC2 (filter — passRate threshold)
- **Test file:** `src/trace-registry/getTraces.test.js`
- **Precondition:** Registry seeded with squad-alpha traces: passRate 100 and passRate 60
- **Action:** Call `getTraces({ squadId: 'squad-alpha', filter: { passRate: '>= 90' } })`
- **Expected result:** Only the passRate: 100 trace is returned; passRate: 60 entry is excluded
- **RED step:** PassRate filter not yet implemented

### get-traces-returns-empty-array-for-no-match

- **Verifies:** AC2 (empty result contract)
- **Test file:** `src/trace-registry/getTraces.test.js`
- **Precondition:** Registry seeded with squad-alpha and squad-beta traces
- **Action:** Call `getTraces({ squadId: 'squad-alpha', filter: { storySlug: 'nonexistent-story' } })`
- **Expected result:** Returns `[]` — empty array, no error thrown
- **RED step:** Function throws on no-match instead of returning empty array

### get-traces-returns-empty-for-unregistered-squad

- **Verifies:** AC5 (opt-out / unregistered squad)
- **Test file:** `src/trace-registry/getTraces.test.js`
- **Precondition:** No traces exist for `squad-gamma` in the registry fixture
- **Action:** Call `getTraces({ squadId: 'squad-gamma' })`
- **Expected result:** Returns `[]` — empty array; no error, no exception thrown
- **RED step:** Function throws "squad not found" or similar error

## Integration Tests

### aggregation-workflow-uses-separate-schedule-trigger

- **Verifies:** AC3 + AC4 (schedule cadence + ADR-009 separate trigger)
- **Test file:** `tests/check-trace-commit.js` (extend) or new `tests/check-trace-registry.js`
- **Precondition:** Aggregation CI workflow YAML exists
- **Action:** Read the aggregation CI workflow YAML; verify it has a `schedule:` trigger with a cron expression equivalent to at minimum "once per 24 hours" (i.e. `cron: '0 * * * *'` or `cron: '0 0 * * *'` or similar); verify it does NOT share the same job-id or workflow file as the delivery pipeline CI
- **Expected result:** Schedule trigger present with ≤24h cadence; workflow is a separate file from the delivery pipeline CI
- **RED step:** Workflow does not exist; or it uses `push:` / `pull_request:` trigger instead of `schedule:`

### aggregation-workflow-uses-read-only-token

- **Verifies:** AC4 (ADR-009 read-only token)
- **Test file:** `tests/check-trace-registry.js`
- **Precondition:** Aggregation CI workflow YAML exists
- **Action:** Read the workflow YAML `permissions:` block; verify `contents: read` (or equivalent) and that no `write` permission is granted for the source squad repos
- **Expected result:** Permissions block present; write access not granted; read-only confirmed
- **RED step:** Permissions block absent or grants write access

### aggregation-output-path-follows-schema

- **Verifies:** AC1 (aggregation path format)
- **Test file:** `tests/check-trace-registry.js`
- **Precondition:** Aggregation script or module exists
- **Action:** Inspect the aggregation script to confirm it writes to `platform/traces/[squadId]/[storySlug].jsonl` — check the output path construction logic (string template or regex-match the path)
- **Expected result:** Output path follows `platform/traces/{squadId}/{storySlug}.jsonl` pattern
- **RED step:** Aggregation script does not exist; or writes to different path
