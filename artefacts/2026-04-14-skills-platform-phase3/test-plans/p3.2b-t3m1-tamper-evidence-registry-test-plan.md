## Unit Tests

### computes-sha256-hash-of-trace-file-as-hex-string

- **Verifies:** AC1 (hash computation function)
- **Precondition:** Fixture file at known path with content `"regulated trace content fixture\n"` written in test setup
- **Action:** Call `computeTraceHash(fixturePath)`; capture return value
- **Expected result:** Returns a hex string matching the expected SHA-256 of the fixture file content (64-character lowercase hex)
- **Edge case:** No

### sha256-hash-is-stable-across-sequential-computations-of-same-file

- **Verifies:** AC2 (hash reproducibility — same file produces the same hash)
- **Precondition:** Fixture file with fixed content written once in test setup
- **Action:** Call `computeTraceHash(fixturePath)` twice in sequence; compare both return values
- **Expected result:** Both calls return the exact same hex string; no variance from timestamps, read order, or internal state
- **Edge case:** Yes — this is the reproducibility guarantee required for auditor recomputation

### sha256-hash-differs-when-file-content-changes

- **Verifies:** AC2 (negative — any content change produces a different hash)
- **Precondition:** Fixture file A with content `"original"` and fixture file B with content `"original "` (trailing space added)
- **Action:** Compute hash of A and hash of B; compare
- **Expected result:** Hash of A !== hash of B — even one-character difference is detected
- **Edge case:** Yes — confirms the hash function is sensitive to any modification

### tamperEvidence-object-written-to-trace-file

- **Verifies:** AC4 (object presence)
- **Precondition:** Mocked post-merge workflow output: a function that writes a trace line with `tamperEvidence` populated
- **Action:** Call the write function with mock attestation data; read the written trace line; parse JSON
- **Expected result:** Parsed JSON contains a `tamperEvidence` key with a non-null value
- **Edge case:** No

### tamperEvidence-registryType-is-github-artifact-attestation

- **Verifies:** AC4 (`registryType` value)
- **Precondition:** Same as above
- **Action:** Read `tamperEvidence.registryType` from the written trace line
- **Expected result:** Value equals the string `"github-artifact-attestation"`
- **Edge case:** No

### tamperEvidence-registryRef-is-non-empty-string

- **Verifies:** AC4 (`registryRef` field)
- **Precondition:** Mock data with a non-empty attestation URL
- **Action:** Read `tamperEvidence.registryRef` from the written trace line
- **Expected result:** Non-empty string; does not throw; not null or undefined
- **Edge case:** No

### tamperEvidence-publishedAt-is-ISO8601-timestamp

- **Verifies:** AC4 (`publishedAt` field)
- **Precondition:** Mock attestation data with timestamp `"2026-04-15T00:00:00Z"`
- **Action:** Read `tamperEvidence.publishedAt` from the written trace line
- **Expected result:** Value matches ISO 8601 datetime format (parseable by `new Date()` without NaN)
- **Edge case:** No

## NFR Tests

### workflow-identity-uses-oidc-not-pat

- **NFR addressed:** Security (CI service identity — OIDC only, no personal access token)
- **Measurement method:** Inspect the post-merge workflow YAML file (`attestation-publisher.yml` or equivalent); confirm the workflow does not reference any `secrets.*` value of the form `PAT*` or `GITHUB_PAT*`; confirm the workflow uses `permissions: id-token: write` for OIDC authentication
- **Pass threshold:** Workflow YAML contains `id-token: write` in permissions; does not reference a `PAT` secret
- **Tool:** Grep / file inspection (`grep -i "pat" .github/workflows/attestation-publisher.yml` must return no matches for secret references; workflow permissions block must include `id-token: write`)

### registry-entry-retrievable-without-special-tooling

- **NFR addressed:** Audit (entry retrievable via plain HTTP GET or `git log`)
- **Measurement method:** After the first real merge with the attestation workflow active, retrieve the attestation record using `gh attestation verify --repo [owner]/[repo]` (standard GitHub CLI, no custom tooling). Alternatively, the `registryRef` URL in the trace file must be accessible via a browser or `curl` request with no API key.
- **Pass threshold:** The attestation record is reachable and the hash value can be read without proprietary tooling
- **Tool:** Manual — CI run log + `gh` CLI (standard GitHub CLI, not a custom tool)

## Test Gaps and Risks

| Gap | Reason | Mitigation |
|-----|--------|------------|
| Live Artifact Attestation publication cannot be tested locally | Requires real OIDC workflow run in GitHub Actions | Integration test covers the workflow logic with mocked API; acceptance is confirmed by CI run log inspection after first merge |
| Access control denial (AC3) is unautomatable | GitHub's Artifact Attestation access control is enforced by platform infrastructure, not application code | Manual: document in DoD artefact that the delivery team's token does not have `attestations: write` scope at the attestation endpoint; confirmed via GitHub role model documentation |
| Hash reproducibility depends on file normalisation | If trace files contain Windows CRLF line endings or variable whitespace, the same logical content may produce different hashes on different platforms | Hash is computed over raw bytes of the written file (no text normalisation); test explicitly verifies hash stability across sequential reads |
