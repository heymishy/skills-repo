## Unit Tests

### exits-1-when-trace-file-stale-beyond-24h

- **Verifies:** AC2
- **Precondition:** A trace file exists at the fixture path with `committedAt` set to 25 hours ago (Date.now() − 90000000)
- **Action:** Run `check-trace-commit.js` pointing at the fixture trace directory; capture exit code
- **Expected result:** Process exits with code 1; stdout or stderr contains the `committedAt` timestamp of the stale trace
- **Edge case:** No — standard stale case

### exits-1-when-trace-file-exactly-24h-old

- **Verifies:** AC2 (boundary)
- **Precondition:** Trace file with `committedAt` = exactly 24 hours ago (Date.now() − 86400000)
- **Action:** Run check-trace-commit.js with the fixture
- **Expected result:** Exit code 1 (>24h threshold is inclusive at the boundary)
- **Edge case:** Yes — boundary value: 24h exactly must trigger failure

### exits-0-when-trace-is-current

- **Verifies:** AC3
- **Precondition:** Trace file with `committedAt` = 1 hour ago
- **Action:** Run check-trace-commit.js with the fixture
- **Expected result:** Exit code 0; no error output
- **Edge case:** No

### exits-1-when-no-trace-file-for-branch

- **Verifies:** AC4
- **Precondition:** No trace file exists for the specified branch slug in the fixture directory
- **Action:** Run check-trace-commit.js specifying that branch; capture exit code
- **Expected result:** Exit code 1; output indicates absent branch trace
- **Edge case:** No — standard absent case

### npm-test-suite-includes-check-trace-commit

- **Verifies:** AC1
- **Precondition:** package.json exists at repo root
- **Action:** Read package.json scripts.test; check that it includes or calls `check-trace-commit.js`
- **Expected result:** The test entry in package.json (or the test runner configuration) references `check-trace-commit.js` or the file is discovered automatically by the test framework
- **Edge case:** No

## Integration Tests

### check-trace-commit-reads-origin-traces-not-local

- **Verifies:** AC2, AC3, AC4 (ADR-010 compliance)
- **Components involved:** `check-trace-commit.js`, fixture trace directory simulating `origin/traces`
- **Precondition:** Fixture directory at a path passed via argument or env var; no local `workspace/traces` directory
- **Action:** Run check-trace-commit.js with `--traces-dir` pointing to the fixture; confirm it does not attempt to read from `workspace/traces`
- **Expected result:** Script uses only the provided traces path; no fallback to local workspace/traces


### stale-trace-output-includes-timestamp-in-message

- **Verifies:** AC2 (timestamp in output)
- **Components involved:** `check-trace-commit.js`, fixture trace file
- **Precondition:** Stale trace fixture (25h ago)
- **Action:** Run check-trace-commit.js; capture stdout + stderr
- **Expected result:** Output string contains the ISO 8601 `committedAt` value from the fixture file

### current-trace-produces-no-error-output

- **Verifies:** AC3
- **Components involved:** `check-trace-commit.js`, fixture trace file
- **Precondition:** Current trace fixture (1h ago)
- **Action:** Run check-trace-commit.js; capture stdout + stderr
- **Expected result:** Exit 0; stderr is empty; stdout contains no error keywords

## NFR Tests

### completes-within-5-seconds

- **NFR addressed:** Performance
- **Measurement method:** Record start time before spawning check-trace-commit.js; record end time after process exits; assert delta < 5000ms
- **Pass threshold:** < 5000ms wall-clock time from process start to exit, including fixture I/O
- **Tool:** Node.js `performance.now()` in test setup / Jest timer assertion

### script-makes-no-write-calls

- **NFR addressed:** Security (read-only)
- **Measurement method:** Monkey-patch `fs.writeFile`, `fs.writeFileSync`, `fs.appendFile`, `fs.appendFileSync` before requiring check-trace-commit.js; assert none are called during a full run
- **Pass threshold:** Zero write-family calls invoked on any path
- **Tool:** Jest mock / sinon spy
