## Test Data Strategy

**Source:** Synthetic — mock payloads hardcoded in test files:
- Valid Teams Adaptive Card webhook payload JSON with `storyId: "p3.8-test-story"`
- Invalid Teams payload with no `storyId` field
- Valid Teams payload with invalid HMAC signature
- Valid Jira transition payload (workflow state → "Approved") with linked story ID
- Valid Jira comment payload (`body: "/approve-dor"`, issue key with story ID)
- Invalid Jira payload (missing issue key)

**PCI/sensitivity:** No — mock payloads; no real credentials or webhook URLs
**External network:** None — all tests use mock adapters; no real Teams or Jira API calls

## Unit Tests

### teams-adapter-valid-payload-writes-signed-off

- **Verifies:** AC1 (happy path — valid Teams payload)
- **Test file:** `src/approval-channel/adapters/teams-adapter.test.js`
- **Precondition:** Teams adapter module exists at `src/approval-channel/adapters/teams-adapter.js`; `process-dor-approval.js` is mocked
- **Action:** Pass the valid Teams webhook mock payload to the adapter handler; check mock was called with correct storyId and that the mock emits `dorStatus: "signed-off"`
- **Expected result:** Adapter calls `process-dor-approval.js` mock with `{ storyId: 'p3.8-test-story', status: 'signed-off' }`; returns 200
- **RED step:** Module does not exist

### teams-adapter-invalid-hmac-returns-401

- **Verifies:** AC1 (security — HMAC verification), NFR (unauthenticated calls rejected)
- **Test file:** `src/approval-channel/adapters/teams-adapter.test.js`
- **Precondition:** Adapter exists; a valid payload is tampered with an incorrect HMAC header
- **Action:** Pass valid Teams payload with an invalid HMAC signature header
- **Expected result:** Adapter returns HTTP 401; `process-dor-approval.js` mock is NOT called
- **RED step:** HMAC verification not yet implemented

### jira-transition-adapter-valid-payload-writes-signed-off

- **Verifies:** AC2 (happy path — Jira workflow transition)
- **Test file:** `src/approval-channel/adapters/jira-adapter.test.js`
- **Precondition:** Jira adapter exists; `process-dor-approval.js` mocked
- **Action:** Pass Jira transition payload (issue transitioned to "Approved" state with story link) to adapter handler
- **Expected result:** Adapter calls `process-dor-approval.js` mock with correct story ID and `status: 'signed-off'`; returns 200
- **RED step:** Module does not exist

### jira-comment-adapter-valid-approve-dor-writes-signed-off

- **Verifies:** AC2 (Jira `/approve-dor` comment path)
- **Test file:** `src/approval-channel/adapters/jira-adapter.test.js`
- **Precondition:** Jira adapter exists; `process-dor-approval.js` mocked
- **Action:** Pass Jira comment payload with `body: "/approve-dor"` and linked issue key containing story ID
- **Expected result:** Same as transition path — `process-dor-approval.js` called with correct storyId; returns 200
- **RED step:** Comment detection path not yet implemented

### missing-story-id-returns-4xx

- **Verifies:** AC4 (failure handling — missing story ID)
- **Test file:** `src/approval-channel/adapters/teams-adapter.test.js` + `jira-adapter.test.js`
- **Precondition:** Both adapters exist
- **Action:** Pass payloads with no story ID field to each adapter
- **Expected result:** Each adapter returns 4xx (400 or 422); error is logged; `process-dor-approval.js` mock NOT called
- **RED step:** Adapter does not validate story ID; passes undefined storyId to downstream

### process-dor-write-failure-returns-5xx

- **Verifies:** AC4 (failure handling — write error)
- **Test file:** `src/approval-channel/adapters/teams-adapter.test.js`
- **Precondition:** `process-dor-approval.js` mock is configured to throw an error
- **Action:** Pass valid Teams payload; `process-dor-approval.js` mock throws
- **Expected result:** Adapter returns 5xx; error is logged; no silent swallow
- **RED step:** Adapter catches error silently and returns 200

### credentials-read-from-env-vars

- **Verifies:** AC5 (no hardcoded credentials)
- **Test file:** `src/approval-channel/adapters/teams-adapter.test.js` or a grep-based verification
- **Precondition:** Adapter source files exist
- **Action:** Grep both adapter source files for any hardcoded secrets, webhook URLs, API tokens, or Jira credentials. Additionally verify that the adapter reads webhook secret from `process.env.TEAMS_WEBHOOK_SECRET` (or equivalent named env var)
- **Expected result:** No hardcoded credential strings found; env var reads present
- **RED step:** Source contains hardcoded webhook URL or token literal

## Integration Tests

### teams-adapter-conforms-to-approval-channel-interface

- **Verifies:** AC1 (ADR-006 interface conformance)
- **Test file:** `tests/check-dor-approval.js` (extend) or `src/approval-channel/interface.test.js`
- **Precondition:** Interface contract defined in `src/approval-channel/` (Phase 2 p2.8 output); Teams adapter exists
- **Action:** Import the Teams adapter and verify it exports the same handler signature defined in the approval-channel interface contract (e.g. `handler(payload, ctx)` or the exact function signature used by the interface)
- **Expected result:** Teams adapter exports conform to interface contract with no missing methods
- **RED step:** Interface contract or Teams adapter not yet present

### jira-adapter-conforms-to-approval-channel-interface

- **Verifies:** AC2 (ADR-006 interface conformance)
- **Test file:** `src/approval-channel/interface.test.js`
- **Precondition:** Interface contract and Jira adapter exist
- **Action:** Same as Teams adapter conformance test but for the Jira adapter
- **Expected result:** Jira adapter exports conform to interface contract
- **RED step:** Jira adapter not yet present

### test-suite-has-no-external-network-calls

- **Verifies:** AC3 (test isolation — no real network)
- **Test file:** `tests/check-dor-approval.js` (extend)
- **Precondition:** Both adapters' test suites can be run with `npm test`
- **Action:** Run full test suite with network interceptor active (or check that all HTTP client calls are mocked via jest.mock or nock); verify no real HTTP calls are attempted
- **Expected result:** Test suite passes with zero real network connections; mock-only
- **RED step:** Test attempts real Teams or Jira API call and fails with network error in CI
