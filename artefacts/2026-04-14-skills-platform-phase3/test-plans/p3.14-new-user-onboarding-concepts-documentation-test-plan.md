### AC2-U1 — All 12 principle and primitive files contain required 4 headings in correct order (parameterised)

- **Verifies:** AC2
- **Precondition:** All 12 files in `docs/concepts/principles/` and `docs/concepts/primitives/` exist
- **Action:** For each of the 12 files, read content and verify: (1) `## What it is` appears before `## Why it exists`, (2) `## Why it exists` appears before `## How it works`, (3) `## How it works` appears before `## What you do with it`. All four headings must be present. Uses `indexOf` on file content string.
- **Expected result:** All 12 files pass. Any file missing a heading, or headings out of order, fails individually.
- **Edge case:** Yes — a file with the 4 headings in wrong order (e.g. `## How it works` before `## Why it exists`) must fail.

### AC2-U2 — No required principle/primitive file contains h1-level heading that replaces a required h2 section

- **Verifies:** AC2 — heading level must be `## ` (h2), not `# ` (h1)
- **Precondition:** All 12 files exist
- **Action:** For each file, verify `## What it is` (two hashes + space) is present — not `# What it is` (one hash)
- **Expected result:** All files use `##` level. A file using `#` instead fails.
- **Edge case:** No.

- **Precondition:** All 20 concept files + README.md exist

- **Edge case:** No — false positives are acceptable; the test is conservative by design.

## NFR Tests

### NFR-Readability — Non-engineer readability (manual)

- **NFR addressed:** Readability
- **Measurement method:** Manual — a non-engineering reviewer (BA, PM, or risk function reviewer) reads any 3 concept files (one from each subdirectory) and attempts to describe each in one paragraph to a colleague. If they require a coding explanation, the file fails.
- **Pass threshold:** All 3 sampled files are comprehensible without engineering background.
- **Tool:** Manual — see AC verification script Scenario NFR-R1.

### NFR-Completeness — Depth sufficient for one-paragraph description (manual)

- **NFR addressed:** Completeness
- **Measurement method:** Manual — reviewer reads each file and judges whether a correct one-paragraph description of the concept could be given to a colleague from that file alone.
- **Pass threshold:** All 20 concept files pass the one-paragraph test.
- **Tool:** Manual — see AC verification script Scenario NFR-C1.

### NFR-Security — Automated credential scan

- **NFR addressed:** Security
- **Measurement method:** Automated regex scan — see NFR-Security-U1 above.
- **Pass threshold:** Zero matches on any credential-like pattern.
- **Tool:** Node.js test with `fs.readFileSync` + regex.

### NFR-Accuracy — Guardrails alignment

- **NFR addressed:** Accuracy
- **Measurement method:** See NFR-Accuracy-I1 above (integration test).
- **Pass threshold:** All 6 principle concept files correspond to a named entry in architecture-guardrails.md.
- **Tool:** Node.js test.

## Out of Scope for This Test Plan

- Testing the *content quality* of each file beyond structural checks — depth and accuracy beyond structural conformance is a manual reviewer judgment.
- Testing the `## Further reading` optional fifth section — it is optional per AC2 and is not required.
- Testing the `## How it relates to` optional section in building-block files — optional per AC6.
- Testing that ONBOARDING.md has no *other* changes beyond the callout addition — that is a PR diff review concern, not a test concern.
- E2E browser testing — this story produces no code; no browser behaviour is involved.
