# DoR Contract — Define and document the governed squad contribution flow for new skills and eval additions

**Story reference:** p3.12-squad-contribution-flow.md
**Date:** 2026-04-18
**Status:** BLOCKED — awaiting p3.4 DoD-complete

---

## What will be built

Two documentation files:

1. `docs/squad-contribution-guide.md` — complete contribution flow covering:
   - Feature branch proposal process
   - Required PR contents: SKILL.md + EVAL.md (with traceId + failurePattern per p3.4) + performance evidence
   - Human approval gate: designated platform reviewer sign-off required (not auto-assigned)
   - Merge path: platform team reviews and merges; contributing squad does not self-merge
   - PR description template guidance

2. `CONTRIBUTING.md` (created or modified at repo root) — "Contributing to the Skills Platform" section with link to docs/squad-contribution-guide.md and note that delivery-repo changes (stories, artefacts, workspace) do not require this process.

---

## What will NOT be built

- No CI automation for contribution review — Phase 4
- No onboarding of existing external SKILL.md contributions — Phase 4
- No CODEOWNERS modifications or branch protection rule changes
- No scripts, GitHub Actions workflows, or any code files
