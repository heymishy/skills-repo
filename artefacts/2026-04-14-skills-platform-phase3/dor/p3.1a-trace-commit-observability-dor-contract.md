# DoR Contract: Add trace-commit observability to npm test and skill guidance

---

## What will be built

1. **`tests/check-trace-commit.js`** — new Node.js script (CommonJS). Reads `origin/traces` latest commit timestamp using a git command (`git log --remotes=origin/traces -1 --format=%ct` or equivalent read-only command). Computes elapsed time in hours. Exits 1 if elapsed hours > 24 or if the branch is absent. Exits 0 if current. Prints a human-readable status line including the last commit timestamp and hours elapsed.

2. **`package.json`** — modified to add `node tests/check-trace-commit.js` (or equivalent) to the npm test script sequence.

3. **`.github/skills/verify-completion/SKILL.md`** — modified to add a diagnostic commands section covering: how to run `git log origin/traces -1`, interpretation of the stale-branch error output, and corrective action.

4. **`.github/skills/trace/SKILL.md`** — modified to add a "Traces Branch Health" section that describes how to check whether `origin/traces` is current, and includes the last commit timestamp in the section output.

---

## What will NOT be built

- No alerting or notification outside `npm test` stdout/stderr
- No auto-remediation, workflow re-triggering, or corrective dispatch
- No configurable staleness threshold — 24 hours is fixed in code
- No writes to `origin/traces` or any other remote — read-only operations only
- No changes to `pipeline-viz.html`
- No changes to `pipeline-state.schema.json`
- No new npm packages

## Assumptions

- `git` is available on PATH in any environment where `npm test` runs.
- `origin` remote exists and points to the GitHub repo.
- The 24-hour threshold is based on commit timestamp of most recent commit on `origin/traces`, not workflow wall-clock time.
- `origin/traces` may legitimately be absent on a fresh repository (AC4 handles this — should not exit unhandled).
