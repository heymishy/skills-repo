# DoR Contract: Add four mandatory T3M1 trace fields to schema and gate enforcement

**Story:** artefacts/2026-04-14-skills-platform-phase3/stories/p3.2a-t3m1-trace-fields.md
**DoR artefact:** artefacts/2026-04-14-skills-platform-phase3/dor/p3.2a-t3m1-trace-fields-dor.md
**Contract version:** 1

---

## What will be built

1. **`.github/pipeline-state.schema.json`** — four new field definitions added (AC1):
   - `standardsInjected`: object or array — list of standards identifiers with hashes; maps to MODEL-RISK Q2.
   - `watermarkResult`: object — pass/fail and passRate fields; maps to MODEL-RISK Q5.
   - `stalenessFlag`: boolean; maps to MODEL-RISK Q6.
   - `sessionIdentity`: object — sessionId (hash/truncated only, no PII), agentType, startedAt (ISO 8601 timestamp); maps to MODEL-RISK Q7.
   All four added in the same commit as gate logic per ADR-003.

2. **`.github/scripts/run-assurance-gate.js`** — extended with T3M1 trace field enforcement (AC2 + AC3 + AC5):
   - For regulated story traces: validate all four fields are non-null; if any field is missing or null, exit 1 and report the specific missing field name.
   - For non-regulated story traces: skip field validation (no retroactive fail).
   - Discriminator logic: distinguish regulated vs non-regulated using the existing story classification field confirmed from the p3.1b implementation (see Assumptions below).

3. **`docs/MODEL-RISK.md`** — four entries added (AC4, manual documentation):
   - One entry per new field: field name, MODEL-RISK question it answers (Q2/Q5/Q6/Q7), description of what a passing reviewer should see in the field, and a reference to the schema definition.
