# DoR Contract: Add estimation calibration as a feedback dimension in the improvement agent

**Story:** artefacts/2026-04-14-skills-platform-phase3/stories/p3.11-estimation-calibration-eval.md
**DoR artefact:** artefacts/2026-04-14-skills-platform-phase3/dor/p3.11-estimation-calibration-eval-dor.md
**Contract version:** 1
**Assessed:** 2026-04-18

---

## What will be built

1. **`src/improvement-agent/calibration.js`** — new Node.js CommonJS module. Reads `workspace/estimation-norms.md` and/or `workspace/results.tsv` to compute the underestimation rate across features with E3 actuals. Triggers proposal generation when underestimation rate > `ESTIMATION_UNDERRUN_THRESHOLD` (0.30) across ≥ `ESTIMATION_MIN_FEATURE_COUNT` (3) features. Stages the proposal at `workspace/proposals/estimation-calibration-[YYYY-MM-DD].md`. Returns silently when conditions are not met. Named constants defined at module level.

2. **`src/improvement-agent/index.js` (or equivalent entry point)** — modified to wire in `calibration.js` as a detection dimension alongside existing dimensions.

---

## What will NOT be built

- No automated application of calibration proposals to SKILL.md or estimation-norms.md
- No overestimation detection (Phase 4)
- No changes to how `/estimate` records actuals (E3 mode)
