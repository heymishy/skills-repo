## DoD Observations

PR #79 initially failed the "Trace Validation / Validate traceability chain" CI check. Root cause was a pre-existing bug in `scripts/validate-trace.sh` (introduced when Phase 3 switched to a flat `feature.stories[]` structure with short slugs) — not a defect in the p3.1a deliverable. Hotfix was applied via PR #82 (merged same day, 2026-04-15) before p3.1a was merged. The deliverable itself is sound. This is recorded here for traceability — the validate-trace bug would have blocked all Phase 3 agent PRs without the fix.

The `check-trace-commit.js` script is implemented as a dual-mode file: when invoked with `--traces-dir` or `--check-origin` flags it acts as a diagnostic tool; when invoked with no args (npm test mode) it runs its own test suite inline. This is consistent with the pattern established for other governance scripts in `tests/`.
