# Story: Add coupled-change workflow debugging pattern to /systematic-debugging skill

**Epic reference:** artefacts/2026-04-14-skills-platform-phase3/epics/e1-governance-chain-integrity.md
**Discovery reference:** artefacts/2026-04-14-skills-platform-phase3/discovery.md
**Benefit-metric reference:** artefacts/2026-04-14-skills-platform-phase3/benefit-metric.md
**Improvement source:** workspace/learnings.md — cross-layer CI-fix coupling pattern surfaced by /improvement run 2026-04-16

## User Story

As a **platform maintainer**,
I want the /systematic-debugging skill to include an explicit coupled-change workflow pattern for failures that cross `script → schema → state data → CI` contract boundaries,
So that agents do not bundle multi-layer fixes together and introduce shape drift or CI parity failures that are harder to root-cause than the original bug.

## Benefit Linkage

**Metric moved:** MM3 — Priority 1 hardening: reduction in governance check failures per story
**How:** During Phase 2/3 delivery, several CI failures were caused by bundled multi-layer fixes that introduced secondary problems (schema shape drift, state data incompatibility). A explicit pattern in the debugging skill prevents agents from taking that shortcut by providing a structured single-axis loop protocol for coupled-change scenarios.

## Architecture Constraints

- Modifications are confined to `.github/skills/systematic-debugging/SKILL.md` — no code or schema changes.
- The new section must be inserted before the existing `## Integration` section.
- The section heading must be `## Coupled-change workflow (use when a fix crosses contract layers)` to be discoverable via search.
- The five numbered steps must be preserved as the primary content of the section.

## Dependencies

- **Upstream:** None.
- **Downstream:** None — this is a new defensive section in the skill. Existing phases and quick reference table are not modified.

## Acceptance Criteria

**AC1 — Coupled-change workflow section present and positioned correctly**
Given `.github/skills/systematic-debugging/SKILL.md` is read,
When the reader locates the file,
Then a `## Coupled-change workflow (use when a fix crosses contract layers)` section is present, positioned before the `## Integration` section.

**AC2 — Section contains exactly the five required steps**
Given the coupled-change workflow section is present,
When the numbered list in the section is read,
Then it contains exactly five steps: (1) Baseline snapshot first, (2) Single-axis fix loops, (3) Dual-path hypothesis logging, (4) Guard against shape drift, (5) CI parity check early — each with substantive description, not just a heading.

**AC3 — Section includes the trigger condition for when the pattern applies**
Given the section is present,
When the introductory text is read,
Then it states the trigger condition: the pattern applies when a fix chain crosses `script → schema → state data → CI` (or equivalent multi-layer contract boundary), and includes a "this pattern applies whenever" statement identifying confirmation triggers (fix for check A reveals problem in check B; fixing script requires also fixing schema which requires also fixing state data).
