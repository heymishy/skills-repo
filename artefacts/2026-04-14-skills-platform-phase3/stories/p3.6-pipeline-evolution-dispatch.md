# Story: Add forward-pointer and Closes guidance to issue-dispatch SKILL.md

**Epic reference:** artefacts/2026-04-14-skills-platform-phase3/epics/e4-windows-ci-parity-pipeline-evolution.md
**Discovery reference:** artefacts/2026-04-14-skills-platform-phase3/discovery.md
**Benefit-metric reference:** artefacts/2026-04-14-skills-platform-phase3/benefit-metric.md

## User Story

As a **squad tech lead** dispatching a DoR-signed-off story to the GitHub Copilot coding agent,
I want the `issue-dispatch` SKILL.md to include a forward-pointer section at the `/definition-of-ready` exit and auto-close guidance (`Closes #[issue]`) in the dispatch output template, so that the agent issue is automatically closed by the implementation PR without a manual lookup.

## Benefit Linkage

**Metric moved:** MM2 — Outer loop self-sufficiency: zero blocking lookups
**How:** A squad tech lead should not need to look up "how do I make the PR close the issue automatically" after running `/issue-dispatch`. The Closes keyword guidance in the dispatch output removes a recurring lookup that currently adds friction to every post-dispatch handoff.

## Architecture Constraints

- PAT-07 applies: this story modifies two files that share the same exit point (`/definition-of-ready → /issue-dispatch`). Both changes are grouped in a single PR — the SKILL.md forward-pointer and the dispatch output template must be delivered together.
- Changes are to `.github/skills/` files — governed by the platform change policy (branch → PR → platform team review → merge). The coding agent opens a draft PR; it does not merge.

## Dependencies

- **Upstream:** None — can begin immediately.
- **Downstream:** None within Phase 3.

## Acceptance Criteria

**AC1:** Given the updated `/issue-dispatch` SKILL.md, when the skill generates dispatch output for a DoR-signed-off story, then the generated GitHub issue body includes a `Closes #[issueNumber]` line in the PR description guidance section — where `[issueNumber]` is replaced by the issue number of the open dispatch issue.

**AC2:** Given the updated `/definition-of-ready` SKILL.md, when the skill produces its sign-off block, then the sign-off section includes a forward-pointer to `/issue-dispatch` with a note that the dispatch skill generates the `Closes #[issue]` guidance automatically — the operator does not need to add it manually.

**AC3:** Given a PR created from a coding agent run, when that PR contains `Closes #[issueNumber]` in its description, then GitHub automatically closes the corresponding dispatch issue on PR merge — this is validated by reviewing the GitHub issue auto-close mechanism documentation, not by a live run.

**AC4:** Given both SKILL.md files are modified in a single PR, when the PR is reviewed, then the diff shows only the two SKILL.md changes described in AC1 and AC2 — no unrelated changes are bundled in the same PR (PAT-07 compliance, AP-07 anti-pattern avoided).

## Out of Scope

- Changes to how `pipeline-state.json` records dispatch — only the issue body template and the SKILL.md forward-pointer are in scope.
- Changes to any other skill's exit or handoff language — only `/issue-dispatch` and `/definition-of-ready` are touched.
- Automated testing of the GitHub auto-close mechanism — the AC is satisfied by documentation review.

## NFRs

None — documentation-only changes, no runtime impact.

## Complexity Rating

**Rating:** 1
**Scope stability:** Stable — two SKILL.md files, one each, no implementation code.
