# Story: Produce AGENTS.md and alternative-agent tooling compatibility matrix

**Epic reference:** artefacts/2026-04-14-skills-platform-phase3/epics/e6-enterprise-adapters-agents-validation.md
**Discovery reference:** artefacts/2026-04-14-skills-platform-phase3/discovery.md
**Benefit-metric reference:** artefacts/2026-04-14-skills-platform-phase3/benefit-metric.md

## User Story

As an **enterprise adopting organisation** evaluating the skills pipeline for use with non-GitHub-Copilot coding agents,
I want a verified compatibility matrix covering the tools their teams already use (Cursor, Claude Code, and equivalent enterprise coding agents), so that I can make an informed adoption decision without commissioning a live delivery run.

## Benefit Linkage

**Metric moved:** M5 — Alternative agent tooling compatibility matrix produced (minimum signal)
**How:** Enterprises with mixed or non-GitHub tooling need a documented compatibility position before committing to adoption. The matrix provides that signal without requiring a full delivery run (which is deferred to Phase 4 per ASSUMPTION-04 confirmation). M5 target is satisfied when the matrix is published and a Phase 4 story for a live delivery run is registered.

## Scope Note — Phase 3 constraint

**This story explicitly excludes a live delivery run.** The compatibility matrix is produced through structured documentation review, prompt-pattern analysis, and checklist verification only. A real end-to-end delivery using a non-GitHub-Copilot agent is a Phase 4 story. The Phase 4 story must be registered in `workspace/` before this story is considered DoD-complete. This constraint was confirmed by the operator on 2026-04-14 per ASSUMPTION-04 gate.

## Architecture Constraints

- The output is a markdown document: `docs/agent-compatibility-matrix.md`.
- The matrix must be produced by systematic review of each tool's documentation against a defined checklist — not by a live delivery run.
- The checklist covers: instruction file surface resolution, system prompt delivery on task start, DoR handoff format recognition, trace generation mechanism (or absence), and known gaps blocking a live run.
- No changes to SKILL.md files, pipeline scripts, or CI configuration result from this story.

## Dependencies

- **Upstream:** None — documentation review only, no code dependencies.
- **Downstream:** Phase 4 live delivery run story (to be registered as part of this story's AC).

## Acceptance Criteria

**AC1:** Given `docs/agent-compatibility-matrix.md`, when it is reviewed, then it contains a row for each of the following tools: GitHub Copilot (baseline), Cursor, Claude Code, and one additional enterprise-equivalent agent (e.g. Amazon Q Developer or JetBrains AI). Each row contains: tool name, instruction surface resolution (yes/no/partial), system prompt delivery on task start (yes/no/partial), DoR handoff format compatibility (yes/no/partial), trace generation (yes/no/not supported), and known gaps column.

**AC2:** Given each "partial" or "no" entry in the compatibility matrix, when the known gaps column is reviewed, then it contains a specific, actionable description of the gap (e.g. "Cursor does not read `.github/copilot-instructions.md` directly — requires manual copy to `.cursorrules`") — not a generic "unknown" entry.

**AC3:** Given the matrix, when the Phase 4 story requirement is reviewed, then `workspace/phase4-backlog-agent-live-delivery-run.md` exists and contains: the story title, the confirmed Phase 4 gate condition (ASSUMPTION-04 operational confirmation), and the minimum prerequisites for a live run (which tools, which story, which squad).

**AC4:** Given this story's scope, when the PR is reviewed, then no SKILL.md files, no pipeline scripts, and no CI configuration files are modified — the only file changes are `docs/agent-compatibility-matrix.md` and `workspace/phase4-backlog-agent-live-delivery-run.md`.

**AC5:** Given the compatibility matrix is published, when the M5 benefit metric is assessed, then the matrix is the evidence artefact — M5 is satisfied at this tier (matrix produced) with the Phase 4 live run as the next evidence tier.

## Out of Scope

- Running a real delivery using Cursor, Claude Code, or any non-GitHub-Copilot agent — Phase 4.
- Modifying any SKILL.md to support non-GitHub-Copilot agent invocation — Phase 4.
- Publishing tooling-specific configuration files for alternative agents — Phase 4 (e.g. `.cursorrules` equivalents).
- Evaluating more than 4 tools — additional tools can be added in Phase 4 based on enterprise demand.
