- **1-L1** [AC quality — AC5] — AC5 closes with "The gate must distinguish between regulated and non-regulated stories using an existing story classification field" without naming the specific field. The dependency note references p3.1b (assurance gate substantive checks) as the recommended upstream story, which likely defines this field, but the AC should name it or cross-reference the story that defines it (e.g. "the `regulated` boolean defined in p3.1b").
