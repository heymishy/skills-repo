### Step 2 — Contract Proposal

**Contract Proposal — Payment status change audit trail**

**What will be built:**
- Module `src/payments/audit-trail.js` with `appendAuditRecord(event)` function
- Subscribes to payment status change events from the state machine (ham.7)
- Appends JSONL records to `logs/payment-audit.jsonl` synchronously within 1 second
- Record schema: `{ timestamp (ISO8601), paymentRef, fromStatus, toStatus, operator, correlationId }`
- Module exports `appendAuditRecord(event)` and `_getLogPath()` (test helper only)
- Errors thrown as-is to caller (state machine) for handling
- Append-only — no truncate or overwrite functions exposed

**What will NOT be built:**
- Application-layer encryption (handled by infrastructure at storage layer)
- Audit record query UI or reporting interface (separate future story)
- Log rotation and archival (infrastructure responsibility)
- Tamper-evident hashing (considered and deferred in /decisions)

**How each AC will be verified:**

| AC | Test approach | Type |
|----|---------------|------|
| AC1 (status change → JSONL record, ≤1s, all fields) | Unit test: mock state machine event, spy on file append, verify record written with all required fields within 1s | unit |
| AC2 (record passes JSON schema validation; all required fields; no extraneous fields) | Unit test: parse written record, validate schema, verify no extra top-level fields | unit |
| AC3 (no truncate/overwrite functions) | Unit test: inspect module API surface, verify only `appendAuditRecord` and `_getLogPath` exported | unit |
| AC4 (write failure → error thrown) | Unit test: mock fs.appendFileSync throw, verify error propagated to caller | unit |

**Assumptions:**
- Payment state machine emits status change events synchronously or makes them available to this module
- The `logs/` directory exists and is writable at runtime
- Operator identity (for the `operator` field) is available from session context
- Correlation ID is provided by the event or is generated by the module
- File I/O errors should not be caught — passed to caller

**Estimated touch points:**
Files: `src/payments/audit-trail.js` (new), `logs/payment-audit.jsonl` (created at runtime)
Services: Payment processing service (event source), state machine
APIs: File system (fs.appendFileSync)
