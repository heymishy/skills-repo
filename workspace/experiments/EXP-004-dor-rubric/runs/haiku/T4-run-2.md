### Step 2 — Contract Proposal

**Contract Proposal — Payment status change audit trail**

**What will be built:**
- Module `src/payments/audit-trail.js` with `appendAuditRecord(event)` function
- JSONL append-only writer to `logs/payment-audit.jsonl`
- Subscribes to payment status change events from state machine (ham.7)
- Record schema: `{ timestamp: ISO8601, paymentRef: string, fromStatus: string, toStatus: string, operator: string, correlationId: string }`
- Synchronous append within 1 second of event receipt
- Errors thrown as-is (not swallowed) to caller for handling
- Append-only API — no truncate or overwrite functions exposed

**What will NOT be built:**
- Application-layer encryption (infrastructure responsibility)
- Audit record query UI or reporting interface (separate future story)
- Log rotation and archival (infrastructure responsibility)
- Tamper-evident hashing (deferred; see /decisions)

**How each AC will be verified:**

| AC | Test approach | Type |
|----|---|---|
| AC1 (status change → JSONL record, ≤1s, all fields) | Unit test: mock state machine event, spy on file append, verify record written with all fields within 1s | unit |
| AC2 (record passes JSON schema; no extraneous fields) | Unit test: parse written record, validate schema, verify no extra top-level fields | unit |
| AC3 (append-only API) | Unit test: inspect module API surface, verify only `appendAuditRecord` and `_getLogPath` exported | unit |
| AC4 (write failure → error thrown) | Unit test: mock fs.appendFileSync throw, verify error propagated to caller | unit |

**Assumptions:**
- Payment state machine emits status change events synchronously or makes them available to this module
- `logs/` directory exists and is writable at runtime
- Operator identity available from session context
- Correlation ID provided by event or generated by module
- File I/O errors are not caught — passed to caller

**Estimated touch points:**
Files: `src/payments/audit-trail.js` (new), `logs/payment-audit.jsonl` (created at runtime)
Services: Payment processing service, state machine
APIs: File system (fs.appendFileSync)
