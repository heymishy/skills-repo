### Scoring notes

**Story testability (2 not 1):** The S9 run scored 1 on this dimension because of interpretive language in Story 3.1 AC3 ("plainly discloses", "if applicable under the scheme's terms"). S10 does not have equivalent interpretive language. The most potentially ambiguous AC is Story 3.1 AC1 ("qualified New Zealand solicitor"), but this is correctly handled in the test plan as a manual compliance review step — the test plan acknowledges it cannot be automated and provides a plain-language verification script step instead. This is appropriate test design, not a testability failure: the DoR W1 warning acknowledges the limit and the gate (Legal Counsel sign-off) resolves it. Rubric score 2 is appropriate.

**DoR gate quality (1 not 2):** The PPSR gate (Story 3.1 AC1, AC2) consistently refers to "Legal Counsel" as the sign-off party. While this is a scoped function designation, the rubric requires a named responsible party at the time of DoR sign-off (example: "Compliance Officer" is used for C1/C2 gates in this very artefact). "Legal Counsel" is equivalent to "a named legal reviewer" in S9 — a function designation that defers identity to execution time. The distinction is subtle: C1/C2 use "Compliance Officer" (a named role within the organisational hierarchy) while the PPSR gate uses "Legal Counsel" (an external function that could be any of several firms/individuals). This gap is small and does not compromise the overall regulatory gate quality of the artefact — the C1, C2, and C5 gates are exemplary — but it is present and prevents a score of 2.

**C5 hard gate (confirmed fully present and enforced):** The C5 gate is implemented at two levels: (1) Story 1.1 AC3 gates all Story 2.x and 3.x development on `BS11_NOTIFICATION_STATUS = ACK_RECEIVED`; (2) Story 1.1 AC1 requires `self_disclosure_filed: true` on any late-filing notification; (3) T-BS11-007 tests the late-filing self-disclosure requirement directly; (4) T-BS11-003 adversarial case tests that the `rbnz_acknowledgement_reference` must be non-empty (preventing status spoofing). The DoR coding agent instruction 1 specifically addresses the self-disclosure implementation and prohibits bypass patterns. C5 gate quality is the strongest element of this DoR.
