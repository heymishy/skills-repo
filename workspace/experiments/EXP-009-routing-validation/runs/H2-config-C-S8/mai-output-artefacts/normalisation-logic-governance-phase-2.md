# Epic: Normalisation Logic Governance & Activation

**Epic slug:** normalisation-logic-governance-phase-2
**Feature slug:** regulatory-reporting-pipeline-automation
**Status:** Future-phase gate; not in the current Phase 1 scope

## Goal
Resolve the governance preconditions for the normalisation transformation layer before any production activation.

## Gate conditions
- FMA s.4.2(a) documentation of the normalisation logic.
- FMA s.4.2(b) independent technical review.
- FMA s.4.2(c) governance sign-off from the responsible finance officer.
- FMA s.4.2(d) filing of the required FMA and RBNZ notifications.
- FMA s.4.2(e) retention of the legacy macro artefact for audit completeness.

## Phase 1 rule
This epic is intentionally excluded from the current implementation scope because the normalisation layer is a material change and must not be activated in production until the Compliance Officer issues written clearance.
