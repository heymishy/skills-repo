# User Experience Policy Floor

**Discipline:** ux
**lastReviewedBy:** Hamish
**lastReviewedDate:** 2026-04-11

These floors are binary. A delivery either meets them or it does not. No domain or squad override may relax these floors.

## Overview

These binary floors define the minimum delivery bar for UX work. No domain, squad, or surface variant may relax these requirements.

## Outcomes

- Users with accessibility needs are not systematically excluded from primary functionality.
- Usability failures are identified before a surface reaches the full user population.
- Design decisions are traceable to validated user needs, not unreviewed assumptions.

## Requirements

- MUST validate key user journeys with at least one representative user session before a surface is released to production so that critical usability failures are not discovered post-launch
- MUST meet the accessibility conformance level declared in the project brief before release so that assistive technology users can complete primary tasks
- MUST record the user research evidence for each significant design decision before the PR is merged so that reviewers can distinguish validated choices from assumptions

## Out of Scope for this standard

- Brand identity and visual language decisions
- Back-end API and data design
- Specific design tooling selection
