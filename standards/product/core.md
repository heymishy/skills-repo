# Product Management Core Standards

**Discipline:** product
**lastReviewedBy:** Hamish
**lastReviewedDate:** 2026-04-11

These standards define the universal baselines for product management delivery across all surfaces and domains. Domain and squad extensions may add requirements; they may not relax these baselines.

## Overview

Product management standards govern how teams discover user needs, define outcomes, and measure whether delivered changes produce the intended results. These standards apply to any delivery that introduces new user-facing functionality or changes existing product behaviour.

## Outcomes

- Delivery teams can trace each feature back to a validated user need and a measurable outcome so that scope decisions are auditable.
- Benefit metrics are defined before delivery begins so that the team can detect whether the intended outcome was achieved post-release.
- Scope is managed against declared priorities so that unplanned scope does not displace committed work without an explicit decision record.

## Requirements

- MUST define at least one measurable benefit metric for each feature before implementation begins so that post-delivery success can be assessed against a declared baseline
- MUST record the user or business need that justifies each new feature in a discovery artefact before implementation begins so that scope decisions are traceable to validated demand
- MUST obtain explicit approval for any scope change that increases story count or delivery timeline beyond the declared estimate so that scope growth is intentional and visible
- SHOULD baseline current user behaviour before a feature ships so that post-release measurements can be compared against a pre-change reference point
- SHOULD review benefit metric results within an agreed window after release so that teams detect whether intended outcomes were achieved before moving on
- MAY defer metric baselining for low-risk incremental improvements when the change is too small to produce a detectable signal, provided a decision log entry documents the rationale

## Out of Scope for this standard

- Organisational product strategy and roadmap prioritisation — those are programme-level decisions
- Specific [FILL IN: product analytics tooling] — any tool that produces auditable metric records satisfies this standard
- UX research methodology — that is governed by the UX discipline standard
