# Product Management Policy Floor

**Discipline:** product
**lastReviewedBy:** Hamish
**lastReviewedDate:** 2026-04-11

These floors are binary. A delivery either meets them or it does not. No domain or squad override may relax these floors.

## Overview

These binary floors define the minimum delivery bar for product management work. No domain, squad, or surface variant may relax these requirements.

## Outcomes

- Features are traceable to validated user or business needs so that scope decisions are auditable.
- Every feature has a declared benefit metric before implementation so that post-delivery assessment is possible.
- Scope changes beyond declared estimates have explicit approval records so that scope growth is visible and intentional.

## Requirements

- MUST record the user or business need that justifies each new feature in a discovery artefact before implementation begins so that scope decisions are traceable to validated demand
- MUST define at least one measurable benefit metric for each feature before implementation begins so that post-delivery success can be assessed against a declared baseline
- MUST obtain an explicit approval record for any scope change that increases story count or delivery timeline beyond the declared estimate so that scope growth is visible and intentional

## Out of Scope for this standard

- Organisational roadmap prioritisation
- Product analytics tooling selection
- UX research methodology
