# SaaS-GUI Policy Floor

**Surface:** saas-gui
**lastReviewedBy:** TBD
**lastReviewedDate:** TBD

These floors are binary. A delivery either meets them or it does not. No domain or squad override may relax these floors.

## Governance Criteria

- MUST obtain change-review approval (`gui.change_review`) so that all SaaS GUI configuration changes are reviewed and approved before deployment
- MUST attach screenshot evidence (`gui.screenshot_evidence`) so that the state of the GUI before and after a change is documented and reviewable
- MUST configure access control review (`gui.access_control`) so that all GUI-facing features require an access control assessment before release
